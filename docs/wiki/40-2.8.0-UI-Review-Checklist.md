# 2.8.0 UI Review Checklist

Status: active milestone checklist for `2.8.0-a`

Roadmap source:

- `docs/wiki/07-Release-Roadmap.md` section `2.8.0 — UI Refinement Sprint (Cross-Device Consistency)`

## Slice

- Milestone slice: `2.8.0-a — intended desktop/mobile navigation behavior + first review pass`

## Checklist

### Shared navigation and shell

- Navigation starts open by default on desktop.
- Desktop navigation can be closed from the nav header toggle.
- When desktop navigation is closed, it reopens on hover.
- Hover-open navigation can be pinned open from the nav header toggle.
- Mobile navigation opens from the top-left header hamburger.
- Navigation remains usable for:
  - library browsing
  - import
  - profile
  - admin sections

### Library review

- Search, quick filters, view toggle, sort, and add actions remain reachable on narrow screens.
- Bulk-selection summary remains readable on mobile and desktop.
- Select, clear, and delete-selected controls do not wrap into unusable layouts.
- Pagination/footer state stays in sync after bulk delete.
- Card and list selection affordances remain visually obvious.

### Import review

- Import page keeps primary actions visible without horizontal overflow on narrow screens.
- Tab/subsection controls remain reachable and wrap acceptably.
- Barcode results and job/audit surfaces remain readable on mobile and desktop.
- Empty and disabled-library states remain understandable.

### Profile review

- Profile form fields remain readable on narrow screens.
- Personal access token create/revoke surfaces remain usable on mobile.
- One-time token reveal area wraps safely without layout breakage.

### Admin review

- Admin pages use consistent page padding across desktop and mobile.
- Admin lists/forms do not rely on oversized desktop-only spacing.
- High-frequency operator actions remain visible without horizontal overflow.
- Activity, settings, users, spaces, feature flags, and integrations remain reachable through the intended navigation model.

### Inference from code review

- Library selection controls should read more clearly on narrow screens after the existing selection toolbar and padding changes.
- Import, Profile, and Admin views should avoid the most obvious mobile crowding caused by fixed `p-6` outer spacing.
- The remaining cross-device work should focus on visual consistency and spacing rather than replacing the intended desktop hover/pin behavior.

### Blocked / still unverified

- Full visual/browser review is not fully automated in the current agent environment:

  - no screenshot-based local UI harness was available for the running app
- Manual browser confirmation is still required for:
