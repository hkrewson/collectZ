# collectZ v3.20.0

## Version and date

- Version: `3.20.0`
- Date: 2026-06-20

## Milestone target and status

- Roadmap slice: `3.20.0 - Plex True Sync Workflow UI Baseline`
- Status: completed

## Summary

This minor release starts the Plex true-sync workflow redesign. The Plex integration tab is split into setup, sync, webhook, and advanced sections, and the old standalone operating-model explainer has been removed from the UI.

## What changed

- Added a documented Plex true-sync workflow plan.
- Split the Plex integration settings surface into `Setup`, `Sync`, `Webhook`, and `Advanced`.
- Moved credentials and detected libraries into `Setup`.
- Kept reconciliation controls and conflict review in `Sync`.
- Kept receiver setup and receiver state in `Webhook`.
- Moved provider diagnostics and now-playing display controls into `Advanced`.
- Removed the separate Plex operating-model explainer from the settings UI.
- Updated backlog and Plex modernization docs so future sync work points to the new true-sync plan.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- No schema migration is required.
- Existing Plex settings, webhook receiver state, display preferences, scheduler readback, and conflict reviews are preserved.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.20.0`.
- Verified locally:
  - `node --check backend/scripts/unit-tests.js`
  - `node --check tests/playwright/specs/integrations.browser.spec.js`
  - `npm --prefix frontend run build`
  - `node backend/scripts/unit-tests.js`

## Rollback guidance

- Roll back to the previous backend and frontend images if the Plex integration tab fails to load.
- No database rollback is required.

## Known issues and follow-up

- This release reorganizes the Plex workflow UI; it does not yet add persisted sync cadence controls or webhook reachability validation.
- The next Plex sync follow-ups are tracked in `docs/wiki/52-Plex-True-Sync-Workflow-Plan.md`.
