# collectZ v3.4.86

## Kavita Digital Library Import/Sync Foundation

This patch adds the first Kavita import path on top of the `3.4.85` connection foundation. Admins can queue a Kavita import from Admin > Integrations, and the backend imports Kavita series through the native API into existing media rows without creating repeat-sync duplicates.

### What changed

- Added `/api/media/import-kavita` with synchronous and queued import support.
- Added Kavita series normalization into the existing media import/upsert pipeline.
- Uses provider-scoped stable IDs like `kavita:series:<id>` so Kavita rows do not collide with CWA/Calibre or other provider IDs.
- Reuses matching existing non-Kavita titles in the active library when the match is safe, then attaches Kavita provider metadata.
- Preserves manual/non-Kavita metadata while adding Kavita provider linkage.
- Added an `Import from Kavita` action to the Kavita integration settings.
- Added Docker-friendly smoke coverage proving non-Kavita title reuse and repeat-sync idempotency against a local fake Kavita-compatible API.
- Kept metadata writeback to Kavita, embedded reading, and reading-progress sync out of scope.
