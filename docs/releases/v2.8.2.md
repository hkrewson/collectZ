# v2.8.2

## Version and date
- Version: `2.8.2`
- Date: `2026-03-28`

## Milestone target and status
- Target milestone: `2.8.2 — Admin Settings Cleanup and Baked-In Feature Flag Retirement`
- Status: `Completed`

## Summary
This release finishes the admin control-plane cleanup that followed the broader `2.8.0` UI refinement pass and the `2.8.1` onboarding work. Permanently accepted behavior is no longer presented as fake optionality through feature flags, the drawer-first library flow is now just the normal path, and admin/debug surfaces now reflect real operator decisions instead of historical rollout scaffolding.

## What changed
### Feature-flag retirement
- Retired baked-in rollout flags from the active feature-flag service:
  - `import_csv_enabled`
  - `import_plex_enabled`
  - `metadata_normalized_read_enabled`
  - `tmdb_search_enabled`
  - `api_docs_enabled`
  - `ui_drawer_edit_experiment`
- Admin feature-flag screens now rely on the backend’s real operator-controlled flag set instead of hiding historical baked-in entries in the frontend.

### Runtime behavior cleanup
- CSV import, Plex import, TMDB search/details, and normalized metadata reads now behave as standard product behavior instead of admin-toggleable features.
- API docs are now controlled by authenticated admin access plus `DEBUG>=1` rather than a user-facing feature flag.
- The library drawer edit flow no longer has stale env/runtime fallback plumbing; the accepted drawer-first behavior is the only path.

### Docs, env, and diagnostics alignment
- Updated environment docs and `env.example` so they no longer advertise retired baked-in flags as active controls.
- Updated OpenAPI/docs guidance so `/api/docs` matches the new debug-only runtime gate.
- Updated metadata-cutover and logging/observability docs so historical rollout references are clearly marked as historical.
- Updated smoke/test fixtures so feature-flag audit examples use active operator-controlled flags instead of retired experiment flags.

## Breaking changes
- No API-breaking changes intended.
- Operators can no longer toggle CSV import, Plex import, TMDB search, normalized metadata reads, API docs, or the drawer-edit experiment through the admin feature-flag surface because these are no longer modeled as active feature flags.

## Environment/config changes
- Removed retired env examples for:
  - `REACT_APP_UI_DRAWER_EDIT_EXPERIMENT`
  - `FEATURE_FLAG_IMPORT_PLEX_ENABLED`
  - `FEATURE_FLAG_IMPORT_CSV_ENABLED`
  - `FEATURE_FLAG_TMDB_SEARCH_ENABLED`
  - `FEATURE_FLAG_UI_DRAWER_EDIT_EXPERIMENT`
- `/api/docs` now depends only on authenticated admin access and `DEBUG>=1`.
- Active env override examples remain for operator-controlled flags such as:
  - `FEATURE_FLAG_LOOKUP_UPC_ENABLED`
  - `FEATURE_FLAG_RECOGNIZE_COVER_ENABLED`
  - `FEATURE_FLAG_EVENTS_ENABLED`
  - `FEATURE_FLAG_COLLECTIBLES_ENABLED`
  - `FEATURE_FLAG_METRICS_ENABLED`
  - `FEATURE_FLAG_EXTERNAL_LOG_EXPORT_ENABLED`

## Migration and data impact
- No schema changes.
- No new migrations.
- Historical feature-flag rows may still exist in long-lived databases, but they are no longer part of the active runtime control model.
