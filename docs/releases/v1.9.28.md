## Deployment and verification
1. Build and start stack:
   - `docker compose --env-file .env up -d --build`
2. Confirm backend/frontend health.
3. Run backend unit tests:
   - `docker compose --env-file .env exec -T backend npm run test:unit`
4. Run backend API integration smoke:
   - `docker compose --env-file .env exec -T backend npm run test:integration-smoke`
5. Run migration rehearsal and capture evidence:
   - `docker compose --env-file .env exec -T backend sh -lc 'DATABASE_URL="postgresql://$DB_USER:$DB_PASSWORD@db:5432/$POSTGRES_DB" MIGRATION_REHEARSAL_ADMIN_URL="postgresql://$DB_USER:$DB_PASSWORD@db:5432/postgres" MIGRATION_REHEARSAL_OUTPUT=artifacts/migration-rehearsal-evidence.json npm run test:migration-rehearsal && cat artifacts/migration-rehearsal-evidence.json' > docs/reports/1.9.28-migration-rehearsal-evidence.json`
