# collectZ v3.18.4

## Version and date

- Version: `3.18.4`
- Date: 2026-06-12

## Milestone target and status

- Roadmap slice: `3.18.4 — Public Source Boundary Frontend Runtime Naming Cleanup`
- Status: completed

## Summary

This patch replaces frontend helper and state names that used private deployment language with neutral local-runtime wording. It keeps the existing backend contract value intact, but makes the frontend source less tied to private product labels.

## What changed

- Renamed frontend runtime helper names away from private deployment terminology.
- Renamed navigation and dashboard state variables to use `localRuntime` wording.
- Preserved the existing backend `product_edition` contract behavior for compatibility.
- Reduced the public-source boundary audit from `27` findings across `5` files to `12` findings across `3` files.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- No schema, migration, or collection data impact.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.18.4`.
- Verify `/api/health` reports version/frontend/backend/build `3.18.4`.
- Verify Help > Releases serves `3.18.4`.
- Run `npm run audit:public-source-boundary` and confirm frontend helper naming findings remain cleared.
- Run `npm --prefix frontend run build`.
- Run `npm run release:local-gate`.

## Rollback guidance

- Revert to `3.18.3` if the frontend helper rename needs to be backed out.
- No schema rollback is required.

## Known issues and follow-up

- Public source publication remains blocked by backend OpenAPI `product_edition`, `edition_contract`, and legacy local-runtime value findings, plus one frontend compatibility constant.
- The next source-boundary cleanup should address the public API/runtime contract naming rather than only frontend helper names.
