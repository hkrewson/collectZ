# collectZ v3.12.17

## Version and date

- Version: `3.12.17`
- Date: 2026-06-05

## Milestone target and status

- Roadmap slice: `3.12.17 — Frontend Node 24 Builder Alignment`
- Status: completed

## Summary

This patch aligns the frontend Docker build image with the current Vite and ZXing dependency contract by moving the frontend builder from Node 20 to Node 24.

## Changes

- Updated the frontend Docker build stage to use `node:24-alpine`.
- Kept the backend Docker runtime on Node 20 for this patch.
- Kept `@zxing/browser@0.2.0` and `@zxing/library@0.22.0` in their currently compatible peer range.
- Added a source-contract assertion that the frontend Docker builder remains on Node 24.
- Documented that the remaining `@zxing/library@0.23.0` dependency PR is still a separate peer-compatibility decision.

## Breaking changes

- None intended.

## Upgrade notes

- No runtime configuration changes are required.
- Operators using the published frontend image do not need to install Node locally.
- Maintainers building the frontend image locally should let Docker use the checked-in Dockerfile so the Node 24 builder is used.

## Migration and data impact

- No schema migration is required.
- Existing collection data, sessions, provider settings, uploads, and runtime configuration are unchanged.
