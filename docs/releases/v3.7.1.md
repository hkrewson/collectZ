# collectZ v3.7.1

## Version and date

- Version: `3.7.1`
- Date: `2026-05-17`

## Milestone target and status

- Milestone: `3.7.1 — Timeline Deep Links`
- Status: `implemented`

## Summary

This patch keeps iterating on the human-readable activity timeline by adding contextual navigation from timeline rows to the related app surface.

## What changed

- Added small timeline action buttons for known targets such as media items, Art, Collectibles, Events, imports, loans, integrations, people, workspaces, and users.
- Wired Admin Activity links into the main dashboard shell so platform timeline entries can open the relevant admin or library surface.
- Wired Workspace Activity links so workspace entries can open the matching workspace tab or jump out to the relevant library item.
- Added focus handoff for media, Art, Collectible, and Event targets so existing rows can open their detail drawer after navigation.
- Extended the workspace activity browser regression to prove a media timeline row can open the related library detail.

## Breaking changes

- None.

## Environment/config changes

- No new environment variables.

## Migration and data impact

- No database migration.
- Existing `activity_log` rows are reused. Links are shown only when the existing entity/action context is specific enough.

## Security triage

- Timeline navigation stays inside the existing authenticated app shell and uses the current scoped Activity endpoints.
- No new public routes or broader activity read permissions were added.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.7.1` closeout if any high findings appear.
