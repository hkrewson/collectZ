# collectZ v3.4.113

Released: 2026-05-07

## Version

- Version: `v3.4.113`
- Backend package: `3.4.113`
- Frontend package: `3.4.113`

## Milestone

- Roadmap slice: `3.4.113 — Plex Real-Server Provider Discovery Readback`
- Status: closed after local verification

## Summary

This release adds a read-only Plex provider discovery probe for saved Plex integrations. Admin and workspace integration screens can now ask the configured Plex Media Server for `/media/providers` and show sanitized capability readback without exposing tokens or changing Plex import behavior.

## What Changed

- Added admin and workspace Plex provider discovery probe endpoints.
- Reused saved Plex API URL and token settings server-side.
- Returned only sanitized provider fields: key, title, type, protocol, identifier, and feature keys.
- Added a Plex Integrations action to probe providers and display the sanitized readback.
- Preserved existing Plex section test and import behavior.

## Compatibility

- No database migration.
- Existing Plex import paths remain unchanged.
- Existing library-section discovery remains available through the normal Plex test action.

## Deployment Notes

- Build and deploy backend/frontend images with `APP_VERSION=3.4.113`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.113`.
- Verify Help > Releases serves `v3.4.113` as the latest release.

## Rollback

Roll back to `v3.4.112` if the real-server provider discovery probe needs to be removed. Plex import behavior is unchanged, so rollback risk is limited to the new read-only endpoints and UI action.

## Known Issues and Follow-Up

- This does not add the Now Playing Viewer.
- This does not add Plex webhooks or scheduled sync.
- This does not replace legacy Plex import paths.
- Real PMS provider contents may vary by Plex server version and enabled server capabilities.
