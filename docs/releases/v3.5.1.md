# collectZ v3.5.1

## Version and date

- Version: `3.5.1`
- Date: `2026-05-17`

## Milestone target and status

- Milestone: `3.5.1 — Dashboard Attention Readback Refinement`
- Status: `released`

## Summary

This patch tightens the new Dashboard command center so attention items show useful readback instead of sending users to broad navigation destinations.

## What changed

- Removed the superfluous Dashboard `Quick actions` panel.
- Removed the `Import` shortcut from `Recent syncs`.
- Removed the duplicate `Latest failures` panel so failed syncs have one source of truth in `Needs attention`.
- Changed failed sync attention from a generic Integrations jump into inline failed-job details with provider, job type, timestamp, status, and error readback.
- Added inline lists for items missing covers and items missing identifiers.
- Made the summary counters and attention tab actions open filtered all-media Library review lists for missing covers and missing identifiers.
- Added `review_filter` support to `GET /api/media` so Dashboard drilldowns and Library use the same backend source of truth for attention lists.
- Made Dashboard more compact with tighter panel spacing, smaller summary counters, and capped attention samples.
- Removed the aggregate `Needs attention` summary metric because it duplicated the tabbed attention panel without adding a distinct workflow.
- Reflowed Dashboard into a denser dashboard layout with three collection-health metrics, a primary attention/provider row, and compact Recent syncs, Recent activity, and Upcoming events panels.
- Renamed the command-center surface to `Dashboard` across the nav, heading, tab id, API route, OpenAPI contract, tests, and docs.
- Changed `Needs attention` into tabs so failed syncs, missing covers, missing identifiers, and Plex conflicts do not stack into a long page on high-resolution screens.
- Extended `GET /api/dashboard/summary` with `attention_details` samples for missing-cover and missing-identifier items.
- Updated OpenAPI and source assertions for the richer Dashboard summary contract.

## Breaking changes

- None.

## Environment/config changes

- No new environment variables.
- No new provider credentials are required.

## Migration and data impact

- No database migration.
- Existing media and sync-job rows are read to build Dashboard attention lists.

- Dashboard summary remains authenticated and active-scope enforced.
- The new item samples are limited to the authenticated user's active workspace/library scope.
- Provider secrets remain masked and server-side.
