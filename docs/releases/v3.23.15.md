## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `useImportJobPolling.js` was the top remaining frontend lint error pocket after `3.23.14`. The slice clears import polling hook purity, empty-catch, and documented leader-sync findings without changing import polling behavior.

## What changed

- Generated the per-tab import polling id through lazy state initialization instead of impure render-time ref initialization.
- Replaced empty local-storage catch blocks with explicit comments for tolerated malformed or unavailable storage.
- Documented the initial import polling leader claim as intentional synchronization with browser focus/storage state.
- Preserved import job persistence, localStorage keys, leader election, heartbeat/stale windows, polling cadence, and job upsert/dismiss/clear behavior.
- Reduced the full frontend quality baseline from `25` errors / `24` warnings to `19` errors / `24` warnings.
