- Status: completed

## Summary

This patch fixes Plex audio imports so music library album rows stay album-shaped in collectZ. Plex artist rows are no longer imported as audio titles, album metadata is persisted, and audio detail drawers now show album, artist, and track count when that data is available. It also fixes runtime config script ordering so local and hosted frontends read container-provided runtime settings before the app starts.

## What changed

- Excluded Plex artist and track rows from audio-section media imports.
- Persisted Plex audio `type_details` during both import create and update paths.
- Added track count normalization for Plex album imports.
- Added an Album details section for audio records in the Library detail drawer.
- Loaded `runtime-env.js` before the Vite app entrypoint so runtime API and platform bridge settings are available during startup.
- Added backend unit/source coverage for audio import filtering, persisted album metadata, and Library rendering.

## Breaking changes

- None.

## Environment/config changes

## Migration and data impact

- No schema migration is required.
- Existing incorrectly imported Plex artist rows are not automatically repaired by this patch.
- Reimporting the Plex music library can update matched album rows with album, artist, and track-count metadata.
