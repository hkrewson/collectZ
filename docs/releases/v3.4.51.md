# v3.4.51

## Event schedule conflict resolution actions

- Version: `3.4.51`
- Target milestone: `3.4.51 — Event Schedule Conflict Resolution Actions`
- Date: 2026-05-01

### What changed

- Added a compact conflict resolver for catalog session plan-state changes.
- When planned, maybe, or backup would overlap another active plan, the drawer now asks how to save the choice.
- Added local actions to keep both, make the selected session planned while moving conflicts to backup, mark the selected session as backup, or skip it.
- Resolver prompts stay near the control that triggered them, so Now / Next and Catalog rows do not duplicate the same prompt.
- Non-conflicting quick plan-state changes continue to save directly.

### Why it matters

Conflict warnings are helpful, but the next step is intent. This patch lets users decide whether an overlapping session is primary, backup, tentative, or skipped without opening every schedule row or introducing social notifications yet.

### Security and privacy notes

- This patch does not add notifications, friend/group attendance, provider import automation, realtime location, presence, native UI, or offline mutation queues.
- Conflict choices are applied only to the current user's event schedule plans through existing scoped Event endpoints.
- Raw personal Sched ICS feed URLs remain hidden and are not used as a full event catalog source.
