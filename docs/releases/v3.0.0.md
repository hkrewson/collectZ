## Summary
This major release completes the frontend build modernization from Create React App to Vite. The app now builds and runs through a single Vite-based toolchain for local development and production static assets, the nginx-served frontend image uses Vite output directly, the legacy CRA fallback rail is removed, and the repo has been re-proven against the live Docker stack and browser-regression suite on the final Vite-only state.

## What changed
### Vite-first frontend toolchain
- Replaced the legacy `react-scripts` frontend workflow with Vite for:
  - local development
  - production builds
  - preview builds
- Added and then completed the Vite migration path with:
  - Vite config and root entry files
  - React plugin wiring
  - proxy support for `/api` and `/uploads`
  - explicit frontend build commands that no longer depend on hidden CRA defaults

### Production Docker/nginx cutover
- The frontend Docker image now builds Vite `dist` output directly.
- nginx continues to serve the built static frontend with SPA fallback behavior intact.
- Build/runtime metadata remains available through the existing app version/footer and health endpoints.

### Local maintainer workflow modernization
- Frontend maintainer defaults now use Vite:
  - `npm --prefix frontend run start`
  - `npm --prefix frontend run build`
  - `npm --prefix frontend run preview`
- The temporary CRA rollback rail was retired after Docker, browser-regression, and runtime proof stayed green on the Vite path.

### Source-tree cleanup for the new toolchain
- Active JSX-bearing frontend files were normalized to `.jsx` so the repo no longer depends on the temporary Vite JSX-in-`.js` bridge.
- The stale CRA entrypoint fallback file was removed once the Vite-only path was proven.
- Frontend docs, repo assertions, and workflow assumptions were updated to reflect the Vite-only build contract.

### CI and stabilization work folded into the migration
- Reduced the app shell below the CI modularity budget by extracting supporting app-shell concerns into dedicated modules.
- Repaired browser and smoke harness drift that surfaced under the newer auth model and during the toolchain migration.
- Fixed a backend media-route shadowing bug uncovered during manual smoke testing so:
  - import progress polling works again
  - movie and game collection requests no longer return `400`

## Breaking changes
- `3.0.0` changes the frontend build/dev system from Create React App to Vite.
- The legacy `react-scripts` commands and fallback rail are removed.
- Any maintainer or CI workflow that still assumed CRA output or CRA commands must use the Vite-based equivalents.

## Environment/config changes
- The frontend now runs through a Vite-first toolchain.
- Existing `REACT_APP_*` compatibility remains in place for now so the runtime contract stays stable during the completed migration.
- Matching Vite-facing build args remain supported in the frontend Docker build:
  - `VITE_API_URL`
  - `VITE_APP_VERSION`
  - `VITE_DEBUG`

## Migration and data impact
- `3.0.0` is a build-tool and workflow migration only.
- No schema/data migration is introduced specifically for the CRA-to-Vite cutover.
- Existing backend API topology, auth/session behavior, media CRUD/import flows, tenancy surfaces, and nginx-hosted production topology remain in place.

   - workspace
   - help
5. Verify the Vite-served frontend image passes the browser-regression suite.
6. Verify the release gates:
   - backend unit tests
   - init parity
   - migration rehearsal

   - browser regression
   - RBAC regression
   - homelab edition boundary

## Rollback guidance
- Roll back application images to `2.11.0` if the Vite migration needs to be reverted.
- Because `3.0.0` removes the CRA fallback rail, rollback should be treated as an image/runtime rollback to the last CRA-era release rather than an in-place toggle.

## Known issues and follow-up
- The Vite production build still emits a large-chunk warning, so bundle/code-splitting cleanup is a sensible follow-up after the major-version cutover.
- `REACT_APP_*` compatibility remains intentionally in place for now even though CRA is gone; further env cleanup can happen in a later slice without blocking this milestone.
- A comics-only double-refresh UI quirk was observed during manual smoke testing and should be folded into a later UI-focused milestone rather than blocking the Vite migration closeout.
- Full release publication still depends on the broader CI-only gates, including secret scan, SBOM/image security, compose smoke, and the edition-boundary checks.
