# collectZ v3.8.7

## Version and date

- Version: `3.8.7`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.7 — Capture Identifier Match Lookup`
- Status: `implemented`

## Summary

This release lets Capture Inbox turn an applied barcode or ISBN into reviewable catalog and provider match candidates.

## What changed

- Adds authenticated `POST /api/capture-items/{id}/lookup-matches`.
- Reuses the backend scanner barcode lookup path used by `/api/media/lookup/barcode`.
- Stores lookup matches and summary counts on the capture row for later review.
- Adds a compact `Find matches` action for capture rows that have a barcode or applied OCR candidate.
- Shows persisted capture matches inline, including existing catalog matches marked as already in the library.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.
- Provider lookups continue to use the existing Barcode and Books integration settings.

## Migration and data impact

- No new database migration.
- Capture match results are stored in `capture_items.review_decision` and `capture_items.source_context`.

## Security triage

- Capture lookup remains authenticated and scoped to the current workspace/library boundary.
- This slice stores provider/catalog candidates for user review only; it does not import, link, or update media automatically.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.7` closeout if any high findings appear.

- Provider quality and availability depend on the configured Barcode and Books integrations.
