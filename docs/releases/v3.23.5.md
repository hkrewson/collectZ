## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `EventsView.jsx` was the top remaining frontend lint pocket after `3.23.4`. The slice extracts small Event UI pieces, cleans the touched Event row/drawer surface, and leaves Comic-Con Hunt behavior frozen.

## What changed

- Extracted Event card, Event list-row, and schedule notification recipient option UI from `EventsView.jsx`.
- Made Event list rows keyboard reachable while preserving the existing open/edit/delete behavior.
- Replaced the Event editor backdrop with a native button.
- Documented remaining Event API/focus synchronization effects instead of rewriting Event data flow.
- Reduced the full frontend quality baseline from `115` errors / `47` warnings to `99` errors / `44` warnings.
