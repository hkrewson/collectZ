## Summary

This patch upgrades the backend Express runtime to Express 5 and removes the old Express 4-era `path-to-regexp` override so route registration, middleware, and API error handling run on the supported Express 5 router stack.

## What changed

- Updated backend `express` from `^4.22.2` to `^5.2.1`.
- Removed the backend `path-to-regexp` override that forced the legacy `0.1.x` matcher.
- Verified the Express 5 router uses its compatible `path-to-regexp` dependency.
- Confirmed existing route registration, OpenAPI validation, and unit coverage still pass.

## Breaking changes

None expected for users.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. This patch changes backend runtime dependencies only.

## Rollback guidance

Rollback to `3.10.26` is safe because this patch has no schema, API contract, or data-format changes. If runtime route behavior regresses, restore Express 4 and the previous backend lockfile, then rebuild the backend image.

## Known issues and follow-up

- Express 5 has stricter route path matching than Express 4. No incompatible route strings were found in this slice, but full CI and normal API use should watch for missed edge routes.
- `@zxing/library@0.23.0` remains deferred until its browser peer relationship is intentionally resolved.
