# collectZ v3.8.11

## Version and date

- Version: `3.8.11`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.11 — Capture Scan Immediate Lookup`
- Status: `implemented`

## Summary

This release shortens the Capture Inbox barcode workflow by showing lookup candidates immediately in the new-capture editor after a barcode or ISBN is scanned or entered.

## What changed

- Capture Inbox can run the scanner barcode lookup contract before the capture row is saved.
- New-capture lookup results appear inline with candidate title, source, type, and library state.
- A selected result can be added or linked through the existing Capture Inbox import path.
- Captures saved for review now keep lookup matches and lookup status on the capture row.
- Added a `Save and scan next` action so scanning another item does not require returning to the inbox list.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- New unsaved lookup metadata is stored only when the user saves the capture or imports a selected match.

## Security triage

- Lookup remains authenticated and scoped through the existing `/api/media/lookup/barcode` contract.
- Import still persists through authenticated Capture Inbox import endpoints.
- No scanner secrets or provider keys are exposed to the browser.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.11` closeout if any high findings appear.
