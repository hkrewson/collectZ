- Status: closed after local verification

## Summary

This release adds a standalone Plex Now Playing page. Authenticated admins can open `/now-playing` for a passive display view backed by the saved Plex now-playing readback and an app-owned image proxy.

## What Changed

- Added the `/now-playing` frontend route outside the dashboard shell.
- Added a read-only backend viewer endpoint for sanitized active-session display data.
- Added an authenticated Plex image proxy for Plex-relative poster/art keys.
- Shows current title, show/season context, player state/platform, progress, and poster/art when Plex exposes usable image keys.
- Added empty and unavailable states for no active sessions or unreachable Plex settings.

## Compatibility

- No database migration.
- Existing Plex imports, provider discovery, now-playing proofs, readback endpoints, and Integrations UI behavior remain unchanged.
- The viewer uses current playing or paused sessions only; next-queue behavior remains out of scope.

## Deployment Notes

- Build and deploy backend/frontend images with `APP_VERSION=3.4.118`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.118`.
- Verify Help > Releases serves `v3.4.118` as the latest release.
- Open `/now-playing` while signed in as an admin to use the display page.

## Rollback

Roll back to `v3.4.117` if the standalone viewer or image proxy needs to be removed. The previous Integrations readback and real PMS proof remain available in `v3.4.117`.

## Known Issues and Follow-Up

- Queue/next-up behavior is not implemented because the real PMS proof did not expose queue hints.
- Viewer access is admin-authenticated in this slice.
- Future work can add a dedicated display token, workspace-scoped viewer ownership, webhooks, scheduled refresh cadence controls, and watch-state behavior.
