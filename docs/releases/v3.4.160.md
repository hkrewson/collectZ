# collectZ v3.4.160

## Version and date

- Version: `3.4.160`
- Date: `2026-05-16`

## Milestone target and status

- Milestone: `3.4.160 — Scanner ISBN Direct Books Lookup`
- Status: `released`

## Summary

This patch makes ISBN-shaped barcode scans use the configured Books provider directly, so the iOS scanner app can get Google Books-style candidates without routing through the web UI or forcing the client to know the media type first.

## What changed

- Scanner barcode lookup now detects ISBN-10/ISBN-13 shaped codes automatically.
- Web UPC lookup now also uses the direct ISBN path when a scanned identifier normalizes to ISBN.
- Direct ISBN matches return with `source: books:isbn-direct` and `mediaTypeGuess: book`.
- Scanner smoke now proves the ISBN path works without `mediaType: book`.
- OpenAPI now documents scanner ISBN behavior on `/api/media/lookup/barcode`.

## Breaking changes

- None.

## Environment/config changes

- None. This uses the existing Books/Google Books integration settings.

## Migration and data impact

- No database migration.
- Existing imports continue to use the selected lookup candidate.

## Security triage

- Lookup auth behavior is unchanged.
- Scanner lookup remains authenticated with `media:read`; import remains authenticated with `import:run`.
- The new smoke fixture uses local fake Books provider data and does not write provider secrets to evidence.

## Security vulnerability triage

- High findings: none in local dependency audits.
- Owner: collectZ maintainers.
- Target remediation milestone: not needed for this release because no high findings were present locally.
