## Version and date

## Milestone target and status

- Roadmap slice: `3.18.2 — Public Source Boundary Audit`
- Status: completed

## Summary

This release adds the first explicit public-source boundary audit. The public mirror remains an install/support mirror for now, but maintainers can now measure exactly which frontend and OpenAPI source surfaces still block a useful source mirror.

## What changed

- Added `npm run audit:public-source-boundary` to scan candidate source-publication surfaces.
- Added a JSON audit report under ignored local artifacts so blockers can be reviewed without committing evidence files.
- Tracked blocker categories for private runtime language, runtime contract names, operations wording, auth/test bypass labels, and private docs references.
- Updated public mirror docs and sanitizer strategy to make clear that application source remains private until this boundary is intentionally cleaned.
- Tightened the public export contract so source folders stay denied until the audit findings are resolved.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.18.2`.
- Verify `/api/health` reports version/frontend/backend/build `3.18.2`.
- Verify Help > Releases serves `3.18.2`.
- Run `npm run audit:public-source-boundary`.
- Run `npm run validate:public-export`.
- Run `npm run public:export -- --force --commit` before any public mirror publication review.

## Rollback guidance

- Revert to `3.18.1` if the source-boundary audit needs to be backed out.
- No schema rollback is required.

## Known issues and follow-up

- The audit currently reports source-publication blockers in frontend runtime naming and the OpenAPI contract. Those must be resolved before frontend source or API contract files are added back to the public export.
- The public mirror still does not push automatically.
