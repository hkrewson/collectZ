## Summary

This patch adds a non-destructive restore rehearsal readback so admins can tell whether they have the pieces needed to practice a restore in a separate test stack.

## What changed

- Added `restore_rehearsal` to `GET /api/admin/settings/portability`.
- Admin Settings now shows a Restore rehearsal card with database export, backup freshness, uploaded image binary, and restore dry-run checks.
- The restore rehearsal response explicitly reports `destructive: false`.
- Restore guidance now calls out rehearsal in a separate test stack.
- The backup runbook now includes a restore rehearsal checklist.
