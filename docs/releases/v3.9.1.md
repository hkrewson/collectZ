# collectZ v3.9.1

## Version and date

- Version: `3.9.1`
- Date: 2026-05-20
- Milestone: `3.9.1 — Apple/iTunes Wishlist Price Readback`

## Milestone target and status

Status: Complete.

This patch extends the Apple/iTunes Wishlist intake with a manual current-price refresh for saved Apple-linked wishlist rows.

## Changes

- Added authenticated `POST /api/wishlist/apple-itunes/refresh-prices`.
- Added Apple/iTunes lookup support for saved provider keys.
- Stores refreshed current price, currency, store/artwork URLs, refresh timestamp, and target-price readback in wishlist `source_context`.
- Added a compact `Refresh saved prices` action on the Wishlist Apple/iTunes panel.
- Renamed the optional per-result price field from “Buy below” to “Target price”.
- Shows Apple current-price readback on saved Apple-linked wishlist rows when available.
- Documented the price refresh route in OpenAPI.

## Security and privacy

- Apple/iTunes price refresh stays authenticated and scoped to the active workspace/library context.
- No Apple API key is required.
- Raw Apple lookup metadata remains stored only inside authenticated/scoped wishlist rows.
