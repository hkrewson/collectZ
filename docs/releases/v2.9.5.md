## Summary
This release expands the Playwright browser-regression layer from foundational coverage into broader product-critical workflows. It adds realistic browser protection for integrations persistence, barcode and CSV import paths, Help/Admin boundary behavior, and the Events/Collectibles surfaces, while keeping the suite deterministic through targeted helpers, cleanup paths, and job polling. The result is stronger browser confidence ahead of later frontend build-system work without bloating the suite into fragile snapshot noise.

## What Changed
### Critical browser flow expansion
- Added Playwright coverage for Integrations save/update flows, including:
  - Barcode settings save and persist across reload
  - Games settings save and persist across reload
  - Metrics Export toggle persistence
  - integration deep-link handling for supported `?integration=` sections
- Added Playwright coverage for Import Media workflows:
  - barcode lookup with deterministic provider stubbing
  - generic CSV upload through the real browser UI
  - queued CSV import completion polling before library verification
  - live library verification after import succeeds
- Added Playwright coverage for Events and Collectibles:
  - event creation
  - collectible creation linked to a created event
  - linked-event verification through the collectible detail surface
- Added browser-visible negative-path coverage for shell and admin/privacy boundaries:
  - support-admin redirect-back-to-help behavior on disallowed routes
  - standard-user denial on direct admin routes

### Browser harness ergonomics
- Added reusable helpers for:
  - integration state snapshot/restore
  - import cleanup
  - event/collectible cleanup
  - sync-job completion polling
- Kept test-created browser data cleaned up after runs so the suite remains reusable and deterministic.
- Expanded roadmap/testing docs so the browser layer’s protected surfaces stay visible to maintainers.

### Browser-driven fixes and refinements
- Fixed the admin integrations save query so the insert/update column list matches the values passed during real browser saves.
- Fixed dashboard integration-section deep-link routing so supported sections like `metrics` and `logs` resolve correctly.
- Hardened support/admin shell expectations with explicit browser coverage around redirect and surface-limit behavior.
- Adjusted Events/Collectibles browser coverage to follow the real shell navigation and user-visible detail contracts instead of brittle direct-route or decorative overlay assumptions.

## Breaking Changes
- No intentional end-user breaking changes ship in `2.9.5`.
- Browser-regression expectations now cover more product flows, so CI can surface UI regressions that previously would have slipped through backend-only validation.
