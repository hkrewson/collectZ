## Summary
This patch release hardens post-merge import durability. It adds runtime proof that canonical records survive repeat imports, cross-source imports, chained merge history, Metron re-syncs, Plex re-syncs, and merge reverts without silently recreating duplicates. It also fixes a real revert bug so absorbed identity aliases are removed from the former canonical when a merge is undone, allowing later imports to update the restored duplicate row cleanly.

## What Changed
### Re-sync durability proofs
- Added Docker-backed Metron re-sync alias proof for merged comic rows.
- Added Docker-backed repeat-import idempotency proof for generic CSV, Calibre CSV, and Delicious CSV import families.
- Added Docker-backed cross-source canonical reuse proof across generic CSV, Calibre CSV, and Delicious CSV.
- Added Docker-backed multi-hop merge alias proof so historical provider identities from chained merges still resolve to the final canonical row.
- Added Docker-backed merge revert re-sync integrity proof for restored duplicate rows.

### Merge identity hardening
- Preserved canonical reuse behavior across provider-item, Plex, and Metron re-sync paths with explicit smoke coverage.
- Fixed revert cleanup so duplicate-only identity alias metadata no longer remains attached to the former canonical after a merge is reverted.
