# v3.4.31

## Release Metadata
- Version: `3.4.31`
- Date: `2026-04-28`
- Target milestone: `3.4.31 — Personal Sched ICS Sync Contract and Parser Spike`

## Summary
This release adds the first personal Sched ICS sync path for Events. A user can connect an encrypted personal iCal subscription link, manually refresh it, and sync selected sessions into private event schedule plans.

## Changes
- Added encrypted per-user/per-event personal ICS source storage.
- Added redacted source status, save/remove, and manual sync endpoints.
- Added a small Sched-style VEVENT parser and sync service.
- Synced ICS entries now create or update private `event_schedule_plans` with `source_type = sched_ics`.
- Added Event drawer controls for connecting, replacing, syncing, and removing a personal Sched ICS feed.
- Added smoke/unit/OpenAPI coverage for the personal ICS contract.

## Privacy Notes
- Raw personal ICS URLs are encrypted at rest and are never returned by API readback.
- Sync responses expose status, counts, and sanitized errors only.
- This is selected-session personal sync, not the full public event schedule catalog.

## Validation
Production/runtime release gates for `3.4.31`:
- Docker-first backend/frontend build and runtime health checks should report `3.4.31`.
- Unit, OpenAPI, init parity, migration rehearsal, Help > Releases, homelab boundary, and personal ICS sync smoke should pass locally before release.

## Rollback
Roll back application images to `3.4.30` if personal ICS source storage or sync causes regressions. Synced schedule plans use `source_type = sched_ics`, so they can be identified and archived separately if needed.
