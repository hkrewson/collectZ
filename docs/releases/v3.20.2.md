# collectZ v3.20.2

## Version and date

- Version: `3.20.2`
- Date: 2026-06-20

## Milestone target and status

- Roadmap slice: `3.20.2 - Plex Webhook Receiver Setup Validation`
- Status: completed

## Summary

This patch adds Plex webhook receiver setup validation. The Plex Webhook tab can check whether a receiver exists, record the latest validation result, and warn when the advertised receiver URL appears local-only.

## What changed

- Added migration `112` for Plex webhook receiver validation readback.
- Added `plexWebhookReceiver.validation` to integration settings payloads.
- Added an admin validation endpoint for Plex webhook receiver setup.
- Added a compact `Check setup` action to Settings > Integrations > Plex > Webhook.
- Kept receiver URLs masked and token details out of browser-visible payloads.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- Adds nullable validation-status, validation-message, and validated-at columns to `app_integrations`.
- Existing installs start with no validation readback until an admin runs the check.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.20.2`.
- Verified locally:
  - `node --check backend/routes/integrations.js`
  - `node --check backend/services/integrations.js`
  - `node --check backend/db/migrations.js`
  - `node --check backend/scripts/unit-tests.js`
  - `npm --prefix frontend run build`
  - `node backend/scripts/unit-tests.js`

## Rollback guidance

- Roll back to the previous backend and frontend images if Plex webhook settings regress.
- No destructive migration rollback is required; the added columns can remain unused.

## Known issues and follow-up

- This validation confirms collectZ has generated a receiver and checks whether the advertised host appears public; it does not make an external callback from Plex.
- Initial import, scheduled pull sync, explicit writeback controls, and activity entries remain follow-up work in the Plex true-sync plan.
