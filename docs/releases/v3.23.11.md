## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `SpaceManagerView.jsx` was tied for the top remaining frontend lint pocket after `3.23.10`, and it was the more contained next slice. The slice clears Space Manager import, hook-baseline, purity, and accessibility findings without changing workspace management behavior.

## What changed

- Removed the stale default React import from the Space Manager view.
- Documented intentional Space Manager state synchronization effects for selected workspace form state, active-scope resets, member loading, and invite loading.
- Kept invite expiry filtering and rendering pure by using stable invite-status time state instead of render-time clock reads.
- Removed an unnecessary non-interactive member-menu click handler while preserving member action buttons.
- Preserved direct `/dashboard?tab=space-manage` navigation while backend-owned scope data finishes loading.
- Reduced the full frontend quality baseline from `52` errors / `30` warnings to `44` errors / `29` warnings.
