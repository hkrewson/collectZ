- Status: `completed`

## Summary

This patch clarifies page-title ownership across the library surfaces: desktop library pages now name the section being viewed, while mobile keeps location in the compact app header instead of repeating it in the page toolbar.

## What changed

- Changed desktop media library headings from generic `Library` to the active section name, such as `Movies`, `TV`, `Books`, `Audio`, `Games`, or `Comics`.
- Kept the aggregate all-library review route labeled `Library`.
- Hid duplicate page-title/count groups from mobile library, Art, Collectibles, and Events toolbars because the mobile app shell already shows the active page title.
- Preserved mobile search, filter, view-toggle, sort, and add controls in their existing page toolbars.
- Updated browser regression coverage for mobile shell title ownership and the new desktop media-library heading.
