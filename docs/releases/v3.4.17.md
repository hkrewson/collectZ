## Summary
This release adds explicit unit metadata to native Art dimensions. Art height and width can now carry inches or centimeters without changing the existing numeric dimension fields or converting existing records.

## What changed
### Art dimension metadata
- Added native Art `dimension_unit` storage for the existing H/W values.
- Supported `in` and `cm` as the initial controlled unit values.
- Art create, update, list, and detail responses now include `dimension_unit`.
- Existing height and width values remain untouched unless a user edits and saves an Art record.

### Drawer experience
- Added a compact `Unit` selector beside the Art drawer H/W fields.
- New Art drawer entries default the visible selector to inches.
- Saves with no H/W values store a null unit so empty dimension metadata remains empty.
- Art detail readback displays dimension values with their unit when available.

### Regression coverage
- Added source-level coverage for the Art schema, validation, routes, OpenAPI contract, and drawer wiring.
- Expanded the focused Art/Event/Collectibles browser regression to round-trip centimeter and inch unit values.
- Re-ran init parity and migration rehearsal through migration `81`.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing Art records continue to read back with null unit metadata until edited or explicitly updated.

## Environment/config changes
- No new required operator configuration in `3.4.17`.

## Migration and data impact
- Adds nullable `art_items.dimension_unit` with a constraint allowing `in`, `cm`, or null.
- Existing Art rows are not destructively rewritten.
- This release does not add unit conversion or separate height-unit and width-unit fields.
