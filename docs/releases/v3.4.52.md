# v3.4.52

## Event schedule shared attendance readback

- Version: `3.4.52`
- Target milestone: `3.4.52 — Event Schedule Friend / Group Attendance Readback`
- Date: 2026-05-01

### What changed

- Added visibility-aware shared attendance readback for catalog-linked schedule plans.
- Now / Next and Catalog rows can show compact shared counts such as `Shared: 1 backup`.
- Catalog expanded rows can show a shared attendance breakdown by current visibility category.
- The user's private linked plan remains the primary quick plan-state control when shared linked plans exist.
- Existing conflict detection and conflict-resolution behavior remains unchanged.

### Why it matters

This is the first web-side step toward friend and group scheduling context. It gives the drawer useful shared-plan readback without pretending the web app already owns platform identity, push notifications, or mobile offline social sync.

### Security and privacy notes

- This patch does not add push notifications, friend discovery, event companion invites, platform relay identity, device registration, realtime location, presence, or offline mutation queues.
- Shared attendance is derived from existing event schedule plans and visibility labels already scoped to the current event.
- Raw personal Sched ICS feed URLs remain hidden and are not used as a full event catalog source.
