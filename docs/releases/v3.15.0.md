## Summary

This minor release turns the backup and portability foundation into a real admin export action.

## What changed

- Added an admin-only one-click export download in Admin Settings.
- Added `POST /api/admin/settings/portability/export`.
- Export downloads are gzip-compressed collectZ JSON bundles.
- Export bundles include a manifest, app version, generation timestamp, portable database tables, upload-storage manifest, restore guidance, and redaction metadata.
- Secret-looking fields and secret-looking URL parameters are recursively redacted before the export is written.
- The existing portability readback now reports manual export capability.
- Admin Settings shows the last successful export download for the current browser session.

## Breaking changes

- None.

## Environment/config

- None.

## Migration/data impact

- No schema migration.
- Export generation reads existing database rows and upload storage metadata.
- Uploaded image binaries are not bundled in this first export slice; the export includes an uploads manifest and continues to instruct admins to back up the upload volume or object storage bucket separately.
