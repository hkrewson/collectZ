# collectZ v3.4.152

## Version and date

- Version: `3.4.152`

## Milestone target and status

- Milestone: `3.4.152 — Reusable Artist Records for Artwork Entry`
- Status: `released`

## Summary

This patch adds reusable artist records to Art entry. Artwork can still keep its own simple artist text, but users can now create an artist once, reuse that record across multiple works, and see linked artist details from the artwork detail view.

## What changed

- Adds reusable Art artist records with names, sort names, aliases, website links, notes, and linked-work counts.
- Adds `artist_id` and `artist_role` to Art rows so each artwork can link to an artist while keeping item-specific text and role metadata.
- Adds typeahead artist lookup and inline artist creation inside the Art drawer.
- Shows linked artist metadata on Art detail with a shortcut to view other works by the same artist.
- Documents and tests the new Art artist API in OpenAPI, migration/init parity, source checks, and browser regression coverage.

## Breaking changes

- None. Existing Art rows without linked artist records continue to use their current item-local artist text.

## Environment/config changes

- None.

## Migration and data impact

- Adds `art_artist_records` and indexes for library-scoped artist lookup.
- Adds nullable `artist_id` and `artist_role` columns to `art_items`.
- No existing Art rows are automatically linked or rewritten by this migration.

## Rollback guidance

- Revert the `3.4.152` code and docs changes if the artist-record workflow needs to be removed.
- The new columns are nullable and the new table is additive; existing Art item-local artist text remains available after rollback.

## Known issues and follow-up

- Existing Art rows are not automatically deduplicated into artist records.
- Keyboard-highlight acceptance is intentionally conservative in this slice; typeahead selection and inline creation are available without implicit irreversible autofill.
- Event exhibitor lookup/import, external creator identity, and artwork edition registry enrichment remain separate future work.
- CI-only secure-cookie compose smoke, secret scan, and image security/SBOM gates still run in GitHub Actions.
