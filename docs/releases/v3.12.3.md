## Version and date

## Milestone target and status

- Status: `completed`

## Summary

This patch tunes Dashboard Review's Missing IDs rules so barcode-backed books, movies, and TV records are no longer treated as needing manual provider identifiers.

## What changed

- Treats UPC/barcode as a useful identity for books when deciding whether a row belongs in `Review > IDs`.
- Treats UPC/barcode as a useful identity for movies and TV rows when TMDB or Plex identity is absent.
- Keeps physical audio and physical game UPC expectations intact.
- Keeps comic issue identity behavior unchanged.
- Adds unit coverage and runtime proof checks so UPC-bearing books, movies, and TV rows do not regress back into Missing IDs.
- Adds the broader Dashboard Review rules and drawer-first resolution track to the backlog so future review work stays anchored in Dashboard rows and drawers.

## Breaking changes

None.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. Existing rows are not modified; Dashboard Review simply stops flagging UPC-backed books, movies, and TV rows as missing identifiers solely because provider-specific IDs are absent.

## Rollback guidance

Rollback is safe because this release only changes review classification logic and release metadata. If the tuned rules hide records that should still require review, roll back to `3.12.2`; no data repair is needed.

## Known issues and follow-up

- Future Dashboard Review slices should improve drawer actions by media type, especially title cleanup and enrichment repair.
- Dismiss/defer should only be added inside Dashboard Review rows/drawers if smarter rules still leave legitimate `not needed` cases.
- Standalone Review pages/nav items remain out of scope unless a future workflow cannot reasonably live in Dashboard Review.
