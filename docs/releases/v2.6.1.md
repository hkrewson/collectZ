# v2.6.1

## Version and date

## Summary
This release adds structured external log shipping with a canonical event contract, runtime-safe feature-flagged rollout, and three verified operator paths: Graylog via GELF, Loki/Promtail via `stdout_json`, and syslog via RFC5424-compatible TCP/UDP export.

## What changed
### Backend
- Added structured log export service with supported backends:
  - `gelf_udp`
  - `gelf_tcp`
  - `stdout_json`
  - `syslog_udp`
  - `syslog_tcp`
- Added request-id middleware and response propagation for stable request correlation.
- Added promoted detail-key fields for searchability across collectors.
- Added bounded payload truncation and fail-safe exporter behavior so collector failures do not block API/import behavior.
- Added debug tracing for exporter gate and transport diagnosis:
  - `LOG_EXPORT_DEBUG`

### Collector paths and docs
- Added Graylog operator path with live smoke verification.
- Added Loki/Promtail operator path with local starter stack and label extraction guidance.
- Added syslog operator path with local collector example and RFC5424 message guidance.
- Added operator troubleshooting guidance for:
  - `backend_off`
  - env drift
  - runtime exporter mismatch

### Verification and safety
- Added unit coverage for:
  - GELF mapping
  - syslog formatting
  - promoted detail fields
  - config normalization
  - request-id propagation
- Verified live collector delivery for:
  - Graylog
  - Loki
  - syslog

## Deployment and verification
1. Rebuild and start:
   - `docker compose --env-file .env up -d --build backend frontend`
2. Verify reported app version is `2.6.1`.
3. Enable log export path as needed:
   - Graylog: `LOG_EXPORT_BACKEND=gelf_udp`
   - Loki: `LOG_EXPORT_BACKEND=stdout_json`
   - Syslog: `LOG_EXPORT_BACKEND=syslog_tcp`
4. Enable feature flag:
   - `external_log_export_enabled`
5. Verify collector-specific smoke path using the documented stack and runtime checks.

## Rollback guidance
- Roll back application images if needed.
- Disable structured export by setting `LOG_EXPORT_BACKEND=off` or disabling `external_log_export_enabled`.
- Collector stacks and local persistence can remain forward; they are additive operator tooling.
