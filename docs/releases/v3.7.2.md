## Milestone target and status

- Milestone: `3.7.2 — Timeline Sync And Failure Detail Readback`
- Status: `implemented`

## Summary

This patch makes sync failures and sync activity inspectable from the Dashboard and Activity timeline instead of sending users to broad import or integration screens.

## What changed

- Added a shared sync-job detail drawer backed by `/api/media/sync-jobs/:id/result`.
- Changed Dashboard failed-sync rows to open detailed job readback with status, provider, timestamps, failure text, summary counters, progress, and technical payload.
- Added timeline actions for sync jobs and import activity so failed rows can open `Open failure` and successful rows can open `Open sync`.
- Kept existing import navigation available for import-shaped activity rows.
- Added source-level regression coverage to keep the Dashboard, Activity timeline, and sync-job drawer wired together.

## Breaking changes

- None.

## Environment/config changes

- No new environment variables.

## Migration and data impact

- No database migration.
- Existing `sync_jobs` rows and existing activity entries are reused.

- No new public routes or broader sync-job permissions were added.
