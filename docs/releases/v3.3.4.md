## Summary
This patch release broadens the shipped library-loans workflow by bringing title-level loan history into the dedicated `Loans` workspace. The release keeps the implementation intentionally narrow: the main loans list stays lean, while each row can expand into per-title current and returned history with attached reminder-event context. The result is a clearer management view for past loans without turning the workspace into a full operator audit console.

## What changed
### Global loan history readback in the Loans workspace
- Added inline `Show History` expansion for loan rows in the dedicated `Loans` workspace.
- Reused the existing `GET /api/media/:id/loans` detail/history contract instead of widening the top-level `GET /api/media/loans` payload for every row.
- Marked the currently selected row as `This loan` inside the expanded history panel so users can orient quickly within multi-loan title history.

### History readability and structure
- Refined the expanded history panel into grouped `Current` and `Returned` sections.
- Added deterministic history ordering so returned and active records render in a predictable sequence.
- Added compact current-versus-returned counts in the history header for faster scanning.
- Kept loan date-range, borrower, notes, and status information visible without introducing a heavier standalone audit page.

### Reminder context reuse
- Reused the event-level reminder history shipped in `3.3.3` so reminder outcomes appear as supporting context under each historical loan.
- Kept reminder-event readback compact and secondary to the loan record itself.
- Preserved the boundary that reminder history belongs with the loan timeline rather than the top-level list row payload.

### Contract and release sync
- Updated source-backed regression coverage for the inline history expansion, grouped history sections, and deterministic history sorting.
- Regenerated the in-app release feed for `Help > Releases`.
- Synced `3.3.4` version metadata across root, backend, and frontend package/app metadata files.
