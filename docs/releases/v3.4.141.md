# collectZ v3.4.141

## Summary

This patch removes the confusing 1000-title Plex reconciliation cap and adds scheduled reconciliation sync automation.

## Changes

- Makes Plex reconciliation scan all fetched Plex rows by default.
- Keeps an optional diagnostic scan limit, now capped at 50000 instead of 1000.
- Adds runtime-controlled scheduled Plex reconciliation sync automation.
- Adds scheduler status and manual sweep endpoints for Plex reconciliation sync.
- Shows automatic sync status in the Plex integrations panel.
- Keeps the existing conservative sync policy: safe creates, strong-ID updates only, conflicts held for review, and Plex writeback disabled.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- This change does not expose Plex tokens, raw Plex URLs, media file paths, or Plex writeback controls.
