# collectZ v3.8.18

## Version and date

- Version: `3.8.18`
- Date: `2026-05-20`

## Milestone target and status

- Milestone: `3.8.18 — Public Homelab Env Surface Cleanup`
- Status: `implemented`

## Summary

This release tightens the public homelab deployment surface by keeping env examples focused on required runtime values and removing testing/private controls from public compose/docs.

## What changed

- Removed the Playwright browser-regression bypass variable from the generated public compose file.
- Tightened public export validation so public compose/env/docs cannot expose Playwright bypass controls, session bearer fallback controls, or private edition controls.
- Clarified `env.example` around required secrets versus optional database and reverse-proxy settings.
- Updated README wording so public homelab setup does not point users at private/platform edition controls.
- Kept public compose defaulting to `NODE_ENV=production` and secure-cookie posture unless explicitly overridden for local HTTP testing.

## Breaking changes

- None intended for normal homelab deployments.
- Browser regression bypass configuration remains CI/test-only and is no longer surfaced in the public compose file.

## Environment/config changes

- No new required environment variables.
- Minimum required public homelab `.env` values remain `DB_PASSWORD`, `SESSION_SECRET`, and `INTEGRATION_ENCRYPTION_KEY`.
- `DB_USER`, `POSTGRES_DB`, `DATABASE_SSL`, `ALLOWED_ORIGINS`, `SESSION_COOKIE_SECURE`, and `TRUST_PROXY` remain optional deployment posture/customization values.

## Migration and data impact

- No database migration.
- No data changes.

## Security triage

- Public compose no longer advertises Playwright browser-regression bypass controls.
- Public export validation now guards against reintroducing public references to bypass controls or session bearer fallback controls.
- Public docs avoid naming private edition controls in the homelab quick-start path.
