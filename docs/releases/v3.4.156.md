# collectZ v3.4.156

## Version and date

- Version: `3.4.156`
- Date: `2026-05-11`

## Milestone target and status

- Milestone: `3.4.156 — Kavita Chapter Issue Cover Proxy`
- Status: `released`

## Summary

Kavita chapter-as-issue rows now use a chapter-specific cover proxy instead of repeating the parent series cover for every imported issue.

## What changed

- Added a scoped `/api/media/kavita-chapter-cover/{chapterId}` proxy that fetches Kavita reader page `0` server-side.
- Updated Kavita chapter fan-out rows so `poster_path` points at the chapter-cover proxy.
- Added a data repair migration for existing Kavita chapter rows that still pointed at the parent series cover proxy.
- Kept series rows on the existing series-cover proxy.
- Updated Kavita docs and smoke coverage for chapter-level issue cover behavior.

## Breaking changes

- None. Existing Kavita series rows keep the existing series-cover proxy behavior.

## Environment/config changes

- None. The chapter-cover proxy uses the existing workspace-owned Kavita URL and API key settings.

## Migration and data impact

- Adds migration `102`, which updates existing Kavita chapter fan-out rows that still point at `/api/media/kavita-cover/{seriesId}` so they instead point at `/api/media/kavita-chapter-cover/{chapterId}`.
- The migration is scoped to Kavita chapter provider rows with existing series-cover proxy poster paths. User-provided/custom poster paths are not rewritten.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.4.156`.
- Verify Help > Releases serves `3.4.156`.
- Run the Kavita import/sync smoke and confirm fan-out issue rows use `/api/media/kavita-chapter-cover/{chapterId}` poster paths.

## Rollback guidance

- Revert the `3.4.156` code and docs changes to return Kavita chapter issue rows to parent-series cover behavior.

## Known issues and follow-up

- Chapter covers use Kavita reader page `0`, so they depend on Kavita being able to serve that page for the workspace-owned account.
- Existing Kavita source-level series cover 404s remain separate cover-recovery work.
- CI-only secure-cookie compose smoke, secret scan, and image security/SBOM gates still run in GitHub Actions.
