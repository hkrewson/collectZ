## Summary
This release adds Events tracking with artifact capture and image attachments, including scoped API enforcement and activity auditing.

## What changed
- Added Events CRUD API under `/api/events` with scope-aware access control.
- Added Event Artifacts CRUD API under `/api/events/:id/artifacts`.
- Added artifact image upload/delete endpoints and audit actions:
  - `events.attachment.upload`
  - `events.attachment.replace`
  - `events.attachment.delete`
- Added Events UI in the library navigation and full drawer-based create/edit/detail flows.
- Added event filters in UI and API (`q`, `location`, `from`, `to`) with pagination.
- Added frontend nginx proxy for `/uploads/*` so attachment links resolve correctly.

## Breaking changes
- None.

## Environment/config changes
- No new required environment variables.

## Migration and data impact
- Added migration `v35`:
  - `events` table
  - `event_artifacts` table
  - indexes and update triggers
- `collectibles.event_id` linkage is migration-ready when a `collectibles` table exists.
- No destructive data migration.

## Deployment and verification
1. Rebuild/start services:
   - `docker compose --env-file .env up -d --build backend frontend`
2. Verify Events UI appears in Library nav.
3. Create an event, add an artifact, upload an image, and confirm image link opens the file.
4. Verify Activity includes:
   - `events.create|update|delete`
   - `events.artifact.create|update|delete`
   - `events.attachment.upload|replace|delete`
