# collectZ v3.9.5

## Version and date

- Version: `3.9.5`
- Date: 2026-05-20
- Milestone: `3.9.5 — Apple/iTunes Movie Result Relevance Guard`

## Milestone target and status

Status: Complete.

This patch makes Apple/iTunes movie fallback results more honest by scoring title relevance and labeling weak generic movie matches.

## Changes

- Added normalized title relevance scoring for Apple/iTunes movie fallback candidates.
- Added `match_strength`, `match_reason`, `match_score`, and `search_source` fields to normalized Apple/iTunes wishlist candidates.
- Kept exact and close movie matches visible as normal choices.
- Marked weak movie fallback candidates in the Wishlist UI.
- Added a movie-result notice when Apple returns movie rows but none closely match the searched title.

## Security and privacy

- No new user data, secrets, credentials, or background polling behavior are introduced.
- Search remains authenticated through the existing Wishlist route protections.
- Relevance scoring uses only the user's search term and Apple's returned public title metadata.
