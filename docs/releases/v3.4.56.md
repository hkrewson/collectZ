# v3.4.56

## Event schedule catalog-to-personal plan matching

- Version: `3.4.56`
- Target milestone: `3.4.56 — Event Schedule Catalog-to-Personal Plan Matching`
- Date: 2026-05-02

## Why This Exists

Catalog ICS import can now seed the canonical event schedule, while personal Sched ICS sync still represents the user's selected sessions. This patch adds the missing bridge between those two sources so imported catalog rows can recognize already-selected personal Sched sessions without rewriting their source identity.

## What's New

- Added a catalog-session link on personal schedule plans via `source_catalog_session_id`.
- Personal Sched ICS sync links selected sessions to matching imported catalog sessions when source references align.
- Catalog ICS import links any existing personal Sched plans after sessions are created or refreshed.
- Event drawer catalog controls now treat linked personal Sched plans as already selected for the matching catalog row.
- Conflict detection and attendance readback now avoid double-counting the same catalog-backed personal plan.

## Compatibility Notes

- Personal Sched plan identity remains `source_type = sched_ics` with the provider source reference intact.
- Catalog-created plans continue to use `source_type = schedule_catalog` and the catalog session id as `source_ref`.
- The new `source_catalog_session_id` field is API output, not a user-editable source field.
- Raw personal or catalog ICS URLs remain hidden from API responses and release artifacts.
