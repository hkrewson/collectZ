# v3.4.59

## Event schedule notification history and readback UI

- Version: `3.4.59`
- Date: `2026-05-02`
- Target milestone: `3.4.59 — Event Schedule Notification History and Readback UI`

## Summary

- Shows Event-local schedule notification history under expanded schedule rows.
- Makes draft/sent notification records visible after they are created or when the Event drawer is reopened.
- Keeps the readback local and informational: no push, email, device delivery, inbox, or read receipts.

## Changes

- Event social planning drawer now loads existing schedule notification records.
- Schedule rows show a compact `Notification history` block with status, timestamp, recipients summary, and message body.
- Newly created draft/sent records are inserted into the visible row history immediately.
- Browser coverage now checks that local notification history appears after sending a local notice.

## Compatibility Notes

- No schema changes in this release; it reads the `event_schedule_notifications` table introduced in `3.4.58`.
- Existing schedule plans, catalog sessions, and notification records remain compatible.
- External delivery remains unsupported.
