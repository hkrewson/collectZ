# collectZ v3.9.4

## Version and date

- Version: `3.9.4`
- Date: 2026-05-20
- Milestone: `3.9.4 — Apple/iTunes Movie Search Fallback`

## Milestone target and status

Status: Complete.

This patch makes Apple/iTunes Wishlist movie search recover when Apple's movie-specific Search API path returns an empty result.

## Changes

- Added a backend fallback for Apple/iTunes Wishlist movie searches.
- Movie search still tries Apple's typed `media=movie` request first.
- If the typed movie request returns no rows, the backend retries Apple's generic search form and keeps only raw `feature-movie` candidates.
- Kept the fallback backend-owned so the Wishlist UI and scanner-adjacent clients keep one normalized candidate contract.

## Security and privacy

- No new user data, secrets, credentials, or background polling behavior are introduced.
- Search remains authenticated through the existing Wishlist route protections.
- The fallback uses Apple's public Search API response only.
