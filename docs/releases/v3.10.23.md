## Version and date

## Milestone target and status

- Status: `completed`

## Summary

This patch upgrades the browser barcode decoder packages used by Capture Inbox and library lookup capture while keeping the version pair within the published ZXing peer dependency contract.

## What changed

- Updated `@zxing/browser` from `0.1.5` to `0.2.0`.
- Updated `@zxing/library` from `0.21.3` to `0.22.0`, which is the compatible peer range required by the browser package.
- Kept the newer `@zxing/library@0.23.0` PR out of this slice because the current browser package does not declare compatibility with it.

## Breaking changes

None expected.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. This patch changes frontend barcode-decoder dependencies only.

## Known issues and follow-up

- `@zxing/library@0.22.0` advertises a Node engine of `>=24.0.0`; local Node 20 Docker install emits an engine warning but does not fail. collectZ uses this dependency in browser barcode decoding, so this should be watched in CI and revisited if package engine enforcement changes.
- `@zxing/library@0.23.0` remains deferred until `@zxing/browser` publishes a compatible peer range or we intentionally choose and test an override.
