# collectZ v3.19.1

## Version and date

- Version: `3.19.1`
- Date: 2026-06-15

## Milestone target and status

- Roadmap slice: `3.19.1 — Plex Movie Import TV-Field Constraint Fix`
- Status: completed

## Summary

This patch fixes Plex movie imports after media cleanup. Plex movie rows no longer carry TV-only fields from the Plex payload, so deleted movie titles can be recreated without tripping the media TV-field consistency constraint.

## What changed

- Stopped mapping Plex movie `studio` values into the collectZ `network` field.
- Stopped carrying Plex `parentIndex` into `season_number` for non-TV rows.
- Added unit coverage proving normalized Plex movie rows keep TV-only fields empty.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- No schema migration is required.
- Existing rows are not modified automatically.
- Re-running Plex import after deployment should recreate movie rows that previously failed with `media_tv_fields_consistency_check`.
