# v3.4.32

## Release Metadata
- Version: `3.4.32`
- Date: `2026-04-28`
- Target milestone: `3.4.32 — Personal Sched ICS Schedule Detail Enrichment`

## Summary
This patch expands personal Sched ICS schedule plans so synced sessions carry more of the useful detail Sched provides, and updates the Event drawer to show those details more clearly.

## Changes
- Added schedule-plan fields for Sched session URLs, categories, source update timestamps, and source sequence values.
- Updated the personal ICS parser to preserve `CATEGORIES`, `URL`, `DTSTAMP`, and `SEQUENCE` from VEVENT records.
- Cleaned ICS description text before storing it as schedule-plan notes.
- Updated the Event drawer schedule-plan list to show time ranges, locations, Sched source state, categories, description previews, and direct session links.
- Extended unit and smoke coverage so local fixture ICS sync proves the richer fields through the API without exposing personal feed URLs.

## Privacy Notes
- Personal ICS URLs remain encrypted at rest and are still never returned by API readback.
- Session URLs from individual VEVENT rows are stored separately from the secret personal feed URL.
- Release evidence and smoke output continue to avoid real personal ICS URLs.

## Validation
- Docker-first backend/frontend rebuild should report `3.4.32` from `/api/health`.
- Unit, OpenAPI, init parity, migration rehearsal, and focused personal ICS sync smoke should pass locally.
- Tagged CI remains authoritative for dependency scan, secret scan, image security/SBOM, full browser regression, RBAC regression, and edition boundary gates.

## Rollback
Roll back application images to `3.4.31` if enriched schedule-plan fields or Event drawer schedule rendering regress. Migration `86` only adds nullable/defaulted columns to `event_schedule_plans`, so existing schedule rows remain compatible with the previous runtime.
