## Summary

This release adds a read-only Plex now-playing provider proof. collectZ can now parse the PMS `/status/sessions` shape into a safe internal readback contract, backed by a fake-PMS runtime smoke, before any Now Playing UI, webhook, sync, or import behavior is added.

## What Changed

- Added Plex `/status/sessions` to the provider-oriented PMS modernization contract.
- Added safe JSON/XML parsing for now-playing session payloads.
- Normalized only non-sensitive readback fields such as title, type, progress, user label, and player state.
- Added a Docker-runnable fake PMS smoke that writes secret-free now-playing evidence.
- Preserved existing Plex provider discovery, section discovery, and import behavior.

## Compatibility

- No database migration.
- No UI surface is added in this slice.
- Existing Plex imports and provider readback endpoints remain unchanged.

## Deployment Notes

- Build and deploy backend/frontend images with `APP_VERSION=3.4.114`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.114`.
- Verify Help > Releases serves `v3.4.114` as the latest release.

## Rollback

Roll back to `v3.4.113` if the now-playing provider proof needs to be removed. This slice only adds read-only parser/smoke coverage, so rollback risk is limited to the new proof code.

## Known Issues and Follow-Up

- This does not add a Now Playing UI.
- This does not add Plex webhooks or scheduled sync.
- This does not write watch-state back to Plex.
- Real PMS session payloads may vary by Plex server version, player type, and active session state.
