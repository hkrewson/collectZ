# collectZ v3.12.1

## Version and date

- Version: `3.12.1`
- Date: 2026-06-03
- Milestone: `3.12.1 — Dashboard Review Inline Resolution`

## Milestone target and status

- Status: `completed`

## Summary

This patch corrects the Review direction from `3.12.0`. Instead of adding another navigation destination that mirrors Dashboard and Library review filters, Dashboard Review rows now open an inline drawer for the exact media record that needs attention.

## What changed

- Removed the standalone Review navigation item and Review page.
- Removed the read-only `/api/review-queue` endpoint and its OpenAPI contract.
- Changed Dashboard Review rows for missing covers, missing identifiers, and sparse metadata into actionable rows.
- Added a Dashboard Review drawer that fetches the selected media record and saves focused metadata/identifier updates through the existing media update API.
- Removed `View all` buttons from Dashboard Review so the primary path is row-to-drawer resolution.
- Changed Dashboard health counters for missing covers, missing identifiers, and sparse metadata to focus the matching Review tab instead of sending users to a temporary Library review page.
- Allowed book and comic `type_details` to store ISBN-13 and Google Books IDs, matching the identifiers already used by review clues.

## Breaking changes

`GET /api/review-queue` is removed before becoming a stable product contract. The Dashboard summary and media update endpoints remain the supported paths for this Review workflow.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. Existing media rows are unchanged until a user saves updates from the Dashboard Review drawer.

## Rollback guidance

Rollback is safe because this release removes the standalone Review surface and adds focused Dashboard editing through existing media APIs. If inline editing behaves unexpectedly, roll back to `3.12.0` while keeping collection data unchanged.

## Known issues and follow-up

- Cover updates in this first drawer slice accept an existing image URL/path; richer upload/enrichment actions can be added later if the Dashboard drawer proves to be the right place.
- Capture Inbox, Wishlist, Plex conflicts, and failed syncs still resolve in their owning surfaces.
- Broader queue-level dismiss/defer semantics remain intentionally deferred until the Dashboard row workflow proves useful.
