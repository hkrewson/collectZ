- Target milestone: `2.8.1 — Space Creation and Member Onboarding Flow`

## Summary
This release completes the next step after the broader `2.8.0` UI refinement pass by turning global space creation into a more guided onboarding workflow and bringing the main management surfaces into a cleaner, scan-first control model. Space setup, membership follow-up, and invite handling now feel more intentional, while `All Spaces`, `All Members`, and `My Space` share a calmer row-based interaction pattern.

## What changed
### Space creation and onboarding
- Extended global space creation so operators can:
  - create a space,
  - assign the first owner,
  - prepare multiple initial invites in the same flow.
- Added a dedicated admin onboarding backend flow for create-plus-invite orchestration with explicit partial-failure reporting.
- Auto-open the new space drawer after creation so follow-up management happens in context instead of through a disconnected result card.

### All Spaces management
- Reworked `All Spaces` into a scan-first row layout with a stronger column structure for:
  - space,
  - owners,
  - members,
  - libraries,
  - status,
  - actions.
- Replaced archive/unarchive as an operator-facing workflow with direct deletion of empty, non-default spaces.
- Simplified the row presentation by removing low-value slug badges.
- Moved destructive row actions behind an attached ellipsis menu so the list stays calmer.

### Space drawer follow-up controls
- Turned the `All Spaces` drawer into the primary follow-up surface after create.
- Added tabbed sections for:
  - `Add People`
  - `Members`
  - `Invitations`
- Added invite status visibility, revocation, and fresh one-time link issuance from the drawer.
- Avoided exposing old invite links again; operators rotate to a new one-time link instead.
- Added password reset actions for space members from the same drawer context.

### Member management surfaces
- Refined `All Members` into a cleaner scan-first row layout with dedicated columns for:
  - member,
  - email,
  - role,
  - spaces,
  - actions.
- Replaced inline role controls with readable text plus attached ellipsis menus.
- Refined `My Space` members the same way with dedicated email and role fields plus attached ellipsis menus.
- Standardized attached ellipsis action triggers across `All Spaces`, `All Members`, and `My Space`.

## Breaking changes
- No API-breaking changes intended.
- The operator workflow for space deletion changed:
  - empty non-default spaces now delete directly instead of requiring an archive-first step.
- Global/space management views changed materially in layout and interaction style, so operators may need a short adjustment period.
