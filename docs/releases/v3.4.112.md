## Summary

This release proves the new Plex `/media/providers` discovery path against a PMS-shaped runtime response without changing the existing Plex import flow. The smoke starts a fake Plex Media Server, calls the real provider-discovery fetch path, and writes secret-free evidence showing the normalized provider readback.

## What Changed

- Added a backend Plex provider-discovery smoke that exercises `fetchPlexMediaProviders()` against a fake PMS server.
- Wrote provider-discovery evidence to `artifacts/plex-provider-discovery/plex-provider-discovery-smoke.json`.
- Verified the provider readback only exposes safe fields: provider key, title, type, protocol, identifier, and feature keys.
- Kept Plex tokens, provider URLs, file paths, and raw locations out of smoke output and browser-visible contracts.
- Preserved the current legacy Plex import paths; this release does not rewrite imports.

## Compatibility

- No database migration.
- No frontend behavior change.
- No change to existing Plex import, duplicate-avoidance, TV season, or metadata behavior.

## Deployment Notes

- Build and deploy backend/frontend images with `APP_VERSION=3.4.112`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.112`.
- Verify Help > Releases serves `v3.4.112` as the latest release.
- Run `npm run test:plex-provider-discovery-smoke` inside the backend container for the runtime proof.

## Rollback

Roll back to `v3.4.111` if the Plex provider-discovery smoke or release evidence needs to be removed. Runtime Plex import behavior is unchanged, so rollback risk is limited to the new smoke and release artifacts.

## Known Issues and Follow-Up

- This is not the Now Playing Viewer.
- This does not add Plex webhooks or scheduled sync.
- This does not replace legacy Plex import paths.
- Future Plex slices should use this runtime proof before adding user-facing PMS provider features.
