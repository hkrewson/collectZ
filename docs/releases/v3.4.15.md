## Summary
This release brings shared multi-signature provenance management into Art and media drawers. Users can add, edit, archive, and promote signature records without relying on API-only workflows or overwriting the primary-signature compatibility shortcuts.

## What changed
### Drawer-side signature management
- Added signature management panels to Art and media edit drawers.
- Users can add and edit signer name, role, signed date, signed location or event context, proof path, and notes.
- Signature rows clearly distinguish primary and secondary provenance records.

### Primary and archive actions
- Added drawer actions to promote a secondary signature to primary.
- Added drawer actions to archive/remove active signature records.
- Primary changes continue to flow through the shared signature backend so Art signed state and media legacy signing fields stay aligned.

### Regression coverage
- Added source-level coverage for the drawer-side multi-signature wiring.
- Added browser regression coverage for Art and media drawer signature creation, editing, primary promotion, and archiving.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing single-primary signature fields remain compatible and continue to act as shortcuts for the active primary signature.

## Environment/config changes
- No new required operator configuration in `3.4.15`.

## Migration and data impact
- No schema migration is included in this release.
- Uses the shared `signature_records` model and Art/media signature endpoints introduced by earlier `3.4.x` signature provenance releases.
- Existing signature rows are not destructively rewritten.

## Known issues and follow-up
- Each signature record still keeps a single proof path; multi-proof upload/remove management remains future work.
- A signer/person directory remains out of scope until repeated signer reuse proves the need.
- Event autograph artifacts remain capture/history and can link to object signatures, but they are not a separate top-level signature library.
