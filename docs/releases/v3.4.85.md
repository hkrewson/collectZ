# collectZ v3.4.85

## Kavita Digital Library Connection Foundation

This patch adds the first read-only Kavita integration foundation. Admins can configure a Kavita URL and API key, save them with encrypted credential storage, and run a native API connection check that authenticates, reads libraries, samples series, and keeps CWA OPDS settings separate.

### What changed

- Added Kavita URL, API key, and timeout settings to Admin > Integrations.
- Added redacted settings readback for Kavita so the raw API key is never returned by the app.
- Added a Kavita connection test endpoint using the native plugin-auth, libraries, and series APIs.
- Added an `Open Kavita` link from the Kavita integration settings when a base URL is configured.
- Added Docker-friendly smoke coverage with a local fake Kavita-compatible API.
- Kept metadata writeback, embedded reading, reading progress, and full import/sync out of scope for later milestones.
