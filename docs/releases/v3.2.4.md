## Summary
This patch release hardens duplicate-avoidance safety during later imports and syncs. It proves that when strong identifiers materially conflict, the system refuses to mutate an existing merged canonical even if weaker title or year signals look similar. The release covers both generic CSV and Plex movie paths, and it fixes a real null-safety issue exposed while expanding the guard proof matrix.

## What Changed
### Strong-identifier conflict guards
- Added runtime proof that a same-title book import with a conflicting ISBN creates a separate row instead of mutating an existing merged canonical.
- Added runtime proof that a same-title movie import with a conflicting UPC creates a separate row instead of mutating an existing merged canonical.
- Added runtime proof that a same-title Plex movie import with a conflicting TMDB id creates a separate row instead of mutating an existing merged canonical.

### Import-path hardening
- Extended the title-fallback import guard logic so strong-identifier conflicts are checked before reusing same-title candidates.
- Hardened the Plex movie import fallback so it no longer trusts title/year alone when TMDB or UPC conflicts make reuse unsafe.
- Fixed a null-safe incoming strong-identifier helper bug surfaced by the new movie-shaped guard coverage.
