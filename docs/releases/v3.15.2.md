## Milestone target and status

- Roadmap slice: `3.15.2 — Browser-Friendly Backup Export Downloads`
- Status: completed

## Summary

This patch adjusts backup export downloads so browsers receive plain JSON files and individual CSV files instead of archive wrappers.

## What changed

- JSON backup exports now download as a single plain `.json` file.
- CSV exports now expose a list of downloadable files instead of one zipped archive.
- Admin Settings shows separate CSV download buttons for the manifest, restore guidance, uploads manifest, and each exported table.
- The export API can return the CSV file list or one selected CSV file by key.
- OpenAPI and browser regression coverage now document and verify the plain JSON and per-file CSV behavior.

## Breaking changes

- None for the app UI.
- API clients that expected `.json.gz` or `.csv.zip` backup downloads should switch to the plain JSON response or the CSV file-list flow.

## Environment/config

- None.

## Migration/data impact

- No schema migration.
- Export generation remains read-only.
- Uploaded image binaries are still not bundled; CSV exports include an uploads manifest only.
