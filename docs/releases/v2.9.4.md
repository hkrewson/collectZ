## Summary
This release adds a first-class Playwright browser-regression layer to collectZ so release confidence no longer depends almost entirely on backend and API assertions. It ships a documented local and CI browser harness, a separate screenshot-capture path for support and docs use, and focused coverage for the most fragile auth, Help Center, Help Admin, and approved support-session workflows. While building that harness, the browser suite also exposed and drove fixes for request-linked support-session evidence, support-request status updates, and support-admin library switching during an active approved tenant session.

## What Changed
### Playwright browser-regression foundations
- Added a root-level Playwright harness with documented local and CI invocation.
- Added deterministic authenticated setup for browser tests, including seeded users and support fixtures where needed.
- Added failure artifact capture for browser-regression runs so traces, screenshots, and videos are available when a flow breaks.
- Added browser-regression coverage for:
  - login/logout
  - admin shell boot and navigation
  - end-user Help Center support request flows
  - Help Admin support workspace flows
  - approved support-request session start/end behavior
  - temporary `My Space` visibility during an active support session
  - support-library switching during an active approved support session
  - integrations tab switching and save feedback
  - docs/debug surface availability when authenticated admin/debug conditions are met

### Screenshot-friendly support and docs captures
- Added named screenshot capture mode separate from failure artifacts.
- Added a separate manual-friendly browser-captures workflow so support and documentation images can be regenerated without coupling that work to the blocking browser-regression gate.
- Added seeded support/admin capture paths that produce stable screenshots for:
  - login
  - Help Admin conversation and triage views
  - Help Center support and releases views
  - admin integrations

### Browser-driven bug fixes
- Fixed support-request status updates to avoid Postgres parameter-type ambiguity during browser-driven close/reopen flows.
- Fixed request-linked support-session auth so active-session banner evidence keeps request and requester context.
- Fixed support-admin library switching during active approved support sessions so scoped tenant-library changes work only inside the allowed support-session boundary.

### CI and packaging boundary
- Added Playwright as tracked repo test infrastructure and as a CI release gate.
- Kept Playwright out of the shipped backend/frontend runtime images.
- Hardened CI artifact handling so saved Playwright auth bootstrap state stays outside uploaded artifact paths.
- Changed the Playwright CI bypass token flow to use a per-run random token instead of a fixed shared value.

## Breaking Changes
- No end-user workflow was intentionally broken in this release.
- CI and release expectations now include a browser-regression gate for the supported flows above.

## Environment and Config Changes
- Added optional Playwright-only test support via `PLAYWRIGHT_E2E_BYPASS_TOKEN` for deterministic local and CI browser runs.
- This token is test infrastructure only and is not intended as a normal runtime operator setting.
- The release-note feed consumed by `Help > Releases` must continue to be regenerated during release closeout.

## Migration and Data Impact
