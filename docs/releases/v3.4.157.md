## Version and date

- Version: `3.4.157`

## Milestone target and status

- Milestone: `3.4.157 — Barcode Scanner Backend Import API`
- Status: `released`

## Summary

This patch adds the backend scanner contract needed for the iOS barcode scanner app to look up decoded barcodes and import selected matches without routing through the web UI.

## What changed

- Added authenticated `POST /api/media/lookup/barcode` for scanner clients.
- The scanner lookup reads scoped existing catalog matches first, then configured barcode/provider enrichment matches when available.
- Empty barcode lookups now return a successful response with an empty `matches` array.
- Added authenticated `POST /api/media/import-barcode` to create or update canonical media from a selected scanner match.
- Added PAT/service-token scope mapping so scanner lookup uses `media:read` and barcode import uses the existing `import:run` import scope.
- Updated OpenAPI and smoke coverage for lookup success, empty lookup, import success, and auth behavior.
- Follow-up `3.4.158` keeps this scanner lookup contract as a true candidate list when a provider returns multiple products for one barcode.

## Breaking changes

- None.

## Environment/config changes

- Scanner clients should send a bearer PAT or service-account key.
- Lookup is intentionally authenticated because it can disclose existing catalog rows.
- `RATE_LIMIT_EXTERNAL_API_MAX` now also covers `POST /api/media/lookup/barcode`.

## Migration and data impact

- No database migration.
- Barcode imports use the existing media upsert path and store scanner readback metadata in `media_metadata`.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.4.157`.
- Verify Help > Releases serves `3.4.157`.
- Run the barcode scanner API smoke against the running stack.

## Rollback guidance

- Revert the `3.4.157` code and docs changes to remove the scanner-specific barcode lookup/import API while preserving the existing web `/api/media/lookup-upc` path.

## Known issues and follow-up

- The iOS scanner app must send an auth token for lookup and import.
- Provider quality still depends on the configured barcode/books/media integrations.
- CI-only secure-cookie compose smoke, secret scan, and image security/SBOM gates still run in GitHub Actions.
