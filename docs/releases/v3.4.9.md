## Summary
This release introduces one shared signature provenance contract for signed collection objects. Art now records richer signature details through the shared model, while existing media title signature fields continue to work as compatibility projections onto the same foundation.

## What changed
### Shared signature model
- Added a shared `signature_records` table keyed by owner type and owner id.
- Supports `media` and `art` owners in the first slice.
- Backfills existing media signature metadata and existing signed Art rows into primary signature records.
- Keeps one primary active signature record per owner for the current UI and compatibility contract.

### Art provenance
- Art can now store signer name, signer role, signed date, signed location, signed event, proof path, and signature notes.
- Art detail responses include a normalized `signatures` array plus primary-signature compatibility fields.
- Art create/edit/detail surfaces show signature provenance without crowding ordinary unsigned items.
- Art's existing `signed` boolean remains the fast visual marker and stays aligned with shared signature data.

### Media title compatibility
- Media title create/update/detail paths now expose shared `signatures` readback.
- Existing `signed_by`, `signed_role`, `signed_on`, `signed_at`, and `signed_proof_path` fields remain supported.
- Media signing proof upload and removal keep the shared primary signature record in sync.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing media signature fields remain readable and writable.
- Existing signed Art records remain readable and are represented in the shared signature model.

## Environment/config changes
- No new required operator configuration in `3.4.9`.

## Migration and data impact
- Adds migration `78 — Add shared signature provenance records for Art and media`.
- Adds nullable signature provenance storage in `signature_records`.
- Backfills existing signed media and Art data into active primary signature records.
- Existing legacy media signature columns are retained for compatibility.

## Rollback guidance
- Prefer rolling forward with a corrected `3.4.x` build if signature readback or Art provenance behavior needs adjustment.
- If rolling back to `3.4.8`, the new shared `signature_records` table will be ignored by older code, while legacy media signature fields remain available.
- Art provenance fields created only in shared signature records will not be visible in `3.4.8` until a fixed build with the shared signature model is restored.
- Keep a database backup before downgrading across migration `78`.

## Known issues and follow-up
- This release keeps proof images path-based; a richer shared attachment service remains future work.
- This release does not convert Event autograph artifacts into object-level signature provenance.
- This release supports one primary active signature record in the UI; multiple signatures per object can build on the shared table later.
