- Status: closed after local verification

## Summary

This release exposes the proven Plex now-playing parser through read-only admin and workspace integration endpoints. collectZ can now call saved Plex settings for `/status/sessions` and return sanitized active-session readback without adding UI, webhooks, sync behavior, or import changes.

## What Changed

- Added admin and workspace Plex now-playing readback endpoints.
- Reused saved Plex API URL and token settings server-side.
- Returned only sanitized session fields such as title, type, progress, user label, and player state.
- Added a Docker-runnable fake PMS smoke that exercises the collectZ endpoint path.
- Preserved existing Plex provider discovery, section discovery, and import behavior.

## Compatibility

- No database migration.
- No UI surface is added in this slice.
- Existing Plex imports and provider readback endpoints remain unchanged.

## Deployment Notes

- Build and deploy backend/frontend images with `APP_VERSION=3.4.115`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.115`.
- Verify Help > Releases serves `v3.4.115` as the latest release.

## Rollback

Roll back to `v3.4.114` if the now-playing readback endpoint needs to be removed. Plex import behavior is unchanged, so rollback risk is limited to the new read-only endpoints and smoke coverage.

## Known Issues and Follow-Up

- This does not add a Now Playing UI.
- This does not add Plex webhooks or scheduled sync.
- This does not write watch-state back to Plex.
- Real PMS session payloads may vary by Plex server version, player type, and active session state.
