## Kavita Chapter Mark-Read Implementation

## Summary

This patch adds the first runtime Kavita read-state action. Workspace admins can explicitly mark a linked Kavita chapter-backed row read in Kavita, using only Kavita's chapter-level mark-read endpoint.

## What Changed

### Shipped

- Added a scoped `POST /api/media/:id/kavita-read-state` endpoint for Kavita chapter-backed rows.
- Calls only `POST /api/Reader/mark-chapter-read` with `seriesId`, `chapterId`, and `generateReadingSession`.
- Added a `Mark Read in Kavita` drawer action beside the explicit page/progress controls.
- Extended the fake-Kavita import/sync smoke to prove one chapter mark-read call and no bulk read-state calls.
- Updated the read-state contract probe so chapter mark-read is enabled while series, volume, panel, KOReader, and unread endpoints remain prohibited.
