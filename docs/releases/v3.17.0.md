# collectZ v3.17.0

## Version and date

- Version: `3.17.0`
- Date: 2026-06-11

## Milestone target and status

- Roadmap slice: `3.17.0 — Local CI/CD Release Gate Foundation`
- Status: completed

## Summary

This release adds the first local CI/CD gate for collectZ maintainers, giving pre-push work one standard command that checks release, test, dependency, and evidence readiness before public CI runs.

## What changed

- Added `npm run release:local-gate` for the standard local pre-push validation profile.
- Added `npm run release:local-gate:full` for heavier local checks such as CodeQL, secret scanning, runtime smoke, browser regression, and image/SBOM readiness when local tools and runtime state are available.
- Added `npm run release:install-hooks` to install an opt-in pre-push hook that runs the standard local gate.
- Added local gate reports under `artifacts/local-ci/` with redaction for common secret-bearing output patterns.
- Documented the local CI/CD workflow, blocked-gate semantics, hook bypass expectation, and relationship to hosted CI.
- Promoted the local CI/CD backlog item into the roadmap as the first `3.17.0` infrastructure slice.

## Breaking changes

- None.

## Environment/config changes

- Optional: maintainers may install the local pre-push hook with `npm run release:install-hooks`.
- Optional: set `PLAYWRIGHT_E2E_BYPASS_TOKEN` before running the full local gate if local browser regression should run.

## Migration and data impact

- No schema, migration, or collection data impact.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.17.0`.
- Verify `/api/health` reports version/frontend/backend/build `3.17.0`.
- Verify Help > Releases serves `3.17.0`.
- Run `npm run release:local-gate` before push.
- Run `npm run release:local-gate:full -- --fail-on-blocked` when a release handoff requires missing heavy local gates to stop the run.

## Rollback guidance

- Revert to `3.16.36` if the local gate tooling or release workflow documentation needs to be backed out.
- No schema rollback is required.

## Known issues and follow-up

- Local image security/SBOM wiring remains a follow-up; hosted CI still performs the authoritative Trivy/SBOM publish gate.
- Heavy local gates may report `BLOCKED` when optional tools or runtime state are unavailable; use `--fail-on-blocked` for stricter handoff behavior.
- Public repository sanitization remains the next planned infrastructure follow-up after the local gate settles.
