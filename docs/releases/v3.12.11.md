## Summary

This patch tightens Dashboard Review drawer saves so Save is only available when there are actual pending updates. No-op submits now keep the drawer open and explain that a change is needed before saving.

## What changed

- Dashboard Review drawers reuse the pending-update comparison as the source of truth for Save readiness.
- The Save button is disabled until lookup, upload, or manual edits create pending updates.
- No-op submit paths now show `Make a change before saving.` instead of silently closing the drawer.
- Defer, dismiss, and cancel behavior remains independent of pending edits.
