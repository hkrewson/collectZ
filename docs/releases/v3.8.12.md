# collectZ v3.8.12

## Version and date

- Version: `3.8.12`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.12 — Capture Batch Scan Mode`
- Status: `implemented`

## Summary

This release makes Capture Inbox faster for scanning several books or items in a row by adding an explicit batch scan mode.

## What changed

- Added a `Batch scan` action to Capture Inbox.
- Batch mode keeps the editor open and launches the next camera barcode scan after saving.
- The editor now shows a compact batch-session readback with saved, added, and review counts.
- `Save and scan next` keeps unresolved captures in Capture Inbox for later review.
- `Add to library` and `Link` continue to the next scan while batch mode is active.
- Normal one-off `New capture` behavior remains available.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- Batch scan uses the existing Capture Inbox rows, lookup metadata, and selected-match import/link paths.

## Security triage

- Batch scan remains authenticated through the existing Capture Inbox and barcode lookup APIs.
- Import/link actions still use the existing scoped capture import contract.
- No provider credentials or scanner secrets are exposed to the browser.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.12` closeout if any high findings appear.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.8.12`.
- Verify Help > Releases includes `3.8.12`.
- In Capture Inbox, start `Batch scan`, scan or enter an ISBN/UPC, and confirm the batch session readback appears.
- Confirm `Save and scan next` saves the current capture and returns to the next scan.
- Confirm stopping batch mode returns the editor to normal single-capture behavior.

## Rollback guidance

- Revert the Capture Inbox batch scan UI changes and restore `3.8.11` version metadata and release feed if the scan loop regresses.
- No schema rollback is required.

## Known issues and follow-up

- Batch mode still asks the user to choose when a lookup has multiple candidates.
- A later slice can add safer multi-scan review summaries and optional scanner-app queue shortcuts.
