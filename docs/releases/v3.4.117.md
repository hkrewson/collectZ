# collectZ v3.4.117

Released: 2026-05-07

## Version

- Version: `v3.4.117`
- Backend package: `3.4.117`
- Frontend package: `3.4.117`

## Milestone

- Roadmap slice: `3.4.117 — Plex Real PMS Now Playing Runtime Proof`
- Status: closed after local verification

## Summary

This release adds a real Plex PMS now-playing proof path. collectZ can now use saved Plex settings from the running stack to probe `/status/sessions` and write secret-free evidence about what a future Now Playing viewer can rely on.

## What Changed

- Added a backend proof script for saved real Plex now-playing readback.
- Captured viewer-readiness field coverage for active sessions, including progress, player state, Plex-relative poster/art key availability, and queue-presence hints.
- Extended sanitized now-playing session readback with Plex-relative `metadataKey`, `thumbKey`, `artKey`, and `hasQueueItem` fields.
- Kept the proof read-only and separate from imports, webhooks, scheduled sync, watch-state writes, and the dedicated viewer.

## Compatibility

- No database migration.
- Existing Plex imports, provider discovery, fake-PMS now-playing proof, readback endpoint, and Integrations UI behavior remain unchanged.
- If no saved Plex settings exist, the proof writes explicit skipped evidence instead of fabricating a runtime result.

## Deployment Notes

- Build and deploy backend/frontend images with `APP_VERSION=3.4.117`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.117`.
- Verify Help > Releases serves `v3.4.117` as the latest release.

## Rollback

Roll back to `v3.4.116` if the real PMS proof path needs to be removed. The previous Integrations now-playing UI readback remains available in `v3.4.116`.

## Known Issues and Follow-Up

- This does not add a standalone Now Playing viewer.
- This does not add Plex webhooks, scheduled sync, or watch-state writeback.
- Real PMS active-session payloads depend on what is playing at proof time; a reachable server with no active sessions proves connectivity but not poster-key coverage.
