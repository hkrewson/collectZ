## Summary
This patch release finishes the book and comic normalization milestone by turning duplicate handling from a one-off audit into a proven repair workflow. High-confidence ingest now attaches to existing canonical rows instead of multiplying obvious duplicates, medium-confidence matches stay review-scoped, historical comic-like books and duplicate clusters have rollback-safe repair tooling, and merged records now explain what happened directly inside the library drawer with durable match evidence.

## What Changed
### Normalization contract and ingest behavior
- Defined explicit confidence bands and match rationale for books and comics so ISBN-backed books and series/issue/volume comics can auto-attach only when the evidence is strong enough.
- Wired high-confidence normalization into ingest before loose title fallback so obvious Metron and OPDS/CWA duplicates stop creating new rows.
- Kept medium-confidence matches out of silent merges and surfaced them as review-tier import audit outcomes instead.

### Historical repair tooling and live validation
- Added dry-run repair planning for duplicate clusters and comic-like book misclassifications.
- Added rollback-safe historical repair tooling for comic-like `book` rows and high-confidence duplicate attach repairs, including revert support.
- Validated the repair model through targeted real-data pilots for obvious OPDS comic issues, ISBN-backed book duplicates, and conservative comic duplicate pairs.

### Merge evidence and drawer provenance
- Persisted first-class merge evidence in repair history so the app can explain confidence, match rule, merge key, and canonical-selection reasoning from durable data rather than reconstructed guesses.
- Added a persistent drawer-level match-evidence surface for merged books and comics with source/provider labels, compared-field validation, record ids, and merge timestamps.
- Aggregated multiple merge events under one canonical record and proved partial revert behavior so one absorbed duplicate can be restored without disturbing another still-attached merge event.

## Breaking changes
- None intended for operators or end users.
- `3.1.6` changes duplicate handling and evidence visibility for books and comics only; it does not introduce a general-purpose manual merge workflow.

## Environment/config changes
- No new operator-facing environment variables are required for `3.1.6`.

## Migration and data impact
- `3.1.6` does not require a new schema migration for normal runtime upgrades beyond the previously introduced repair-history support used by this milestone.
- Historical repair tooling can mutate book/comic rows when explicitly invoked, but all repair paths used in this milestone were constrained to high-confidence or obvious targeted cases and include revert history.
