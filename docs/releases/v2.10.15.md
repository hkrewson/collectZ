## Summary
This patch release establishes a real platform-owned SMTP delivery model for the SaaS product. Platform operators can now see whether email delivery is configured, manage a masked app-level SMTP override on top of environment defaults, and validate live delivery with a protected test-send flow before the next registration, password-reset, and workspace invite milestones build on top of it.

## What changed
### Platform-owned SMTP model
- Defined SMTP as platform infrastructure rather than workspace-owned configuration.
- Kept runtime `.env` values as the bootstrap/default source for deployment-safe mail delivery.
- Added an optional app-managed SMTP override inside platform settings for operators who need to edit mail delivery without redeploying the stack.

### Safe operator visibility and editing
- Added a protected `Platform Settings > Email Delivery` surface.
- Exposed masked SMTP status details so operators can verify host, port, security mode, sender, and auth state without revealing stored secrets.
- Preserved encrypted-at-rest SMTP password storage instead of echoing plaintext credentials back into the UI or API.
- Added the ability to revert an app-managed SMTP override back to environment-backed delivery.

### Test-send proofing
- Added a protected SMTP test-send action so operators can validate the same platform delivery path future invites, password resets, and self-registration mail will use.
- Verified a live SMTP test send through the running stack after correcting the operator TLS mode for the configured provider.

### Operator docs and API contract
- Updated environment-variable and operator docs so the relationship between env bootstrap, app-managed override, and masked secret handling is explicit.
- Kept the admin API and OpenAPI contract in sync for the new email-delivery settings and test-send surfaces.

## Breaking changes
- No backend or API-breaking changes are introduced in `2.10.15`.
- This release adds platform-owned SMTP configuration capabilities without changing workspace ownership boundaries.

## Environment/config changes
- No new required environment variables are introduced for `2.10.15`.
- Existing SMTP env variables remain the bootstrap/default source of truth:
  - `SMTP_HOST`
  - `SMTP_PORT`
  - `SMTP_USER`
  - `SMTP_PASSWORD`
  - `SMTP_FROM`
  - `SMTP_SECURE`
- Platform operators may now optionally layer an app-managed SMTP override on top of those defaults.

## Migration and data impact
- `2.10.15` adds platform SMTP override fields to `app_settings`.
- Deployments should run the normal backend migration path before using the new `Platform Settings > Email Delivery` surface.
