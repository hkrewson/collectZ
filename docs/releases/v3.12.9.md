## Summary

This patch adds a compact known-identity readback to Dashboard Review drawers, so users can see what collectZ already knows before deciding whether lookup, title cleanup, cover upload, or manual details are needed.

## What changed

- Added a `Known identity` section to Dashboard Review drawers.
- Shows existing UPC/barcode, TMDB, ISBN, Google Books, provider, Kavita, and Plex-like identity values when present.
- Shows a plain empty state when the record has no recognized identifier yet.
- Kept assisted lookup, cover upload, manual details, defer/dismiss, and decision history behavior unchanged.
- Preserved the drawer-first product rule: no standalone Review page or nav item.

## Breaking changes

None.

None.

## Migration and data impact

No schema migration or data backfill is required. The drawer reads existing media record fields and type details already returned by current APIs.

## Known issues and follow-up

- The readback shows known identity values only; it does not yet rank identity quality or explain provider trust.
- Records with a noisy title and no stable identifier still need the assisted lookup/title cleanup path.
- Standalone Review pages/nav items remain out of scope unless a future workflow truly cannot live in Dashboard Review.
