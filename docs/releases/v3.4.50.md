# v3.4.50

## Event schedule conflict detection

- Version: `3.4.50`
- Target milestone: `3.4.50 — Event Schedule Conflict Detection`
- Date: 2026-05-01

### What changed

- Added read-only conflict detection for overlapping event schedule plans.
- Shows quiet `Conflicts with ...` cues in the personal Schedule list.
- Shows conflict cues on Catalog rows and Now / Next items when a catalog session overlaps an active personal plan.
- Treats planned, maybe, and backup sessions as active conflict candidates.
- Skipped plans do not create conflict warnings.
- Uses a one-hour fallback window for sessions with a start time but no end time.

### Why it matters

Quick plan states are useful, but overlapping choices need to be visible. This patch gives users immediate awareness when two sessions collide without forcing a replacement workflow or sending social notifications yet.

### Security and privacy notes

- This patch does not add notifications, friend/group attendance, provider import automation, realtime location, presence, or offline mutation queues.
- Conflict detection is computed from event-scoped schedule data already visible in the drawer.
- Personal Sched ICS feed URLs remain hidden and are not used as a full event catalog source.
