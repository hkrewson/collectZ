# collectZ v3.13.0

## Version and date

## Milestone target and status

- Roadmap slice: `3.13.0 — Backup, Export, and Portability UX Foundation`
- Status: completed

## Summary

This minor release starts the backup/export/portability product surface by adding read-only Admin Settings status for database records, uploaded images, provider metadata, restore guidance, and the backup runbook.

## What changed

- Added an admin-only `GET /api/admin/settings/portability` readback endpoint.
- Added a `Backup and portability` section to Admin Settings.
- Shows database reachability with redacted database location details.
- Shows uploaded image storage status, file count, and size when local uploads are visible to the backend.
- Shows export coverage counts for collection records, wishlist rows, loans, captures, events, libraries, and workspaces.
- Shows provider metadata coverage and restore guidance without exposing secret values.
- Documented the endpoint in OpenAPI and moved the selected backlog item into the roadmap.

## Breaking changes

- None.

## Environment/config changes

- None required.
- Existing `DATABASE_URL`, `DATABASE_SSL`, `STORAGE_PROVIDER`, local uploads, and optional object-storage settings are read for status only.

## Migration and data impact

- No schema migration is required.
- Existing collection data, uploads, provider settings, sessions, and runtime configuration are unchanged.
- The new endpoint performs read-only database/storage checks.

## Known issues and follow-up

- One-click export archives, scheduled backups, restore rehearsal UI, downloadable image bundles, and backup recency tracking are follow-up work.
- The app can verify runtime reachability and configured storage shape, but it cannot prove off-host backup freshness without a future backup job or external backup integration.

## Security vulnerability triage

- Owner: collectZ maintainers.
