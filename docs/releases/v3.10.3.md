- Status: `completed`

## Summary

This patch brings Wishlist, Import, and Admin > Integrations onto the same fixed-header page pattern used by the library surfaces.

## What changed

- Added a shared `FixedPageShell` primitive for pages with stationary headers and independently scrolling content.
- Converted Wishlist to keep its title/add-item header fixed while wishlist search, filters, and rows scroll underneath.
- Converted Import to keep its page title and active-library readback fixed while source tabs and import controls scroll underneath.
- Converted Admin > Integrations to keep its page title fixed while integration settings scroll underneath.
- Added browser regression coverage for the three exception pages so page-level scrolling cannot carry their headers away again.

None.
