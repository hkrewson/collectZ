# collectZ v3.18.6

## Version and date

- Version: `3.18.6`
- Date: 2026-06-13

## Milestone target and status

- Roadmap slice: `3.18.6 — Public Mirror Frontend Source Export`
- Status: completed

## Summary

This patch expands the generated public mirror beyond install/support files by adding the audited frontend source surface and public OpenAPI contract to the export manifest. Backend implementation source remains private.

## What changed

- Added `backend/openapi/` to the public export allowlist.
- Added curated frontend source/build files to the public export allowlist: `frontend/index.html`, `frontend/package.json`, `frontend/package-lock.json`, `frontend/src/`, `frontend/tailwind.config.js`, and `frontend/vite.config.js`.
- Updated public export validation policy to allow those audited paths while continuing to deny backend implementation, private CI, maintainer docs, artifacts, generated builds, and local overrides.
- Updated maintainer public-repository documentation to reflect the new source-publication boundary.

## Breaking changes

- None.

## Environment/config changes

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.18.6`.
- Verify `/api/health` reports version/frontend/backend/build `3.18.6`.
- Verify Help > Releases serves `3.18.6`.
- Run `npm run audit:public-source-boundary` and confirm zero findings.
- Run `npm run validate:public-export`.
- Regenerate `public-export` and confirm the generated tree includes the curated source paths without denied content matches.
- Run `npm run release:local-gate`.

## Rollback guidance

- Revert to `3.18.5` if the public source export boundary needs to be narrowed again.
- No schema rollback is required.

## Known issues and follow-up

- Backend implementation source remains private.
- Public mirror publication still requires an explicit maintainer push from the generated `public-export` repository.
