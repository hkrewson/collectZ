# collectZ v3.4.122

## Summary

This patch adds the first Plex webhook and ratings sync contract. It does not enable automatic Plex webhook ingestion yet; it defines and proves the safe event/readback/writeback shape first.

## Changes

- Added a Plex webhook contract for `library.new`, `media.scrobble`, and `media.rate`.
- Added normalized webhook event readback that treats webhooks as hints and strips Plex tokens, URLs, server identifiers, and raw file paths.
- Added the collectZ-to-Plex rating writeback contract for Plex `PUT /:/rate`.
- Kept watched-state writeback as a future explicit opt-in milestone.
- Added a Docker-runnable fake webhook smoke that proves new-title hints, watched-state hints, rating hints, playback observation, and rating writeback request shape.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- Secret-bearing Plex values remain out of browser-visible webhook readback.
- Webhooks are treated as event hints; PMS metadata readback by `ratingKey` is required before mutating collectZ records.
- Raw Plex payloads, tokens, provider URLs, media file paths, IP addresses, and machine identifiers are intentionally not preserved in the normalized contract evidence.
- Rating writeback is contract-only in this release and does not write to Plex automatically.
