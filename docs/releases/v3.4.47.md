# v3.4.47

## Event schedule catalog entry polish

- Version: `3.4.47`
- Target milestone: `3.4.47 — Event Schedule Catalog Import/Entry Polish`
- Date: 2026-05-01

### What changed

- Added an Event drawer Catalog section for browsing canonical event schedule sessions separately from personal schedule plans.
- Added manual catalog session entry from the Event drawer.
- Added inline catalog session editing for title, time, location, room, track, categories, source URL, description, and status.
- Added quiet archive behavior for catalog sessions.
- Added a guarded `Add to schedule` action that creates a personal/private schedule plan from a catalog session and disables once that catalog session is already in the schedule.
- Kept Sched personal ICS plans separate from the canonical catalog model.

### Why it matters

The catalog object from `3.4.46` is now usable from the web app. Users can start building an event's session catalog manually and selectively add catalog sessions to their own schedule without confusing catalog data with personal plans.

### Security and privacy notes

- This patch does not add provider scraping, background catalog sync, notifications, realtime location, presence, or offline mutation queues.
- Catalog entries remain event-scoped through the existing backend event access boundary.
- `Add to schedule` creates private schedule plans by default.
