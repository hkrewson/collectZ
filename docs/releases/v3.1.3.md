## Summary
This patch release cleans up the shared library workflow after the `3.1.2` shell pass by simplifying pagination, clarifying bulk-selection behavior, and flattening valuation presentation. The collection surface now reads more clearly during day-to-day browsing and bulk management without widening back into broader shell or drawer redesign work.

## What Changed
### Pagination and footer control cleanup
- Replaced the older verbose `Previous` and `Next` footer wording with quieter directional pagination controls.
- Tightened the page-size and results-footer treatment so navigation feels lighter and less repetitive.
- Kept pagination behavior intact while reducing the amount of interface copy users have to parse.

### Bulk-selection behavior cleanup
- Removed the idle `Bulk Actions` framing and clarified the difference between selecting the current page and selecting all matching results in the active library type.
- Moved selection state into the results header so it reads as content-region state instead of a second heavy toolbar.
- Kept escalation from page selection to full-result selection available without overloading one ambiguous `Select all` affordance.

### Valuation presentation cleanup
- Simplified valuation display in library detail drawers into a straightforward `Low` / `Mid` / `High` row.
- Reduced the visual weight of the refresh affordance so valuation reads like normal metadata rather than a noisy utility widget.
- Kept the valuation treatment compact so it supports the drawer instead of dominating it.

### Supporting regression alignment
- Carried the browser-regression contract correction for the space-manager flow so fallback manageable-workspace behavior is asserted correctly after suspension or removal with surviving unrelated workspace access.
