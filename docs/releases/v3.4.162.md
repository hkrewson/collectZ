# collectZ v3.4.162

## Version and date

- Version: `3.4.162`
- Date: `2026-05-17`

## Milestone target and status

- Milestone: `3.4.162 — Scanner ISBN Lookup Fallback Guard`
- Status: `released`

## Summary

This patch keeps valid ISBN scanner and web lookups on the Books/ISBN path even when the Books provider has no match.

## What changed

- Valid ISBN-10/ISBN-13 barcode values now return `provider: books:isbn-direct` from scanner lookup without falling through to UPCItemDB/general barcode lookup.
- ISBN no-match lookups return a successful empty `matches` list instead of a potentially unrelated barcode-provider candidate.
- Checksum-valid non-ISBN EAN/UPC values, such as `0076783005990`, remain on the barcode-provider path so provider title variants still surface.
- The web barcode lookup path follows the same ISBN no-fallback behavior.
- Public templates, browser metadata, package names, generated compose defaults, and operator docs now use `collectZ`/`collectz` naming instead of the old app-name defaults.
- Scanner smoke coverage now proves ISBN direct success and ISBN direct no-match behavior for both scanner and web lookup routes.
- OpenAPI now documents that ISBN no-match scans stay on `books:isbn-direct`.

## Breaking changes

- None.

## Environment/config changes

- New template/default install values use `collectz` for database/user/bucket examples. Existing `.env` values are unchanged and continue to override these defaults.
- This also uses existing Books and barcode integration settings.

## Migration and data impact

- No database migration.
- No existing media rows are changed by this release.
- Existing deployments with explicit `DB_USER`/`POSTGRES_DB` values keep their configured database identity.

## Security triage

- Lookup auth behavior is unchanged.
- Scanner lookup remains authenticated with `media:read`.
- Scanner import remains authenticated with `import:run`.
- Provider secrets remain server-side integration settings.
