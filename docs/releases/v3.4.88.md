# collectZ v3.4.88

## Kavita Volume/Chapter Detail Enrichment

This patch enriches read-only Kavita imports with native volume and chapter details so book and comic rows carry more useful issue/chapter context without adding writeback or reader/progress scope.

### What changed

- Added Kavita volume/chapter readback during import through the native `/api/Series/volumes` API.
- Stored lightweight volume counts, chapter counts, volume numbers, first chapter id/title/number/publication date/page count, total chapter pages, and chapter title summaries in Kavita provider detail fields.
- Improved comic issue mapping by deriving missing comic volume, issue number, and cover date from the first clear Kavita volume/chapter detail.
- Kept existing non-Kavita title reuse, repeat-sync idempotency, and manual/local metadata preservation intact.
- Kept metadata writeback, embedded/in-frame reading, reading progress, cross-server identity, and provider framework cleanup out of scope.
