# v2.8.4

## Version and date
- Version: `2.8.4`
- Date: `2026-03-29`

## Milestone target and status
- Target milestone: `2.8.4 — Scope Privacy Tightening and Explicit Support Access`
- Status: `Completed`

## Summary
This release tightens the tenancy privacy model so global admins no longer drift into tenant context through everyday scope controls. Cross-space troubleshooting now happens through explicit, audited support sessions with clearer shell treatment, support-only library selection, and platform-mode bootstrap outside support mode.

## What Changed
### Privacy tightening
- Removed everyday sidebar space switching from the normal shell flow.
- Limited ordinary library switching to the current non-admin tenant context where it is actually meaningful.
- Converted admin bootstrap outside support mode to platform-mode session context instead of returning tenant-scoped active space/library data.

### Explicit support access
- Added explicit admin support-session start and end flows:
  - `POST /api/auth/support-session/start`
  - `DELETE /api/auth/support-session`
- Added session-scoped support metadata and audit context to session storage.
- Support access is now explicitly named, time-bounded, and auditable instead of acting like background scope switching.

### Support-session UX
- Added stronger support-session shell treatment so the app more clearly signals when an admin is operating inside tenant support mode.
- Replaced the earlier support-entry prompt flow with an in-drawer support-start flow in `All Spaces`.
- Support sessions now allow an explicit initial library selection at session start when the target space has more than one active library.
- Admin library changes are routed through support-session controls only.

### Admin path hardening
- Generic admin scope mutation paths are blocked in favor of support-session controls:
  - `POST /api/auth/scope`
  - `POST /api/spaces/select`
  - `POST /api/libraries/select` when no support session is active
- Added/strengthened runtime smoke coverage proving:
  - admin bootstrap stays platform-mode outside support mode,
  - generic admin space/library selection is denied before support starts,
  - tenant access opens only during the explicit support session,
  - tenant access closes again when the session ends.

## Breaking Changes
- Global admins no longer use generic active-space or active-library selection as a normal navigation model.
- Admin bootstrap outside support mode no longer advertises tenant-space lists or active tenant ids.
- Any operator automation or internal tooling that previously depended on generic admin scope-selection endpoints must move to explicit support-session flows.

## Environment and Config Changes
- No new required environment variables.
- Existing session, CSRF, and audit/logging configuration remains the basis for support-session behavior.

## Migration and Data Impact
- Existing schema migration for session-scoped support metadata remains required for this milestone line.
- No new migration was added in the final `2.8.4` release closeout pass.
- Support-session state is session-scoped and cleared on explicit session end.
