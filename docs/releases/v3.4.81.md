# collectZ v3.4.81

## Event Self-Attendee Default Creation on First Social Action

This patch makes Event social planning feel less procedural. When the signed-in user takes a first social action like creating a group, meetup, or manual plan without a self attendee yet, collectZ now creates that self attendee automatically before completing the action.

### What changed

- Auto-creates the signed-in user's Event attendee record when they create their first group, meetup, or manual schedule plan.
- Uses that just-created self attendee as the first member when the first social action is creating a group.
- Keeps the explicit `Add me to this event` path available when the user wants to add themselves directly first.
- Stops observability structured-log evidence from toggling the Events sidebar feature flag by switching that smoke path to a non-library platform flag.
- Preserves the existing Event-local attendee ownership model and linked-user readback.
- Keeps external contact identities, cross-event identity, native companion mutation UX, and broader friend-graph work out of scope.
