# collectZ v3.4.92

## Kavita Chapter-as-Issue Row Fan-out Contract

This patch defines the future contract for importing selected Kavita comic/manga chapters as individual collectZ `comic_book` rows while keeping the existing Kavita series-level import as the default behavior.

### What changed

- Added `docs/wiki/43-Kavita-Chapter-Issue-Fanout-Contract.md`.
- Defined distinct provider identities for series rows and future chapter/issue rows: `kavita:series:{seriesId}` and `kavita:chapter:{chapterId}`.
- Documented opt-in, comic-only eligibility rules for future fan-out.
- Defined duplicate handling, local metadata preservation, and parent-series linkage expectations.
- Added a smoke plan for repeat sync idempotency, local issue preservation, book-library exclusion, and secret-free launch/cover URLs.
- Linked the fan-out contract from the Kavita setup guide.
- Added unit/source assertions that keep the identity boundary and off-by-default behavior visible.
