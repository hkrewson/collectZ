## Summary
This patch release hardens collection-side duplicate prevention after collection merges. It proves that a merged collection can survive a later boxed-set CSV import without recreating the absorbed duplicate container, and it fixes the collection identity persistence needed to make that behavior durable in runtime.

## What Changed
### Collection import alias durability
- Added collection-side absorbed identity preservation for merged collections through `import_collection_aliases`.
- Taught collection-only import lookup to reuse those stored aliases instead of relying only on the current `source_title + import_source + media_type + scope` key.
- Fixed alias normalization so stored snake_case collection alias fields survive dedupe and matching correctly.

### Collection re-sync runtime proof
- Added a Docker-backed runtime smoke that:
  - merges a duplicate collection into a canonical collection,
  - preserves the absorbed CSV collection identity,
  - reruns a real boxed-set CSV collection import,
  - proves no duplicate collection row is recreated,
  - and confirms the surviving canonical collection remains the landing point for later collection item updates.
