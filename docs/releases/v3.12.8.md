## Summary

This patch adds recent review decision context to Dashboard Review drawers, so users can see prior defer, dismiss, and restore activity when a finding returns.

## What changed

- Added recent Dashboard Review decision activity to review row payloads.
- Added a compact `Recent review decisions` section to the Dashboard Review drawer.
- Kept the history scoped to the same media item and finding type.
- Preserved the drawer-first product rule: no standalone Review page or nav item.
- Updated OpenAPI and source-contract coverage for review decision history.

## Breaking changes

None.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. The new readback uses existing activity log entries created by Dashboard Review defer, dismiss, and restore actions.

## Known issues and follow-up

- The drawer shows a small recent history sample, not a full audit trail.
- Restored decisions from older versions only appear when corresponding activity log entries exist.
- Standalone Review pages/nav items remain out of scope unless a future workflow truly cannot live in Dashboard Review.
