- Version: `3.15.3`
- Date: 2026-06-06

## Milestone target and status

- Roadmap slice: `3.15.3 — Backup Freshness Readback`
- Status: completed

## Summary

This patch lets the backup and portability surface show whether an external scheduled backup job has reported a recent successful run.

## What changed

- Added optional backup freshness readback to `GET /api/admin/settings/portability`.
- Added a marker-file contract using `COLLECTZ_BACKUP_STATUS_PATH` and `COLLECTZ_BACKUP_FRESHNESS_HOURS`.
- Admin Settings now shows a compact Backup freshness card next to database, images, and provider metadata.
- Backup checks now include a freshness row with plain status detail.
- The backup runbook documents the scheduled-backup marker JSON shape.

## Breaking changes

- None.

## Environment/config

- Optional: set `COLLECTZ_BACKUP_STATUS_PATH` to a JSON marker file written by your backup job.
- Optional: set `COLLECTZ_BACKUP_FRESHNESS_HOURS` to adjust the freshness target. The default is 24 hours.

## Migration/data impact

- No schema migration.
- Manual JSON and CSV exports are unchanged.
- collectZ still does not perform scheduled backups or restore automation; it reads status from an external backup job.
