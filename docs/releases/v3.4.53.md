# v3.4.53

## Event schedule catalog filters

- Version: `3.4.53`
- Target milestone: `3.4.53 — Event Schedule Catalog Filters`
- Date: 2026-05-01

### What changed

- Added compact filters to the Event drawer schedule catalog list.
- Catalog sessions can now be filtered by time window: All, Now, Next, and Later today.
- Catalog sessions can now be filtered by plan state, including Not in schedule.
- Added quick toggles for Conflicts only and Has shared attendance.
- Empty filtered results now show a clear no-matches message instead of leaving the list ambiguous.

### Why it matters

The catalog is becoming a real day-of-con planning surface. These filters make long schedules easier to scan without changing the underlying event/social model or adding notification behavior before the platform contract is ready.

### Security and privacy notes

- This patch is read/filter UI only and does not add notification delivery, friend discovery, platform relay identity, device registration, realtime location, presence, or offline mutation queues.
- Shared-attendance filtering uses existing visibility-aware schedule-plan readback.
- Raw personal Sched ICS feed URLs remain hidden and are not used as catalog filter input.
