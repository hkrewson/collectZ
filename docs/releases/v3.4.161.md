# collectZ v3.4.161

## Version and date

- Version: `3.4.161`
- Date: `2026-05-16`

## Milestone target and status

- Milestone: `3.4.161 — Scanner Barcode Import Enrichment`
- Status: `released`

## Summary

This patch makes scanner barcode imports run the existing backend enrichment pipeline before creating or updating the canonical media row.

## What changed

- `POST /api/media/import-barcode` now enriches the selected scanner match with available ISBN, UPC, title, and provider data before upsert.
- ISBN book imports can populate Books/Google Books metadata such as overview, cover image, publisher, and type details.
- Import responses now include `enrichment_status`, `lookup_path`, and `lookup_status` so scanner clients can show what happened.
- Barcode import activity and metadata record the scanner enrichment outcome for readback/debugging.
- Scanner smoke now proves ISBN import enrichment end-to-end with a fake Books provider.

## Breaking changes

- None.

## Environment/config changes

- None. This uses existing integration settings.

## Migration and data impact

- No database migration.
- New scanner imports may create richer canonical rows when provider metadata is available.
- Existing rows updated by scanner import can receive missing enrichment fields through the normal import upsert behavior.

## Security triage

- Import auth behavior is unchanged.
- Scanner import remains authenticated with `import:run`.
- Enrichment uses server-side integration settings; scanner clients do not receive provider secrets.

## Security vulnerability triage

- High findings: none in local dependency audits.
- Owner: collectZ maintainers.
- Target remediation milestone: not needed for this release because no high findings were present locally.
