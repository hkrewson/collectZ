## Summary
This patch migrates existing bridge-era Art data into the native Art storage introduced in `3.4.1`. Existing `collectibles.subtype = art` rows now receive native `art_items` mappings, and event-linked Art purchases now receive shared `event_purchased_items` links so Events can carry Art purchase history through the same canonical relationship used for tracked purchased objects.

## What changed
### Legacy Art migration
- Added migration `v75 — Backfill native art rows and shared event purchased item links`.
- Backfilled existing Art-classified Collectibles into native `art_items` through `source_collectible_id`.
- Preserved:
  - title
  - artist
  - series
  - vendor
  - booth
  - price
  - exclusive state
  - image path
  - notes
  - creator attribution
  - library and space scope
  - created/updated timestamps
  - archived state

### Event purchase backfill
- Backfilled shared `event_purchased_items` rows for legacy Art records with an existing `event_id`.
- Preserved stable purchase snapshots:
  - `title_snapshot`
  - `vendor_snapshot`
  - `booth_snapshot`
  - `price_snapshot`
- Kept non-Art Collectibles out of the Art migration backfill.
- Kept `event_artifacts` as the freeform event-history lane for notes, ad-hoc purchases, freebies, signings, and other non-canonical event memories.

### Migration safety
- Made the backfill idempotent so it can be safely rerun without duplicate native Art rows or duplicate shared purchased-item links.
- Added transactional smoke coverage for active, standalone, archived, and event-linked legacy Art rows.
- Preserved bridge-era `/api/art` reads for this release; native Art read cutover remains planned for `3.4.3`.

## Breaking changes
- None intended.
- Runtime Art reads still use the bridge-era `/api/art` surface in this release.
- Existing Art rows are now mapped into native Art storage, but the product-facing cutover is intentionally deferred.

## Environment/config changes
- No new required operator configuration in `3.4.2`.

## Migration and data impact
- Adds migration `v75`.
- Existing Art-classified Collectible rows are copied into native `art_items`.
- Existing event-linked Art rows are copied into shared `event_purchased_items`.
- The migration preserves archived state and does not delete or reclassify legacy Collectible rows.

## Rollback guidance
- Roll back application images to `3.4.1` if migrated Art mappings or shared event purchase links cause runtime issues.
- Because this release adds derived rows rather than destructive changes, rollback should keep the database in place and restore the prior application image unless a maintainer explicitly chooses a database restore path.
- Native Art rows and shared purchased-item links created by the migration can remain present for a later corrected release.
