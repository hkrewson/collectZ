# collectZ v3.8.17

## Version and date

- Version: `3.8.17`
- Date: `2026-05-20`

## Milestone target and status

- Milestone: `3.8.17 — Capture Review Reason Readback`
- Status: `implemented`

## Summary

This release makes Capture Inbox rows explain why they need attention, using backend-derived review reasons.

## What changed

- Added `review_reasons` readback to capture items.
- Derives reasons for replay conflicts, lookup problems, multiple choices, no matches, ready-to-add captures, and missing details.
- Shows those reasons directly on Capture Inbox rows as compact text.
- Documents the new readback in OpenAPI.
- Preserves existing review tabs, source filters, batch summary, scanner shortcuts, and import/link actions.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- Review reasons are computed from existing capture row fields and `review_decision` metadata.

## Security triage

- Capture Inbox remains authenticated and scoped.
- Review reasons are derived from capture metadata already returned for the active workspace/library.

## Security vulnerability triage

- High findings: none identified in local checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.17` closeout if any high findings appear.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.8.17`.
- Verify Help > Releases includes `3.8.17`.

## Rollback guidance

- Revert the `review_reasons` backend readback and Capture Inbox row text, then restore `3.8.16` version metadata and release feed.
- No schema rollback is required.

## Known issues and follow-up

- Review reasons are readback only; they do not create a durable unified review queue yet.
- Visible offline queue management and cover/art recognition remain future Capture workflow work.
