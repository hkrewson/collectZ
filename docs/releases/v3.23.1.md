## Summary

This release tunes the maintainability guardrails added in `3.23.0` so they are more useful during everyday work. It adds a changed-file quality command and makes the baseline report show the highest-volume rules and files without turning lint into a blocking gate yet.

## What changed

- Added `npm run quality:frontend:changed` for report-first checks against changed, staged, and untracked files.
- Enhanced the frontend quality report with top ESLint rule and file summaries.
- Tuned frontend ESLint so legacy `no-unused-vars` findings are warnings while React hooks and accessibility rules remain visible.
- Updated maintainability guardrails documentation for the changed-file workflow.
