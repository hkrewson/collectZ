## Summary
This release completes the next shared signature provenance slice by giving Art the same proof-image upload and removal path already available to media signing proofs. Art proof files now update the shared primary signature record directly, while media proof removal keeps the legacy media field and shared signature record aligned.

## What changed
### Art proof attachments
- Added `POST /api/art/:id/upload-signature-proof` for uploading or replacing an Art signature proof image.
- Added `DELETE /api/art/:id/signature-proof` for removing the primary Art signature proof path.
- Art proof uploads create or update the active primary `signature_records` row with `owner_type = 'art'`.
- Art proof removal clears only the proof path and preserves the rest of the signer/provenance details.

### Art drawer workflow
- Added a signature proof image picker to the Art drawer.
- Added inline proof upload, remove, and open-current-proof actions.
- Kept the existing proof URL field as a simple fallback for externally hosted proof paths.
- New Art records can upload a selected proof file immediately after the Art item is saved.

### Media compatibility
- Tightened media signing proof removal so it clears `signed_proof_path` and syncs the shared primary signature record from the same update result.
- Existing media signing proof upload and legacy readback behavior remain compatible.

### API contract and regression coverage
- Updated OpenAPI for the new Art proof upload/remove endpoints and response shape.
- Expanded source-level regression checks for shared signature proof attachments.
- Expanded the Events/Collectibles/Art browser regression to cover Art proof upload, readback, and removal.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing Art proof URL values remain readable through the primary signature record.
- Existing media `signed_proof_path` compatibility remains supported.

## Environment/config changes
- No new required operator configuration in `3.4.10`.

## Migration and data impact
- No schema migration is included in `3.4.10`.
- Art proof attachments reuse the `signature_records` table introduced in `3.4.9`.
- Existing proof paths remain path-based; no general attachment library is introduced in this release.

## Rollback guidance
- Prefer rolling forward with a corrected `3.4.x` build if Art proof upload or removal behavior needs adjustment.
- If rolling back to `3.4.9`, uploaded Art proof files may remain in object storage/local upload storage, but Art proof file upload/remove controls will not be available until `3.4.10` or later is restored.
- Because no schema migration is included, rollback does not require a database schema downgrade beyond the existing `3.4.9` shared signature model.
