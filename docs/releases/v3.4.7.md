## Summary
This release makes Event-side tracked purchase management feel first-class now that Art and Collectibles share the same purchased-item relationship. Event details can search, link, read back, and edit tracked Art or Collectibles purchases without sending the user through each library first.

## What changed
### Event purchase linking
- Added Event detail controls to search existing Art or Collectibles and link them as tracked purchases.
- Art search/linking uses native Art item IDs while remaining compatible with bridge-era Art responses that expose `native_art_id`.
- Duplicate purchase links are disabled in the search results and backend conflict responses now surface clearer duplicate-link copy.
- Successful linking now collapses the search panel so the Event drawer returns to the tracked purchase readback list.

### Event purchase editing
- Added inline Event-side editing for purchase snapshots: title, vendor, booth, and price.
- Snapshot edits update the shared `event_purchased_items` relationship without changing the source Art or Collectibles record.
- Existing readback still prefers source item details when snapshots are empty, so archived or edited source items remain readable from the Event.

### Regression coverage
- Added browser coverage for searching, linking, and editing a native Art purchase from Event detail.
- Extended unit source assertions so the Event purchase manager keeps the Art/Collectibles search split, duplicate-link feedback, and purchase edit save path wired.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing freeform Event artifacts remain separate from tracked purchased-item links.

## Environment/config changes
- No new required operator configuration in `3.4.7`.

## Migration and data impact
- No schema migration is required.
- The release keeps using the existing shared `event_purchased_items` relationship.
- Event-side snapshot edits only update purchase-link snapshot fields; they do not mutate Art or Collectibles source records.

- Roll back application images to `3.4.6` if the Event purchase manager causes runtime issues.
- Because this release does not add a schema migration, rollback does not require a schema downgrade.
