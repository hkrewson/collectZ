## Summary

This patch improves the Dashboard Review drawer so missing-cover and missing-identifier rows feel like actionable review work instead of wide manual edit forms.

## What changed

- Made the Dashboard Review drawer narrower, padded, and focused around the review action first.
- Added direct cover upload to missing-cover review rows using the existing media cover upload path.
- Kept a cover URL/path field as a fallback for pasted or provider-hosted images.
- Added provider search to missing-identifier review rows so users can search by corrected title/year and apply candidate data.
- Applying a match can stage title, year, cover, and provider identity details before saving.
- Moved broad manual fields behind a `Manual details` section so manual identifier entry is available but no longer the primary path.
- Added source-contract and browser coverage for assisted review drawer behavior.

## Breaking changes

None.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. Existing media rows are unchanged until a user uploads a cover, applies a candidate, or saves manual updates from Dashboard Review.

Rollback is safe because this release only changes Dashboard Review UI behavior and uses existing media upload/search/update APIs. If assisted review behaves unexpectedly, roll back to `3.12.1`; collection data remains valid.

## Known issues and follow-up

- Some missing-identifier rows may still be better handled by improved title cleanup, enrichment, or source-specific import repair.
- A future review rules milestone should decide when collectZ should ask for identifiers versus when it should attempt repair automatically.
