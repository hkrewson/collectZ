## Summary

This patch brings utility pages into the mobile header cleanup line. Wishlist, Import, Integrations, and Capture Inbox now use a shared utility page header pattern with their main search, filter, or section controls kept in the fixed page header and compacted after the user scrolls into the page content.

## What changed

- Added a shared `UtilityPageHeader` primitive for utility-page title, subtitle, actions, and workflow-specific controls.
- Extended `FixedPageShell` so utility pages can react to body scroll and compact their fixed header.
- Moved Wishlist status/search filters into the fixed header and compacted the add action on scroll.
- Moved Import source tabs into the fixed header so the source selector stays available while the import form scrolls.
- Moved the mobile Integrations selector into the fixed header while keeping the desktop integration tabs in the page body.
- Converted Capture Inbox to the fixed shell and moved capture status, source, search, and review filters into the shared utility header.

## Known issues and follow-up

- If the fixed controls still feel too tall on small phones, the next UI slice should consider icon-expanded search or an overflow filter drawer for secondary filters.
