## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `LibraryLoansView.jsx` was the top remaining frontend lint pocket after `3.23.12`. The slice clears Library Loans import, accessibility, and hook-baseline findings without changing loan tracking behavior.

## What changed

- Removed stale import and unused prop surface from the Library Loans view.
- Replaced the loan editor backdrop click target with a native button control.
- Initialized the loan editor from the selected loan without a form synchronization effect.
- Moved filter/search pagination resets into the user-input paths that trigger them.
- Documented the loan-list fetch effect that intentionally synchronizes backend-owned loan state from the active filter/page query.
- Reduced the full frontend quality baseline from `37` errors / `27` warnings to `31` errors / `25` warnings.
