## Summary
This patch release makes the add/edit drawers feel more like normal working surfaces and less like staged modal moments. The editor header keeps the same structure and actions, but the title no longer shouts in forced all-caps display styling and instead reads like a calmer product workspace heading.

## What changed
### Add/edit drawer heading normalization
- Removed the forced all-caps transformation from the drawer title.
- Removed the tracking-heavy display heading treatment from the editor header.
- Switched the add/edit drawer heading to a calmer product-style title treatment with normal casing and stronger readability.

### Drawer workspace tone
- Preserved the current header layout, close/back affordance, and editor actions.
- Kept the add/edit drawer hierarchy intact while reducing the staged modal feel in the title row.

## Breaking changes
- No backend or API-breaking changes are introduced in `2.10.10`.
- This release intentionally changes add/edit drawer title presentation to be calmer and less display-oriented.
