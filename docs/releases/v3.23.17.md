## Summary

This release adds a dedicated mobile auth contract for the native SwiftUI scanner companion app. Mobile clients can now use opaque, mobile-scoped bearer tokens to read/write Capture Inbox records without relying on browser cookies, CSRF, personal access tokens, provider keys, or direct enrichment APIs.

## What changed

- Added `POST /api/mobile/auth/login`, `POST /api/mobile/auth/refresh`, `POST /api/mobile/auth/logout`, and `GET /api/mobile/auth/session`.
- Added hashed-at-rest mobile auth sessions with short-lived access tokens, longer-lived refresh tokens, refresh rotation, revocation, device metadata, and last-used tracking.
- Added mobile bearer principal support in auth middleware with scopes limited to `profile:read`, `capture:read`, and `capture:write`.
- Allowed mobile tokens on `/api/auth/me` and `/api/capture-items` only, while keeping provider/enrichment/import/admin route families unsupported by default.
- Documented the native scanner barcode capture payload in OpenAPI.

## Breaking changes

- None. Existing browser session auth, CSRF behavior, personal access tokens, and service account keys continue to use their existing scopes.

## Environment/config changes

- Optional `MOBILE_ACCESS_TTL_MINUTES` controls mobile access token lifetime, defaulting to `15`.
- Optional `MOBILE_REFRESH_TTL_DAYS` controls mobile refresh token lifetime, defaulting to `30`.

## Migration and data impact

- Adds `mobile_auth_sessions` through migration `116`.
- Stores only token hashes server-side; raw mobile access and refresh tokens are returned once to the client.
