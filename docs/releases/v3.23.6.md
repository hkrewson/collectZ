## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `CollectiblesView.jsx` was the top remaining frontend lint pocket after `3.23.5`. The slice extracts small Collectibles UI pieces, cleans the touched row/drawer surface, and leaves Collectibles product behavior unchanged.

## What changed

- Extracted Collectible card and list-row UI from `CollectiblesView.jsx`.
- Made Collectible list rows keyboard reachable while preserving existing open/edit/delete behavior.
- Replaced the Collectible editor backdrop with a native button.
- Documented remaining Collectibles API/focus synchronization effects instead of rewriting Collectibles data flow.
- Reduced the full frontend quality baseline from `99` errors / `44` warnings to `89` errors / `39` warnings.
