# collectZ v3.4.124

## Summary

This patch lets Plex `library.new` webhooks create a safe import hint job without silently running a full Plex import.

## Changes

- Added webhook import-hint enqueueing for Plex `library.new` events with a valid receiver token and `ratingKey`.
- Kept `media.scrobble`, `media.rate`, and playback observation events read-only in the receiver.
- Added duplicate protection so repeated `library.new` webhooks for the same queued/running `ratingKey` reuse the existing import hint job.
- Added safe job scope and summary readback for future single-rating-key processing.
- Updated the Plex webhook receiver smoke to prove enqueueing, duplicate reuse, read-only watched-state behavior, revocation, and secret-free evidence.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- Receiver tokens remain hash-only at rest and redacted from request/error logs.
- Webhook import jobs store only sanitized event metadata, `ratingKey`, and metadata readback path.
- This release does not fetch raw PMS payloads, expose Plex tokens, or write watched/rating state.
