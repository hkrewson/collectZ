## Summary
This release cleans up mobile image uploads so choosing an existing photo and taking a new photo are separate, intentional actions instead of one ambiguous upload control that could force mobile users into the camera.

## What changed
### Clear image source choices
- Art, Collectibles, and Events now use a first-tab cover-style native picker matching the media editor pattern.
- Event artifact and signature proof uploads keep explicit `Choose from Library` and `Take Photo` actions where there is no cover tile.
- Generic library/file upload inputs no longer use `capture="environment"`, so mobile browsers can offer the device photo library.
- Existing image actions hide once a replacement file is selected, keeping edit drawers focused on the pending replacement.

### Shared upload primitive
- Added shared image source and cover-picker controls so future image/proof upload surfaces can reuse the same source-selection behavior.
- Added source assertions and a responsive mobile browser regression for the Art drawer image controls.
