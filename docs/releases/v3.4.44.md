# v3.4.44

## Platform companion personal Sched ICS sync visibility

- Version: `3.4.44`
- Target milestone: `3.4.44 — Platform Companion Personal Sched ICS Sync Visibility`
- Date: 2026-05-01

### What changed

- Extended `GET /api/events/:id/companion/today` with `sync.personal_ics_visibility`.
- The new visibility object gives platform clients a UI-safe way to display personal Sched sync state, freshness, stale threshold, last sync times, item count, redacted errors, and manual refresh support.
- OpenAPI now documents the sync visibility payload inside `EventCompanionTodayResponse`.
- Added platform companion sync visibility guidance for Apple/native app planning.

### Why it matters

The platform app can show whether a user's personal Sched-backed plan is connected, fresh, stale, failed, or never synced without fetching Sched directly or exposing sensitive feed URLs.

### Security and privacy notes

- Raw personal Sched ICS URLs are not returned.
- URL-like values in sync error summaries are redacted before they reach the companion payload.
- Personal ICS data remains personal selected-schedule state, not the authoritative full event catalog.
- No background polling, push notification, offline mutation queue, realtime location, or full catalog discovery behavior was added.
