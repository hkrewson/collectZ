- Status: `completed`

## Summary

This patch moves mobile library search controls into the compact shell header row beside the active section icon, removing the extra search band that was still taking vertical space.

## What changed

- Added a mobile shell toolbar slot next to the current section icon.
- Let the shared page search toolbar portal mobile controls into that shell slot.
- Hid the page-local search toolbar on mobile when a shell-inline toolbar is active.
- Applied the pattern to Library, Art, Collectibles, and Events surfaces.
- Kept desktop page headers and toolbar behavior unchanged.

## Breaking changes

None.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. This patch changes frontend presentation, browser regression coverage, version metadata, and release feed data only.

## Known issues and follow-up

- Utility pages such as Capture Inbox, Wishlist, Import, and Integrations still use their own compact header/search patterns.
- Library filter controls still need a more deliberate single funnel/menu primitive if we want every mobile control to fit the shell row cleanly.
