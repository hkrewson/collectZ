## Summary

This patch adds manual relationship workflows for linking related records without duplicating them across libraries.

## What changed

- Added a scoped `object_relationships` persistence table and migration.
- Added authenticated relationship APIs for searching link targets, listing relationships for a record, saving links, and unlinking archived relationships.
- Added a compact `Related` drawer section for Library, Art, and Collectibles records.
- Relationship types include includes, part of, included with, companion to, purchased with, and event acquired with.
- Relationship readback shows the related record title, record family, optional subtitle context, and optional note from both sides.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- Adds migration `108`, creating `object_relationships`.
- Existing records are unchanged until a user links records together.
- Relationships are archived on unlink instead of hard-deleted.
