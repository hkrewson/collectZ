# collectZ v3.19.4

## Version and date

- Version: `3.19.4`
- Date: 2026-06-19

## Milestone target and status

- Roadmap slice: `3.19.4 — Durable Library Saved Views`
- Status: completed

## Summary

This patch makes Library saved views durable. Saved views now sync through authenticated, scope-aware backend endpoints when available, while keeping browser-local saved views as a fallback for older or unavailable backends.

## What changed

- Added a `saved_library_views` persistence table.
- Added scoped Library saved-view list, create, update, and delete endpoints.
- Updated the Library saved-view controls to load backend views first and fall back to local storage.
- Preserved the 3.19.3 saved-view snapshot shape for search, filters, sort, display mode, and library-specific context.
- Added unit/source coverage for the durable saved-view contract.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- Adds migration `110` for `saved_library_views`.
- Existing browser-local saved views are not automatically migrated to the backend in this slice.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.19.4`.
- Verify migrations apply through migration `110`.
- Verify `/api/health` reports version/frontend/backend/build `3.19.4`.
- Verify Help > Releases serves `3.19.4`.
- Run backend unit checks and frontend production build.
- Save, apply, update, and delete a Library saved view, then reload the browser and confirm the view remains available.

## Rollback guidance

- Revert to `3.19.3` if backend saved-view persistence causes unexpected Library behavior.
- If rolling back after migration `110`, leaving the unused table in place is safe; no app code before `3.19.4` reads it.

## Known issues and follow-up

- Browser-local saved views are not migrated into the backend automatically.
- Shared workspace views, smart rules, dashboard navigation, and smart collection badges remain backlog work.

## Security vulnerability triage

- High findings: none introduced or remediated in this patch.
- Owner: CollectZ maintainers.
- Target remediation milestone: none.
- Saved views are authenticated, owner-scoped, and constrained by the current workspace/library scope.
