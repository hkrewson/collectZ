# collectZ v3.19.0

## Version and date

- Version: `3.19.0`
- Date: 2026-06-15

## Milestone target and status

- Roadmap slice: `3.19.0 — Bulk Library Actions and Rate-Limit Safety`
- Status: completed

## Summary

This release makes library bulk delete behave like one intentional operation instead of hundreds of individual delete requests. The change protects normal cleanup workflows from tripping API rate limits while keeping scoped authorization and audit readback intact.

## What changed

- Added `POST /api/media/bulk-delete` for confirmed bulk media deletes.
- Preserved active workspace/library scoping and the same ownership rules as single-record delete.
- Added per-operation counts for requested, deleted, skipped, and failed records.
- Updated the Library bulk delete UI to call the bulk endpoint once instead of firing one request per selected row.
- Documented the new endpoint in OpenAPI.
- Added a targeted smoke test proving more than 200 selected media rows can be deleted through one request.
- Updated the frontend build toolchain patch versions to clear the current `esbuild` audit finding.
- Fixed an event autograph artifact regression found during browser verification where creating an autograph artifact could fail before returning its signature readback.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- No schema migration is required.
- Bulk delete still removes selected media rows; the behavior is now routed through one scoped backend operation.

## Known issues and follow-up

- This slice fixes media bulk delete first. Bulk archive/update and richer long-running progress UI remain future work only if real usage shows a need.
- The host shell's default Node/npm is too old for the project lockfiles, so the root local release gate still needs either the bundled runtime path or hosted CI to provide the final gate summary.

## Security vulnerability triage

- Owner: CollectZ maintainers.
- Target remediation milestone: none.
- This release preserves existing media delete authorization rules and reduces accidental rate-limit lockouts from legitimate user cleanup traffic.
