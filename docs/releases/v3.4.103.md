## Kavita Mark Read/Unread Contract

## Summary

This patch defines the Kavita read-state boundary before collectZ exposes any mark read/unread action. The contract documents Kavita's series, volume, and chapter mark endpoints, keeps runtime mark actions disabled, and identifies chapter-level mark-read as the only first candidate for a future implementation.

## What Changed

### Shipped

- Rechecked Kavita's upstream OpenAPI mark read/unread endpoint shapes.
- Documented why series-wide mark read/unread and volume-wide mark read are too broad for the first collectZ action.
- Extended the Kavita progress/read-state probe with disabled mark endpoint evidence.
- Added source assertions for the read-state contract and chapter-scoped payload shape.
- Kept runtime routes, drawer controls, import/sync read-state writes, background polling, KOReader sync, and shared provider abstractions out of scope.
