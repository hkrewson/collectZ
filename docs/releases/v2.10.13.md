## Summary
This patch release finishes the remaining `2.10.x` shared-language cleanup by moving the app away from display-font chrome and auth-page mini-marketing toward calmer, more product-native shared UI. Workspace headings, panel headings, and the authentication surface now follow the same quieter language instead of drifting into uppercase dashboard theatrics.

## What changed
### Shared heading normalization
- Reworked the shared heading primitives in `frontend/src/index.css`.
- Normalized workspace headings such as `Library`, `Import Media`, `Help`, `Integrations`, `Activity`, `All Spaces`, and `Members` so they read like normal product headings.
- Added a shared panel-heading treatment and applied it to surfaces such as `Space Controls` and `Member Details`.

### Authentication surface cleanup
- Removed the ornamental media-type pills and flattened the left-side auth layout into calmer product copy.
- Replaced the all-caps marketing-style hero and CTA posture with normal-case product language.
- Normalized the reset-password heading and submit-button wording to match the rest of the app.
- Hid the `Register` affordance by default unless registration is actually available through an invite-driven entry path in the current platform model.

### Shared UI language consolidation
- Centralized the heading shift in shared style layers rather than applying one-off fixes on each page.
- Kept view-specific composition local while reducing shared-language drift in the auth and admin surfaces.
- Updated focused Playwright auth helpers/specs to align with the calmer sign-in wording so the browser proof continues to validate the real UI contract.

## Breaking changes
- No backend or API-breaking changes are introduced in `2.10.13`.
- This release intentionally changes shared workspace, panel, and auth-surface presentation to be calmer and more truthful to the current product state.
