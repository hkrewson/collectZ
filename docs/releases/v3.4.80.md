# collectZ v3.4.80

## Event Self-Attendee Header Affordance Polish

This patch makes the self-attendee path easier to spot in the Event drawer. Instead of hiding the signed-in user flow inside the expanded People form, the drawer now surfaces `Add me to this event` directly in the People header and keeps the generic attendee form focused on adding other people.

### What changed

- Surfaces `Add me to this event` in the People section header so the self path is visible before the section is expanded.
- Keeps a lightweight reminder inside the People panel until the signed-in user adds themselves.
- Leaves the regular People form available for adding other attendees.
- Preserves the existing Event-local attendee ownership model and linked-user readback.
- Keeps external contact identities, cross-event identity, native companion mutation UX, and broader friend-graph work out of scope.
