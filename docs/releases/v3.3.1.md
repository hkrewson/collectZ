## Summary
This patch release builds on `3.3.0` library loans tracking by adding manual loan reminders, clearer reminder status in the drawer and Loans view, a more accurate due-soon count, and focused browser coverage for the loan-first drawer flow. The result is a tighter follow-up workflow for items that are due soon or already overdue without widening the release into automatic scheduling.

## What changed
### Manual reminder workflow
- Added reminder eligibility for due-soon and overdue active loans.
- Added a manual reminder-send action for eligible loans with borrower email.
- Added same-day resend guardrails so the same loan cannot send repeated reminders immediately.
- Added reminder activity logging for successful sends.

### Loans view and drawer clarity
- Added reminder state and send actions to the dedicated `Loans` view.
- Added reminder state and send actions to the media detail drawer for active loans.
- Shifted the drawer into a loan-first mode for currently loaned items with `Show Details` to reveal the rest of the record.
- Removed redundant summary/header copy and tightened the summary strip so the loans surface stays operational instead of dashboard-heavy.

### Scoped totals and browser proof
- Moved the `Due soon` count to a backend-scoped summary contract instead of a page-derived frontend count.
- Added focused Playwright coverage for opening a loaned game card, verifying the loan-first drawer, toggling details, and exercising the reminder action path in the live browser flow.
- Regenerated the in-app release feed for `Help > Releases`.

## Breaking changes
- None intended for operators or end users.
- `3.3.1` is a patch refinement on top of `3.3.0`, not a schema redesign.

## Environment/config changes
- No new required environment variables for `3.3.1`.
- Manual reminder send still depends on the existing email delivery configuration when operators want real outbound email.

## Migration and data impact
- No new schema migration is required for `3.3.1`.
- Existing `media_loans` reminder-ready fields are reused for manual reminder status and last-sent timing.
- Automatic reminder scheduling and deeper reminder-event history remain deferred to follow-up backlog work.
