- Milestone: `3.7.3 — Timeline Missing Target Snapshot Readback`
- Status: `implemented`

## Summary

This patch continues the human-readable timeline work by giving deleted or archived activity rows a readable snapshot drawer.

## What changed

- Added an Activity snapshot drawer for timeline rows whose original target is no longer openable.
- Added `View snapshot` actions for deleted or archived timeline entries that still have saved audit details.
- Shows the saved title, target, action, timestamp, user, type, provider, scope hints, and reason when those fields exist.
- Keeps the raw activity payload behind a technical disclosure.
- Added browser coverage for a deleted media row opening its saved timeline snapshot.

## Breaking changes

- None.

## Environment/config changes

- No new environment variables.

## Migration and data impact

- No database migration.
- Existing `activity_log.details` snapshots are reused.

## Security triage

- Snapshot readback stays inside the authenticated Activity surfaces and uses already-scoped Activity API responses.
- No new public routes or broader activity permissions were added.
