# v3.4.54

## Event schedule catalog metadata filters

- Version: `3.4.54`
- Target milestone: `3.4.54 — Event Schedule Catalog Metadata Filters`
- Date: 2026-05-02

### What changed

- Added Event drawer catalog filters for Track, Category, and Room / Location.
- Filter options are derived from the current event's catalog sessions, so manual and imported schedule rows use the same controls.
- Metadata filters work alongside the existing All / Now / Next / Later today, plan-state, conflict, and shared-attendance filters.
- The filtered session count and no-match empty state continue to update from the active filter set.

### Why it matters

Long convention catalogs are easier to scan when users can narrow the list to the rooms, tracks, or category tags they care about. This keeps the web catalog useful without adding new schema, sync, notification, or platform companion behavior.

### Security and privacy notes

- This patch is local catalog filtering only and does not add notification delivery, friend discovery, relay identity, device registration, realtime location, presence, or offline mutation queues.
- Filters use existing catalog metadata already visible in the Event drawer.
- Raw personal Sched ICS feed URLs remain hidden and are not used as catalog filter input.
