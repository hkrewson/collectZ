## Summary

This patch continues maintained-source CodeQL cleanup by reshaping platform registration so invite lookup, public self-registration, and failure metrics are separated into explicit server-policy decisions.

## What changed

- Normalized platform invite tokens before lookup and always ran the bounded invite lookup for platform registration.
- Kept invite-based onboarding, first-user bootstrap, SMTP-backed public self-registration, and homelab local-account registration behavior intact.
- Centralized registration failure recording so user-provided invite-token presence no longer guards the auth metrics action.
- Updated authentication documentation to reflect the current configurable platform self-registration behavior.

## Breaking changes

- None.

## Environment/config changes

- None.
- Platform public self-registration still depends on `self_registration_enabled=true` and configured SMTP delivery.

## Migration and data impact

- No schema, migration, or data impact.
- Existing invite tokens remain compatible through the hashed-token lookup and legacy plaintext-token fallback.
