## Summary

This patch adds an explicit export format choice to the backup and portability workflow.

## What changed

- Admin Settings now lets admins choose between `JSON archive` and `CSV archive` before downloading an export.
- `POST /api/admin/settings/portability/export` accepts `format: "json"` or `format: "csv"`.
- JSON remains the default `.json.gz` portability archive.
- CSV downloads a `.csv.zip` archive with manifest, restore guidance, uploads manifest, and one CSV per exported table.
- JSON and CSV export paths share the same backend redaction behavior.

- None.
