# v3.4.46

## Event schedule catalog foundation

- Version: `3.4.46`
- Target milestone: `3.4.46 — Event Schedule Catalog Foundation`
- Date: 2026-05-01

### What changed

- Added canonical Event schedule catalog sessions through `event_schedule_sessions`.
- Added event-scoped schedule catalog endpoints for listing, creating, updating, and archiving sessions.
- Catalog sessions store title, start/end time, location/room, description, track, categories, source metadata, and status.
- `GET /api/events/:id/companion/today` now includes catalog session counts, a `schedule_catalog` list, and a schedule catalog write endpoint reference.
- The platform companion offline packet now advertises schedule catalog support and includes catalog sessions when present.
- Personal Sched ICS sync remains personal selected-schedule data and does not become the canonical event catalog.

### Why it matters

This gives the web app and future platform companion app a clean model boundary: catalog sessions answer what exists at the event, while schedule plans answer what the user or workspace intends to attend.

### Security and privacy notes

- Raw personal Sched ICS URLs remain hidden from companion and offline packet responses.
- Provider import automation, Now / Next discovery, notifications, realtime location, presence, and offline mutation queues were not added.
- Catalog sessions inherit existing event scope checks and are archived rather than hard-deleted through the API.
