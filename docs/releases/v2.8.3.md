# v2.8.3

## Version and date
- Version: `2.8.3`
- Date: `2026-03-29`

## Milestone target and status
- Target milestone: `2.8.3 — Import Review Retirement and Debug Import Diagnostics`
- Status: `Completed`

## Summary
This release retires the old standalone Import Review workflow and finishes the product transition toward debug-oriented import diagnostics instead of a user-facing review queue. Ambiguous import rows are now surfaced through audit exports and external diagnostic logging, while the queue UI, queue endpoints, and queue table are removed from the live runtime.

## What changed
### Import Review retirement
- Removed the standalone `Library -> Import Review` UI surface and its dashboard routing.
- Legacy `library-import-review` routes now normalize back to the standard import surface instead of opening a separate workflow.
- Removed backend Import Review queue endpoints:
  - `GET /api/media/import-reviews/unresolved-count`
  - `GET /api/media/import-reviews`
  - `PATCH /api/media/import-reviews/:id`

### Debug diagnostics replacement
- Import audit exports now expose `diagnostic_flagged` instead of queue-oriented `review_queued`.
- Ambiguous import rows now emit `media.import.diagnostic.flagged` through the external log pipeline via `logActivity`.
- Import job summaries now preserve `diagnosticsFlagged` so operator/debug telemetry remains available without reviving the queue model.

### Schema and runtime cleanup
- Added migration `45` to drop the retired `import_match_reviews` table.
- Removed queue insertion and queue-transfer plumbing from import and library runtime code.
- Updated operator docs so import troubleshooting now points to audit exports and external debug logging rather than a review queue.

## Breaking changes
- The standalone Import Review page is no longer available.
- The old Import Review API endpoints are removed.
- Any tooling that expected the `import_match_reviews` table or queue-specific audit naming must move to the new audit/debug diagnostic path.

## Environment/config changes
- No new required environment variables.
- Existing external log export controls remain the supported debug path:
  - `FEATURE_FLAG_EXTERNAL_LOG_EXPORT_ENABLED`
  - any configured external logger destination already wired through the runtime

## Migration and data impact
- New schema migration: `45`
- The retired `import_match_reviews` table is dropped from the live schema.
- Historical queue data is not preserved as an active workflow surface after this release.
