# collectZ v3.8.8

## Version and date

- Version: `3.8.8`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.8 — Capture Selected Match Import`
- Status: `implemented`

## Summary

This release lets Capture Inbox turn a selected lookup match into a real library link or import, keeping the user in control of the final decision.

## What changed

- Adds authenticated `POST /api/capture-items/{id}/import-match`.
- Reuses the scanner barcode import pipeline so Capture Inbox and the iOS scanner share the same backend source of truth.
- Links captures to existing library items when the selected match is already in the catalog.
- Imports provider candidates into canonical media rows when the selected match is new.
- Marks converted captures with the linked media id and stores selected-match/import readback in capture review metadata.
- Adds compact `Link` and `Import` actions beside persisted capture lookup matches.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.
- Provider enrichment continues to use existing Barcode and Books integration settings.

## Migration and data impact

- No new database migration.
- Selected match and import result readback are stored in `capture_items.review_decision`.
- Converted media captures now set `capture_items.linked_media_id`.

## Security triage

- Capture match import remains authenticated, CSRF-protected for browser sessions, and scoped to the active workspace/library boundary.
- The route requires an explicit selected match; lookup results are not auto-imported.
- Existing library matches are verified in the current scope before linking.
