# collectZ v3.4.71

## Event Schedule Delivery Attempt Model Design

This patch defines the future delivery-attempt audit model for Event schedule notifications without creating attempt records or enabling provider-backed delivery.

### What's New

- Added `event-schedule-notification-delivery-attempt-model.v1` to the schedule notification delivery boundary.
- Documented the future relationship as one attempt per notification-recipient-provider when delivery providers are enabled later.
- Defined future attempt fields for provider, channel, status, attempted/completed timestamps, retry metadata, provider message id, and provider error details.
- Kept delivery attempts explicitly unsupported and uncreated in the current contract.
- Updated OpenAPI, smoke coverage, and platform companion docs for native/platform client planning.

### Notes

Runtime behavior is unchanged: schedule notifications remain Event-local records with recipient readback only. There are still no push, email, native device, global inbox, delivery queue, or delivery-attempt records in this release.
