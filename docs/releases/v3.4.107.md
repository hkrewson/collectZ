## Kavita Runtime Unread/Reset Implementation

## Summary

This patch enables an explicit Kavita reset-progress action for chapter-backed Kavita rows while keeping it separate from true chapter unread behavior.

## What Changed

### Shipped

- Added a scoped `POST /api/media/:id/kavita-reset-progress` endpoint for Kavita chapter-backed rows.
- Reset Kavita progress by writing `pageNum: 0` and `bookScrollId: null` through Kavita's progress endpoint.
- Added a media detail drawer `Reset Progress` action beside the existing read/progress controls.
- Preserved chapter mark-read behavior and continued to prohibit bulk unread endpoints.
- Updated Kavita smoke coverage to prove reset progress writes page `0`, returns secret-free readback, and avoids series/volume/multiple unread shortcuts.
