## Summary
This patch release turns the product’s space-separation promise into real workspace behavior. Settings, integrations, and activity now belong to the current workspace instead of quietly depending on global admin surfaces, while the platform layer is narrowed toward true platform concerns.

## What changed
### Workspace-scoped settings
- Reworked `Workspace > Settings` so it is no longer just editable record metadata.
- Kept the editable workspace name at the top of the tab.
- Removed `Slug` and `Description` from the normal workspace settings surface.
- Restored real workspace-owned controls for theme plus `Events Library` and `Collectibles Library`.
- Updated shared settings and feature-flag reads to honor the active workspace before falling back to legacy global rows where needed.

### Workspace-scoped integrations
- Added manager-facing workspace integrations inside `Workspace`.
- Kept supported free-token integrations owned by the active workspace.
- Preserved `Logs` and `Metrics` as explicit platform-only exceptions.
- Narrowed the platform integrations surface so it now represents platform observability concerns instead of pretending to own the workspace provider keys.

### Workspace-scoped activity and ownership boundaries
- Added the missing manual media activity logging needed for real workspace activity proof.
- Verified that workspace-local media actions stay visible inside `Workspace > Activity`.
- Verified that platform activity excludes those workspace-local media actions.
- Kept the resulting activity model aligned with workspace ownership rather than a mixed global feed.

### Workspace surface cleanup
- Continued the product-language shift away from `My Space` toward `Workspace`.
- Updated supporting admin/help/workspace surfaces so the ownership model reads more clearly.
- Simplified the top of `Workspace > Settings` so the tab keeps a single `Settings` label, an inline `Name` and `Save` row, and no duplicated explanatory chrome.

### Local auth/runtime stabilization during milestone work
- Stabilized local development auth/runtime while tracing the workspace settings regression.
- Prevented the in-progress workspace settings fetch issue from repeatedly exhausting the local shared API limiter during milestone work.
- Kept production limiter behavior unchanged while disabling rate limiting only for local development runtime debugging.

## Breaking changes
- No backend or API-breaking changes are introduced in `2.10.14`.
- This release intentionally redefines ownership boundaries so workspace settings/integrations/activity are no longer presented as global admin controls.

## Environment/config changes
- No new runtime environment variables are required for `2.10.14`.
- Existing deployments do not need env changes for this patch release.

## Migration and data impact
