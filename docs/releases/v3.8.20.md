# collectZ v3.8.20

## Version and date

- Version: `3.8.20`
- Date: `2026-05-20`

## Milestone target and status

- Milestone: `3.8.20 — Public Homelab Environment Reference`
- Status: `implemented`

## Summary

This release adds a public homelab environment reference so `env.example` can stay minimal while operators still have a safe place to understand optional deployment settings.

## What changed

- Added `docs/wiki/48-Public-Homelab-Environment-Reference.md` for operator-safe public homelab settings.
- Updated README and deployment docs to point public users at the new homelab-specific environment reference.
- Marked the broad environment-variable page as a maintainer/runtime reference rather than the first public setup path.
- Removed stale Portainer/deploy guidance that implied version metadata variables changed the public prebuilt image tag.
- Removed public deploy guidance that recommended putting audit verbosity in the default Portainer environment.
- Added the new public reference to public export surface validation.
- Kept the Playwright browser-regression bypass out of public compose/docs while restoring the CI-only compose override that passes the per-run bypass token to the backend test stack.
- Kept CI-only debug-gated API docs available to browser regression, and seeded Events on during Playwright setup so event-social browser specs do not inherit a disabled Event feature flag.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.
- `env.example` remains the minimum startup surface.
- Optional public settings are now documented separately from private, CI-only, and deep maintainer runtime controls.

- No database migration.
- No data changes.
