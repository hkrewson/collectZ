# collectZ v3.18.0

## Version and date

- Version: `3.18.0`
- Date: 2026-06-11

## Milestone target and status

- Roadmap slice: `3.18.0 — Public Repository Sanitization Strategy`
- Status: completed

## Summary

This release defines the public repository sanitization strategy for collectZ: a private source-of-truth repository can generate a clean public mirror without exposing maintainer workflow, release evidence, roadmap/backlog detail, local overrides, or private git history.

## What changed

- Added `public-export.manifest.json` as the first machine-readable public mirror contract.
- Extended `npm run validate:public-export` so it requires the manifest, clean public history policy, required denylists, and local-gate publication expectation.
- Added maintainer documentation for the private-source/public-mirror operating model, public mirror contents, publication workflow, public checks stance, and accidental leak response.
- Promoted the public repository sanitization backlog item into the `3.18.0` roadmap slice.
- Kept actual mirror publication and clean export commit automation as a follow-up.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.18.0`.
- Verify `/api/health` reports version/frontend/backend/build `3.18.0`.
- Verify Help > Releases serves `3.18.0`.
- Run `npm run validate:public-export`.
- Run `npm run release:local-gate` before push.

## Rollback guidance

- Revert to `3.17.0` if the public export manifest or validator contract needs to be backed out.
- No schema rollback is required.

## Known issues and follow-up

- The public mirror export builder is not implemented in this release.
- Future work should create a temporary public tree, validate it, and commit it to a separate public target without private git history.
- Public GitHub checks remain useful, but the private/local gate should continue to run before any public export.
