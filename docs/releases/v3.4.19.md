## Summary
This release makes signature proof evidence more descriptive. Art and media signature proofs can now store an evidence type, short label, and notes per proof image, while the shared multi-proof signature model and legacy primary proof projection remain compatible.

## What changed
### Proof metadata model
- Added per-proof metadata fields to `signature_proofs` for evidence type, label, and notes.
- Signature proof readback now includes metadata on each proof in the `proofs` array.
- Existing proof rows remain compatible and simply read back with empty metadata until users add it.

### Art and media API
- Art and media proof upload endpoints accept metadata alongside the image file.
- Added per-proof metadata update endpoints for Art and media signatures.
- Existing proof upload/remove compatibility routes remain available.
- Primary proof compatibility fields continue to project through `proof_path`, Art `signature_proof_path`, and media `signed_proof_path`.

### Drawer experience
- The shared Art/media signature drawer now displays proof labels/types where available.
- Users can add evidence type, label, and notes while uploading a proof.
- Users can edit metadata for an existing proof without replacing or removing the image.

### Regression coverage
- Added source-level coverage for migration, init parity, service, route, OpenAPI, and drawer wiring.
- Expanded focused Art/media browser regressions to prove metadata upload and metadata update for signature proofs.
- Re-ran init parity and migration rehearsal through migration `83`.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing proof images and compatibility proof paths remain readable.

## Environment/config changes
- No new required operator configuration in `3.4.19`.

## Migration and data impact
- Adds nullable metadata columns to `signature_proofs`.
- Existing rows are not destructively rewritten.
- Older clients can continue reading proof paths without understanding the new metadata fields.
