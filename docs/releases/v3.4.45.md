# v3.4.45

## Platform companion offline event packet

- Version: `3.4.45`
- Target milestone: `3.4.45 — Platform Companion Offline Event Packet`
- Date: 2026-05-01

### What changed

- Extended `GET /api/events/:id/companion/today` with `offline_packet`.
- The packet gives platform clients read-only cache guidance for poor-connectivity event use: packet version, cache key, TTL/stale guidance, retry policy, counts, freshness, limitations, planned sessions, and key locations.
- Key locations are derived from Event, meetup, and schedule-plan location/vendor/booth/location-note data.
- The packet includes personal planned sessions from current Event schedule plans, including Sched ICS-derived plans.
- OpenAPI now documents the offline packet inside `EventCompanionTodayResponse`.

### Why it matters

The platform app can cache a useful active-event snapshot for convention-floor use while still treating the backend as authoritative after reconnect.

### Security and privacy notes

- The packet is read-only and does not support offline mutation queues.
- Raw personal Sched ICS URLs are not returned.
- Realtime location, presence, broad social discovery, push notifications, and background polling were not added.
- Full schedule catalog support remains explicitly unavailable until the separate catalog milestone exists.
