# collectZ v3.23.21

## Version and date

- Version: `3.23.21`
- Date: `2026-07-22`

## Milestone target and status

- Roadmap slice: `3.23.21 - iOS Home Screen App Scope`
- Status: `completed`

## Summary

This patch gives CollectZ an explicit web-app identity so an iOS Home Screen installation owns the full site instead of treating the page used during installation as its return destination.

## What changed

- Added a web app manifest with a root app identity and route scope.
- Set `/dashboard` as the stable Home Screen launch URL while keeping every CollectZ route inside the installed app.
- Added standalone display, theme, and Apple Home Screen metadata.
- Reused the existing CollectZ mark and color palette for standard, maskable, and Apple touch icons.
- Added browser coverage for the manifest contract, MIME type, metadata links, and install icon responses.
- Updated Axios to `1.18.1` in backend and frontend and refreshed backend's transitive `body-parser` to `2.3.0` after newly published advisories surfaced during the release audit.

## Security vulnerability triage

- High findings: newly published Axios findings were resolved by updating backend and frontend to `1.18.1`; the accompanying low-severity `body-parser` finding was resolved at `2.3.0`.
- Owner: collectZ maintainer.
- Target remediation milestone: completed in `3.23.21`.

## Blocked or CI-only gates

- The local observability evidence refresh was blocked when Docker did not complete the Graylog/MongoDB/OpenSearch image pull within a ten-minute bounded attempt after the local image cleanup. The standard local release gate therefore finished with 11 passing gates and one failed preflight aggregation gate; the manifest, runtime, browser, unit, version, release-feed, and dependency checks themselves passed.
- Hosted CI still needs to confirm `compose-smoke`, `rbac-regression`, hosted full `browser-regression`, `runtime-smoke` core/control-plane, `dependency-scan`, `secret-scan`, and `image-security-and-sbom` before push-ready release promotion.

## Rollback guidance

- Roll back frontend images to `3.23.20` if the manifest or Home Screen metadata causes an installation regression. Existing browser use is unaffected.

## Known issues and follow-up

- iOS caches installed web-app metadata. Remove the old Home Screen entry and install CollectZ again after this version is deployed.
