## Summary
This release establishes the TV season/watch-state foundation: dedicated season storage, season-level APIs, UI watch/completion controls, and Plex season ingestion into the new model.

## What changed
- Added TV season source-of-truth table (`media_seasons`) and migrated TV season ownership away from `media_variants`.
- Added/updated TV season APIs:
  - `GET /api/media/:id/tv-seasons`
  - `PUT /api/media/:id/tv-seasons`
  - `PATCH /api/media/:id/tv-seasons/:seasonNumber`
- Updated TV drawer UI to render seasons from `media_seasons` and support per-season state updates.
- Added season + series completion indicators in library UI.
- Extended Plex import to upsert season rows from show-season discovery.
- Added TV season mutation audit events:
  - `media.tv_seasons.update`
  - `media.tv_season.update`

## Breaking changes
- None. Existing TV season update contract remains compatible.

## Environment/config changes
- No new required environment variables.
- Existing Plex/TMDB integration settings remain unchanged.

## Migration and data impact
- Added migration `v34`: `media_seasons` table, indexes, constraints, and backfill from legacy TV season variant data.
- Existing movie/file edition behavior remains in `media_variants`.
- TV season data now persists in `media_seasons` and is non-destructively updated on Plex reruns.

## Deployment and verification
1. Rebuild and start:
   - `docker compose --env-file .env up -d --build backend frontend`
2. Verify health/version:
   - `curl -sS http://localhost:3000/api/health`
   - Expect `"version":"2.3.0"`.
3. Verify TV seasons:
   - Run Plex import twice.
   - Confirm job summary includes season counters and no season duplication across reruns.
4. Verify audit events in Activity:
   - `media.tv_seasons.update`
   - `media.tv_season.update`

## Rollback guidance
- Roll back to previous images/tag if needed.
- If rollback crosses migration boundary, keep DB forward and roll app images back only after validating compatibility.
