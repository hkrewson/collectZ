## Summary
This patch release hardens merge durability boundaries. It adds a Docker-backed proof that preserved merge identities only resolve inside the currently active library scope, so later re-syncs cannot silently update or recreate rows in the wrong library when the same external/provider identifier exists elsewhere.

## What Changed
### Scope-isolated merge re-sync proof
- Added a dedicated runtime smoke for:
  - one manual merge in library A,
  - an overlapping `provider_item_id` row in library B,
  - a later re-sync in library A,
  - and proof that only library A is touched until library B runs its own scoped re-sync.
- Proved the live CSV sync path updates the correct row with:
  - `created: 0`
  - `updated: 1`
  - `matchedBy: "provider_item_id"`
  - and `isolationPreserved: true` on both platform and homelab stacks.

### Smoke harness hardening
- Refactored the new scope-isolation smoke to use:
  - a normal user session for real library switching,
  - and the direct manual merge helper for setup,
- so the proof exercises actual route-level scope rules instead of getting tangled in admin support-session restrictions.
