# collectZ v3.12.15

## Version and date

- Version: `3.12.15`
- Date: 2026-06-05

## Milestone target and status

- Roadmap slice: `3.12.15 — Vite Compatibility Shim Removal`
- Status: completed

## Summary

This patch removes the remaining CRA-era frontend compatibility shims now that the Vite-first frontend path has baked through multiple releases without React or browser-regression instability.

## Changes

- Frontend env reads now use `VITE_*` only through the shared Vite env helper.
- Removed legacy `REACT_APP_*` fallback reads and the `process.env.REACT_APP_*` Vite define bridge.
- Removed legacy `REACT_APP_*` Docker build args/env plumbing from the frontend image build.
- Removed transitional `dev:vite`, `build:vite`, and `preview:vite` script aliases; the normal `start`, `build`, and `preview` scripts are the Vite commands.
- Updated docs and source-contract tests so `VITE_*` is the maintained frontend env contract.

## Breaking changes

- Frontend `REACT_APP_*` build-time names are no longer supported configuration.
- Local, Docker, and CI frontend configuration should use `VITE_API_URL`, `VITE_APP_VERSION`, `VITE_DEBUG`, and `VITE_CSRF_COOKIE_NAME`.

## Upgrade notes

- Removed frontend `REACT_APP_*` compatibility support.
- No backend runtime env variables changed.

## Migration and data impact

- No schema migration is required.
- Existing collection data, sessions, provider settings, and uploads are unchanged.
- Any private/local frontend build script still passing `REACT_APP_*` values should be updated to the equivalent `VITE_*` names.
