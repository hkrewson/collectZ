# collectZ v3.8.16

## Version and date

- Version: `3.8.16`
- Date: `2026-05-20`

## Milestone target and status

- Milestone: `3.8.16 — Capture Scanner Queue Shortcuts`
- Status: `implemented`

## Summary

This release makes scanner-app captures easier to find and continue from inside Capture Inbox.

## What changed

- Added backend source filtering for Capture Inbox so scanner-origin captures can be requested directly.
- Added source counts for scanner and web captures in Capture Inbox responses.
- Added a compact scanner shortcut on Capture Inbox when scanner captures are waiting.
- Added a source selector with Scanner app and Web capture filters.
- Marks scanner-origin rows as Scanner app in capture row details.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- Scanner classification uses existing `source_context.source` and `source_context.client_source` capture metadata.

## Security triage

- Capture Inbox remains authenticated and scoped.
- Scanner source filtering does not expose captures outside the active workspace/library scope.

## Security vulnerability triage

- High findings: none identified in local checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.16` closeout if any high findings appear.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.8.16`.
- Verify Help > Releases includes `3.8.16`.
- Create or receive a capture with scanner-shaped `client_source` or `source_context.source`.
- Confirm Capture Inbox shows the scanner shortcut and `Scanner app` row readback.
- Confirm `source_filter=scanner` returns only scanner-origin captures within scope.

## Rollback guidance

- Revert the Capture Inbox source filter UI and backend source-filter query additions, then restore `3.8.15` version metadata and release feed.
- No schema rollback is required.

## Known issues and follow-up

- Scanner detection is intentionally lightweight and string-based against existing source metadata.
- Durable unified review routing, visible offline queue management, and cover/art recognition remain future Capture workflow work.
