## Summary

This release adds a Comic-Con field-test workflow inside Events so an event can become a mobile hunt list, quick haul capture surface, and post-con cleanup queue. The slice is additive and keeps collectZ on the `3.x` line because it builds on existing Events, Wishlist, Art, Collectibles, purchased-item, and companion snapshot contracts.

## What changed

- Added nullable `booth` support to wishlist items.
- Added `event_id` filtering to the Wishlist list API.
- Added `GET /api/events/:id/field-kit` for event-scoped hunt items, tracked purchases, cleanup gaps, and a secret-free companion snapshot summary.
- Added a Comic-Con field kit panel to the Event drawer.
- Refined the Event drawer into Overview, Hunt, Haul, Plans, Purchases, and Cleanup tabs so the Comic-Con field workflow is available without turning the drawer into one long scroll.
- Flattened the Plans and Purchases tabs into divider-led rows and inline readbacks so event workflow sections match the rest of the drawer surface.
- Added Con Hunt actions for event-linked wishlist items: mark ordered, acquired, or dismissed; edit vendor, booth, and notes; launch quick haul capture.
- Added Quick Haul Capture for event-linked Art and Collectibles, including purchased-item linking and optional wishlist acquisition.
- Added Post-Con Cleanup readback for linked purchases missing vendor, booth, price, or photo.
- Hardened the backend container startup path so reused local upload volumes are writable by the non-root Node process.
