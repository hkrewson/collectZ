## Summary

This patch continues the reusable collectible traits line by adding the backend persistence contract that future grading, COA/provenance, bundle, and edition editors can build on.

## What changed

- Added migration `107` for `collectible_trait_records`.
- Added scoped persisted trait APIs under `/api/collectible-traits/{ownerType}/{ownerId}`.
- Supported trait owners: media, Art, and Collectibles.
- Persisted traits now merge into existing `collectible_traits` readback before derived compatibility traits with the same key.
- Expanded the OpenAPI trait contract for stored traits and future families such as graded, bundle, and provenance.
- Added personal access token scope mapping for the trait API as collection data.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- Adds the `collectible_trait_records` table and indexes.
- Existing media, Art, Collectibles, signature records, and derived trait readback remain compatible.
- No existing data is backfilled or rewritten in this slice.
