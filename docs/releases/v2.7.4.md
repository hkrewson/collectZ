# v2.7.4

## Version and date
- Version: `2.7.4`
- Date: `2026-03-20`

## Milestone target and status
- Target milestone: `2.7.x maintenance lane`, `2.7.4` low-risk dependency and CI action maintenance
- Status: `Completed`

## Summary
This patch release batches the low-risk open maintenance PRs that fit the active `2.7.x` line. It updates safe runtime dependencies on the backend and frontend, refreshes the PostgreSQL client library on the backend, and advances the GitHub Actions workflow references used for Docker publishing, artifact handling, and build metadata without pulling any major framework or toolchain upgrades into the release.

## What changed
### Dependency maintenance
- Updated backend `axios` to `1.13.6`.
- Updated frontend `axios` to `1.13.6`.
- Updated backend `pg` to `8.20.0`.

### CI workflow maintenance
- Updated `docker/setup-buildx-action` from `v3` to `v4`.
- Updated `docker/metadata-action` from `v5` to `v6`.
- Updated `docker/build-push-action` from `v6` to `v7`.
- Updated `actions/upload-artifact` from `v6` to `v7`.
- Updated `actions/download-artifact` from `v7` to `v8`.

### Docs and release evidence
- Added a new release note for `2.7.4`.
- Refreshed local dependency-audit and SBOM artifacts.
- Recorded a new local preflight result in `preflight-go-no-go.md`.

## Deployment and verification
1. Rebuild and start:
   - `docker compose --env-file .env up -d --build backend frontend`
2. Verify reported app version is `2.7.4`.
3. Verify core release regressions:
   - `npm run test:integration-smoke`
   - `npm run test:rbac-regression`
   - `npm run test:admin-space-control`
   - `npm run test:tenancy-platform-boundary`
4. Verify migration safety:
   - `npm run test:init-parity`
   - `npm run test:migration-rehearsal`
5. Verify release security evidence:
   - `gitleaks` returns no findings
   - `npm audit --omit=dev` is clean for backend and frontend production trees
   - Trivy critical image scans pass for backend and frontend
   - CycloneDX SBOM artifacts are generated for backend and frontend

## Rollback guidance
- Roll back application images to `2.7.3` and restore from backup if release rollback is required.
- No new database rollback procedure is introduced by this patch.

## Security vulnerability triage
Production/runtime posture for this patch line:
- backend production dependency audit is clean at `low=0`, `moderate=0`, `high=0`, `critical=0`
- frontend production dependency audit is clean at `low=0`, `moderate=0`, `high=0`, `critical=0`

Development-only follow-up:
- Remaining frontend development advisories are still concentrated in the legacy `react-scripts` toolchain and are intentionally deferred out of this patch line.
- The dedicated remediation path for that remaining CRA-coupled surface is tracked in the planned `3.0.0` frontend build modernization milestone.

## Known issues and follow-up
- The remaining open major-version Dependabot PRs for `express`, `zod`, `react`, `react-dom`, and `tailwindcss` are intentionally out of scope for `2.7.x`.
- This release updates workflow action references but does not broaden workflow permissions or change the publish/deploy model.
