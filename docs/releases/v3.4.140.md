# collectZ v3.4.140

## Summary

This patch turns the Plex reconciliation proof into a conservative one-way sync job.

## Changes

- Adds a queued Plex reconciliation sync endpoint.
- Automatically creates missing Plex titles.
- Automatically updates collectZ rows only when the match is backed by strong TMDB identity.
- Keeps already-linked Plex rows as no-op readback.
- Stores conflict rows in the sync job result for review instead of applying them.
- Updates the Plex integrations panel with a `Sync Plex Library` action and sync issue readback.
- Keeps Plex writeback separate and manual.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- This change does not expose Plex tokens, raw Plex URLs, media file paths, or Plex writeback controls.
