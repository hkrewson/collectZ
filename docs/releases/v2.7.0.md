## Summary
This release activates first-class tenancy across collectZ. Spaces and space memberships are now real platform concepts, tenant invites are space-scoped, active space/library selection is explicit, and the server-admin control plane is separated from routine tenant membership and content access.

## What changed
### Tenancy model and backend policy
- Added first-class `spaces` and `space_memberships` activation with real `libraries.space_id` backfill.
- Existing installs now land in a valid shared/default space with valid `active_space_id` and `active_library_id` pointers for all users.
- Added explicit auth scope bootstrap and selection APIs so session, PAT, and service-account flows resolve active scope consistently.
- Added space-scoped membership APIs, invite APIs, and explicit transfer-to-new-space behavior for owned libraries.
- Removed the legacy global `/api/admin/invites` path in favor of tenant space-scoped invite flows.

### Tenant UI and control surfaces
- Added active space/library switching in the app shell.
- Added tenant `Space Control` UI for space details, memberships, and scoped invites.
- Hardened tenant role behavior so space admins cannot self-promote to owner and member/viewer users cannot use tenant management mutations.
- Fixed stale tenancy UI refresh issues across space switches, including admin invitation/state panels.

### Server-admin control plane
- Added a dedicated `Admin > Spaces` platform control plane for:
  - space creation
  - owner assignment/recovery
  - archive/delete for empty spaces
- Kept global server-admin authority distinct from tenant membership.
- Kept `Admin > Members` focused on platform user administration rather than tenant invite history or tenant content metrics.
- Kept `Admin > Activity` explicitly in the platform control plane.

### Regression and upgrade verification
- Added self-bootstrapping admin smoke coverage for:
  - platform space control
  - platform-vs-tenant boundary isolation
- Updated RBAC regression to use space-scoped invite flows and explicit auth-scope selection.
- Verified the upgraded live dataset has:
  - no users missing `active_space_id`
  - no users missing `active_library_id`
  - no libraries missing `space_id`

## Breaking changes
- Legacy global admin invite creation via `/api/admin/invites` is removed.
- Tenant onboarding now belongs to space-scoped invite flows.
- Global admins no longer implicitly gain tenant-space roster, invite, or content access without explicit membership.

## Environment/config changes
- No new required environment variables for this release.
- Existing tenancy/runtime behavior depends on the current schema migrations being applied at startup.

## Migration and data impact
- Schema migrations through `v43` are required.
- Existing installs are backfilled into valid spaces and memberships.
- Existing libraries are attached to concrete `space_id` values.
- Existing users receive valid active space/library pointers.
- Cross-space moves only transfer libraries owned by the moved user through explicit transfer flows.
