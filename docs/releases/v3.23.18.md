## Summary

This patch fixes scanner barcode lookups for console games that generic UPC providers describe with retail listing titles. Game-shaped UPC results such as Nintendo 3DS listings are now typed as games instead of falling through to movie matching.

## What changed

- Added barcode-provider game inference for common console/platform terms including Nintendo 3DS, Switch, Wii, PlayStation, Xbox, Game Boy, and PC game listings.
- Cleaned game retail suffixes such as platform tags, `Pre-Played`, and color/listing noise before provider enrichment search.
- Preserved platform metadata on scanner lookup matches so imported game records can retain the detected console.
- Treated Capture Inbox `other` media type as unknown during scanner lookup instead of forcing a concrete media type before provider normalization can infer one.
- Made the scanner API smoke deterministic without requiring TMDB configuration for the existing UPC title-variant check.

## Breaking changes

- None. Existing scanner clients can continue to omit `mediaType` for unknown barcode captures.

## Environment/config changes

- None.

## Migration and data impact

- No database migration is required.
- Existing movie-typed game records are not automatically rewritten; future scanner lookups/imports receive the corrected game type.
