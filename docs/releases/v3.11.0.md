# collectZ v3.11.0

## Version and date

- Version: `3.11.0`
- Date: 2026-06-03
- Milestone: `3.11.0 — Collection Health Rules and Identifier Expectations`

## Milestone target and status

- Status: `completed`

## Summary

This minor release starts the collection-health rule model. Review now separates true identifier gaps from sparse metadata, so digital/manual records are not pushed toward impossible identifiers while physical retail items can still ask for UPC/EAN when that is useful.

## What changed

- Added a shared backend review classifier for `missing_identifier` and `sparse_metadata`.
- Updated Dashboard Review with a separate Metadata lane and row-level clues.
- Updated Library review filters so Dashboard can open either missing identifiers or sparse metadata directly.
- Changed audio/game identifier expectations so digital rows can ask for artist, year, platform, or format metadata instead of UPC.
- Kept physical audio/game rows eligible for UPC/EAN recommendations.
- Updated OpenAPI and tests for the expanded review fields.

## Breaking changes

None. Existing clients can ignore the new optional review fields.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. Counts may change because the missing-identifier rule is now more precise and sparse metadata is counted separately.

## Rollback guidance

Rollback is safe because this release changes classification and API/UI readback only. If review counts look noisy, revert to the previous release while keeping collection data unchanged.

## Known issues and follow-up

- Art and collectibles remain outside the generic identifier rules until a later domain-specific health model defines the right expectations.
- This release does not add automatic enrichment or repair automation.
- Sparse metadata is intentionally a first-pass rule set and may need tuning as real records expose better category-specific expectations.
