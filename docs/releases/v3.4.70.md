# collectZ v3.4.70

## Event Schedule Notification Delivery Provider Prep

This patch prepares the Event schedule notification contract for future provider-backed delivery without enabling push, email, native device delivery, delivery attempts, or global inbox behavior.

### What's New

- Extended the Event schedule notification delivery boundary with provider-prep metadata.
- Kept `event_local` as the only active provider.
- Added disabled provider descriptors for future `push`, `email`, and `platform_device` channels.
- Explicitly reports that delivery attempts, delivery attempt readback, device registration endpoints, and external provider configuration are not available yet.
- Updated OpenAPI, smoke coverage, and platform companion docs so native clients can hide unavailable delivery controls safely.

### Notes

This is contract prep only. Schedule notifications still create local Event records and recipient readback rows; they do not send push notifications, email, native device messages, or realtime/global inbox updates.
