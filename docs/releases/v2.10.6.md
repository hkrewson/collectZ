## Summary
This patch release packages the post-`2.10.0` UI and workflow follow-through that was completed across the library and add/edit drawers. Import now reads like the rest of the product, bulk selection is calmer and more usable, the shell has less decorative chrome, and Add/Edit search now behaves like a true universal lookup flow instead of a barcode-only side path. The result is a quieter, more predictable product surface without changing the underlying multi-format ownership model that shipped in `2.10.0`.

## What changed
### Library workflow UX polish
- Flattened `Import Media` so it uses the shared horizontal tab language and calmer section spacing instead of one-off vertical/card-heavy treatment.
- Removed barcode as a first-class Import tab so Import is again focused on file and connected-service ingestion flows.
- Simplified library selection styling:
  - smaller rounded-square selectors,
  - flatter bulk-selection row,
  - quieter selected-state treatment.
- Added shift-select range behavior for library bulk selection in the current visible order.

### Shell and card cleanup
- Removed the global shell radial gradient wash so the app relies more on typography and layout than decorative background chrome.
- Flattened the active support-session banner into normal product status chrome with plainer copy.
- Reduced poster-card hover theatrics by removing zoom/fade/showcase-style hover treatment.
- Simplified poster-card action treatment back toward browse-first cards instead of hover-reveal action clutter.

### Add/Edit universal search
- Replaced the field-attached barcode-coded lookup affordance with a dedicated universal `Search` panel at the top of the Core step in Add/Edit drawers.
- Title-only searches now use the media-type default title provider path.
- Identifier-only searches still use the identifier/barcode provider path.
- When both title and identifier are present, both lookups now run and results are combined into one picker with visible provenance.
- When the chosen result is identifier-led, the drawer now performs the follow-up provider-default title enrichment before applying metadata.
- Search results now display in an in-place overlay instead of pushing the core details form down the drawer.

### Browser and gate follow-through
- Updated Playwright auth/bootstrap helpers so browser regression aligns with the current platform auth model.
- Refreshed browser specs for the current import, support-session, admin-shell, and drawer-search contracts.
- Repaired stale source-gate assertions that still expected the retired barcode-import tab and older support-session banner copy.

## Breaking changes
- No API-breaking backend contract changes are introduced in `2.10.6`.
- This release intentionally changes some UI behavior:
  - Import no longer presents Barcode as a top-level import tab.
  - Add/Edit search is now a dedicated drawer section rather than a control attached to one field row.
  - Poster/library cards remain browse-first instead of surfacing inline hover action clutter.
