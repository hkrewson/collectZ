# collectZ v3.4.91

## Kavita Reader and Progress Contract Discovery

This patch documents the next Kavita boundary before collectZ attempts embedded reading, page streaming, or progress sync. The approved behavior remains native link-out to Kavita's own reader; collectZ should not iframe Kavita, proxy reader pages, or write Kavita reading progress until a later opt-in contract exists.

### What changed

- Added `docs/wiki/42-Kavita-Reader-Progress-Contract.md` with the reader/progress recommendation and security boundary.
- Reviewed Kavita's upstream OpenAPI reader/progress surface and called out reader content, navigation, progress read, progress write, bookmark, and KOReader sync paths.
- Clarified that some Kavita reader `GET` endpoints can have side effects such as caching reader images.
- Linked the new contract from the Kavita setup guide.
- Added unit/source assertions that keep the no-iframe, no-reader-proxy, and no-progress-writeback boundary visible in the repo.
