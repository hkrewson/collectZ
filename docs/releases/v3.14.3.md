## Summary

This patch adds owner-level proof and provenance editing for collectible-like records.

## What changed

- Added a compact Proof editor to Library, Art, and Collectibles detail drawers.
- Proof details are stored through the reusable trait API as the `provenance` family.
- Supported fields include proof type, issuer/source, certificate number, acquired-from/source context, evidence date, reference, and notes.
- Saved proof details appear through existing collectible trait readback and trait pills.
- Reference readback shows that a reference exists without echoing raw local path-style values in the drawer.

## Breaking changes

- None.

## Migration and data impact

- No new migration.
- Uses the `collectible_trait_records` table introduced in `3.14.1`.
- Existing records are unchanged until a user adds proof/provenance details.

## Known issues and follow-up

- This slice does not add file upload, document storage, or external authenticity verification.
- Proof/provenance is owner-level in this patch; attaching evidence directly to a specific signature, grading record, or bundle remains follow-up work.
- Proof/provenance does not affect valuation in this patch.
