## Summary
This patch release removes the remaining eyebrow-style uppercase language from the shared form label primitive so forms throughout the app inherit calmer, more product-native labeling by default. Instead of continuing to reintroduce tracked uppercase microheadings through the shared form layer, drawers, admin screens, auth flows, and supporting forms now share a quieter label treatment from the root.

## What changed
### Shared form label normalization
- Reworked the shared `.label` primitive in the global frontend style layer.
- Removed the global uppercase + tracking-heavy eyebrow treatment from shared form labels.
- Shifted the shared label style to calmer normal-case form typography while preserving spacing and readable hierarchy.

### Form-layer consistency
- Applied the shared typography shift at the root so existing forms that already rely on `.label` inherit the calmer treatment without one-off screen patches.
- Preserved field structure, spacing rhythm, and accessibility-oriented label usage.

## Breaking changes
- No backend or API-breaking changes are introduced in `2.10.12`.
