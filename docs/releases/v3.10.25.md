## Summary

This patch upgrades the frontend React runtime pair to React 19 and verifies the existing app shell, library surfaces, and browser regression coverage still run against the upgraded renderer.

## What changed

- Updated frontend `react` from `^18.2.0` to `^19.2.6`.
- Updated frontend `react-dom` from `^18.2.0` to `^19.2.6`.
- Kept the existing Vite entrypoint using `react-dom/client` and `createRoot`.

## Breaking changes

None expected.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. This patch changes frontend runtime dependencies only.

## Rollback guidance

Rollback to `3.10.24` is safe because this patch has no schema, API route, or data-format changes. If browser runtime behavior regresses, restore the React 18 dependency pair and rebuild the frontend image.

## Known issues and follow-up

- React 19 is a major frontend runtime upgrade, so UI/runtime behavior should be watched in full CI browser regression and normal mobile/desktop use.
- Tailwind 4 and Express 5 remain separate PR-phase upgrades.
