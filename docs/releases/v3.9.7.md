## Milestone target and status

Status: Complete.

This patch turns Apple/iTunes target price hits from passive readback into a small review surface with direct follow-up actions.

## Changes

- Added compact `Mark ordered` and `Dismiss` actions to Apple/iTunes target price hits in Library > Wishlist.
- Reused the existing scoped Wishlist update path instead of adding a provider-specific status endpoint.
- Removed acted-on target hits from the compact hit list after a successful status update.
- Focused the default target-hit readback on actionable wishlist states (`wanted`, `watching`, and `preordered`) so rows marked `ordered` do not come back as fresh hits on refresh.
- Kept the Store link available beside each target hit.

## Security and privacy

- No new provider calls, secrets, alerts, notifications, or background jobs are introduced.
- Actions use the existing authenticated and scoped Wishlist update route.
- Store links remain user-initiated outbound navigation.
