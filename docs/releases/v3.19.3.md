# collectZ v3.19.3

## Version and date

- Version: `3.19.3`
- Date: 2026-06-19

## Milestone target and status

- Roadmap slice: `3.19.3 — Local Saved Library Views`
- Status: completed

## Summary

This patch adds browser-local saved views to the Library. Users can save the current search, filters, sort, card/list mode, and library-specific context, then reopen that view later from the same browser.

## What changed

- Added local saved Library views scoped by media type.
- Added compact Library header controls to save/update, apply, and delete saved views.
- Saved search text, quick filters, review filters, sort direction, card/list mode, collection mode, and comic view context where relevant.
- Added unit/source coverage for the saved-view contract.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- No schema migration is required.
- Saved views are stored in the browser's local storage, not in the backend database.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.19.3`.
- Verify `/api/health` reports version/frontend/backend/build `3.19.3`.
- Verify Help > Releases serves `3.19.3`.
- Run backend unit checks and frontend production build.
- Save, apply, update, and delete a Library saved view in a browser.

## Rollback guidance

- Revert to `3.19.2` if the local saved-view controls cause unexpected Library header behavior.
- No schema rollback is required.

## Known issues and follow-up

- Saved views are browser-local in this slice.
- Durable user/workspace-scoped saved views, shared views, smart rules, and dashboard navigation remain backlog work.
