## Kavita Metadata Writeback Contract and API Probe

## Summary

This patch confirms Kavita exposes metadata mutation endpoints that collectZ can safely target in a later opt-in writeback implementation, while keeping the current Kavita integration read-only at runtime.

## What Changed

### Shipped

- Added a Kavita metadata writeback contract documenting the first writable series/chapter endpoints, the conservative field allowlist, and the required preview/audit/opt-in safety model.
- Added pure backend payload builders for future series and chapter metadata writeback.
- Added a fake-server probe proving the planned payload shape for `POST /api/Series/metadata` and `POST /api/Chapter/update` without calling a real Kavita mutation endpoint.
- Updated Kavita setup documentation so operators can see writeback is feasible but still disabled until a later implementation milestone.
