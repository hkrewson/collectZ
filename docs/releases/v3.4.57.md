# v3.4.57

## Event schedule selected-recipient change preview

- Version: `3.4.57`
- Target milestone: `3.4.57 — Event Schedule Selected-Recipient Change Preview`
- Date: 2026-05-02

## Why This Exists

The event schedule now has catalog sessions, personal plans, conflicts, shared attendance, and catalog-to-personal matching. This patch adds the first notification-adjacent contract: a safe preview of who could be affected by a schedule change before any actual notification or platform push work exists.

## What's New

- Added `POST /api/events/:id/schedule-change-preview` for schedule-plan or catalog-session change previews.
- The preview returns the schedule subject, requested status/visibility, scoped people/groups, conflicts, and a simple message template.
- The contract is explicitly preview-only: no push delivery, no device registration, no message persistence, and no broadcast behavior.
- Event schedule rows now include a `Preview share` action that reads back affected recipients and conflicts without sending anything.
- Event social smoke coverage now verifies the preview contract, recipient counts, and preview-only delivery boundary.

## Compatibility Notes

- This does not introduce real notifications, native device tokens, delivery jobs, or persisted message drafts.
- Private schedule changes return no recipients.
- Shared visibility previews use existing Event attendees/groups only; broader friend identity remains a future milestone.
