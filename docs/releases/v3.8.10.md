# collectZ v3.8.10

## Version and date

- Version: `3.8.10`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.10 — Capture Barcode Camera ISBN Recognition`
- Status: `implemented`

## Summary

This release makes the Capture Inbox barcode camera flow more useful for books by looking for an ISBN in the captured barcode photo instead of stopping at the retail UPC.

## What changed

- After the camera barcode action reads a code, Capture Inbox now runs the existing client-side identifier OCR against the same image.
- If an ISBN is found, the capture form uses the ISBN, marks the object type as Book, and sets the symbology to `ISBN-13`.
- If no ISBN is found, the flow falls back to the detected UPC/barcode as before.
- Preserves OCR text readback when available so the capture row can still be reviewed.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- This changes only pre-save Capture Inbox form population for camera barcode scans.

## Security triage

- The barcode image is still processed in the browser and is not uploaded by the scan action.
- Capture Inbox remains authenticated and scoped to the active workspace/library.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.10` closeout if any high findings appear.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.8.10`.
- Verify Help > Releases includes `3.8.10`.
- For a book with both a retail UPC and nearby ISBN text, use Capture Inbox > New capture > Barcode / ISBN scan and confirm the form prefers the ISBN when recognized.

## Rollback guidance

- Revert the Capture Inbox ISBN-recognition scan changes and restore `3.8.9` version metadata and release feed if barcode scanning regresses.
- No schema rollback is required.

## Known issues and follow-up

- ISBN recognition depends on photo quality and readable printed ISBN text near the barcode. Poor focus or glare may still fall back to UPC.
