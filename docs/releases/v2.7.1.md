# v2.7.1

## Version and date
- Version: `2.7.1`
- Date: `2026-03-19`

## Milestone target and status
- Target milestone: `2.7.1 — Security Maintenance and Dependency PR Triage`
- Status: `Completed`

## Summary
This patch release tightens `2.7.0` release readiness without reopening the tenancy feature scope. It focuses on CI-blocking security and regression maintenance: frontend image hardening, RBAC regression stability, dev-tooling dependency cleanup, and release-gate verification.

## What changed
### Release-gate and regression fixes
- Fixed the RBAC regression around ownership transfer and cross-tenant scope expectations.
- Updated the RBAC smoke harness so clean-stack CI runs isolate tenant users correctly before asserting cross-tenant denial behavior.
- Reconfirmed the admin-space and platform-boundary tenancy smokes against the current patch line.

### Image and runtime hardening
- Patched the frontend runtime/build image to upgrade Alpine `expat/libexpat` packages so the Trivy critical image gate no longer fails on the known `libexpat` finding.
- Verified the local compose runtime after recreating the detached frontend service and restoring backend upstream resolution on the compose network.

### Dependency maintenance
- Added low-risk dependency overrides for dev/build-only vulnerability cleanup:
  - frontend `minimatch` transitive chains to `3.1.4`
  - frontend `filelist -> minimatch` to `10.2.3`
  - frontend `jsonpath -> underscore` to `1.13.8`
  - frontend `flat-cache -> flatted` to `3.4.0`
  - backend `nodemon -> minimatch` to `10.2.3`
- Left the remaining frontend `nth-check` issue explicitly deferred because it sits behind the legacy CRA/SVGO toolchain and is not part of the shipped production runtime.

## Security vulnerability triage
Current `2.7.1` maintenance intent:
- clear current CI-blocking runtime/image issues
- reduce low-risk dev-tooling vulnerability noise where safe
- defer larger build-toolchain modernization

Production/runtime posture for this patch line:
- backend production dependency audit remains below the blocking threshold with `critical=0`, but still reports `low=1` and `high=1`
- the remaining backend `high` is transitive via `@aws-sdk/xml-builder -> fast-xml-parser`
- frontend production dependency audit is clean at `low=0`, `moderate=0`, `high=0`, `critical=0`
- frontend production runtime remains static nginx content, not a node-based runtime
- the remaining `nth-check` advisory is confined to the legacy frontend build toolchain and remains a follow-up item rather than a `2.7.1` runtime blocker

Residual follow-up:
- broader dependency PR triage and any CRA/toolchain modernization continue in later `2.7.x` maintenance slices

## Known issues and follow-up
- The remaining `nth-check` path is still inside `react-scripts -> svgo@1.3.2` and should be handled in a later frontend tooling modernization slice rather than forced through this patch release.
- `2.7.x` maintenance is intentionally broader than a single patch; additional dependency PR batches may still land in `2.7.2+`.
