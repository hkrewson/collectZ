- Status: implementation complete locally; CI release gates must pass before tag/publish.

## Summary

This release adds item-local numbered print metadata to Art so limited prints can show their edition signal while browsing and in detail views. It also simplifies Art poster cards so signed numbered prints read like `#150/200 Signed Print` under the title instead of layering badges over and below the artwork.

## What changed

### Backend

- Added positive-integer Art metadata fields for print number and print run.
- Added validation and API read/write support for partial or complete numbered print values.
- Kept numbered print metadata item-local without adding reusable artist records, external print registries, or valuation enrichment.

### Frontend

- Added Art editor inputs for print number and print run.
- Added list, mobile detail, and desktop detail readback for numbered prints.
- Simplified Art poster cards by removing the top `#ID`, `Art`, and `Signed` badges plus the lower metadata pills.
- Shows concise Art poster subtitle copy such as `#150/200 Signed Print`.
- Preserved non-numbered artwork entry with no new required fields.

### Database

- Added `art_items.print_number` and `art_items.print_run` as nullable positive-integer metadata fields.
- Updated bootstrap schema parity so fresh installs include the numbered print columns.

### Docs/Ops

- Updated roadmap closeout and Help > Releases feed for `v3.4.109`.
- Reduced the local Graylog evidence stack's default journal size so release observability checks can run in constrained Docker Desktop storage.

## Breaking changes

None.

- Roll back to the previous published backend/frontend images and restore from the pre-release database backup if needed.

## Known issues and follow-up

- Numbered print metadata is intentionally item-local.
- Reusable artist records, valuation enrichment, external print registries, certificate verification, and broad edition-series modeling remain separate backlog/roadmap work.
- Local `gitleaks` and `trivy` binaries were not installed during local closeout, so `secret-scan` and `image-security-and-sbom` remain CI-only release gates.
