# collectZ v3.8.5

## Version and date

- Version: `3.8.5`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.5 — Capture Replay Conflict Review UI`
- Status: `implemented`

## Summary

This release turns stored Capture Inbox replay conflicts into a visible review workflow.

## What changed

- Shows unresolved replay conflicts directly on Capture Inbox rows.
- Lists the conflicting fields with current and replayed values.
- Adds explicit `Keep current` and `Use replayed values` actions.
- Adds a scoped backend endpoint to resolve the latest replay conflict without editing raw review metadata.
- Records replay conflict resolution activity for timeline/readback.

## Breaking changes

- None.

## Environment/config changes

- No new environment variables.

## Migration and data impact

- No new database migration.
- Conflict resolution updates existing `capture_items.review_decision` and `source_context` metadata.

## Security triage

- Replay conflict resolution remains authenticated and scoped to the current workspace/library access boundary.
- Applying replayed values is explicit and only uses fields already captured in stored replay metadata.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.5` closeout if any high findings appear.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.8.5`.
- Verify Help > Releases includes `3.8.5`.
- Verify a conflicting capture replay appears in Capture Inbox with current and replayed values.
- Verify `Keep current` resolves the conflict without changing the existing capture values.
- Verify `Use replayed values` applies the stored incoming replay values.

## Rollback guidance

- Revert the replay conflict review UI/API changes and restore `3.8.4` version metadata and release feed if conflict resolution causes regressions.
- No schema rollback is required for this slice.

## Known issues and follow-up

- This is still Capture Inbox-local review, not the broader unified review queue.
- Later work can group scanner conflicts, sparse imports, enrichment choices, and missing-cover review in one surface.
