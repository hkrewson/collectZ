## Summary

This patch continues CodeQL baseline remediation by documenting and testing the reviewed outbound URL boundaries for Kavita connector requests and personal Sched ICS feed fetching.

## What changed

- Added a shared trusted connector URL normalizer for admin-configured HTTP(S) integrations that rejects credentials and fragments while preserving LAN Kavita support.
- Updated Kavita URL normalization to use the shared trusted connector boundary.
- Added in-source CodeQL request-forgery suppressions at the reviewed Kavita and ICS request sinks, backed by unit coverage for the helper contracts and suppression wiring.
- Added the JavaScript `AlertSuppression.ql` query to the CodeQL workflow so in-source suppressions are represented in SARIF/code scanning.

## Security vulnerability triage

- The Kavita findings are reviewed as intentional admin-configured connector requests. The base URL path is normalized through `normalizeTrustedConnectorHttpUrl`, which only accepts HTTP(S), strips fragments, rejects credentials, and preserves the documented homelab/LAN connector use case.
- The Sched ICS finding is reviewed as guarded by `assertPublicHttpUrl`, which rejects credentials, localhost, private IP hosts, and DNS resolutions to private or link-local addresses unless the operator explicitly enables the documented private-feed escape hatch.
- Local CodeQL with `AlertSuppression.ql` marks the four request-forgery results as in-source suppressed with `0` active `js/request-forgery` findings.
