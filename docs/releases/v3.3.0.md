## Summary
This minor release adds a new library loans workflow so borrowed items can be tracked from one management surface instead of disappearing into title-level notes. The release introduces scoped loan records, item-attached loan actions, a dedicated `Loans` view, and stronger current-loan visibility for active, overdue, and returned items across both platform and homelab stacks.

## What Changed
### Loans workflow foundation
- Added a dedicated `media_loans` record shape with one-active-loan-per-media protection.
- Added scoped create, edit, list, and return APIs for loan lifecycle management.
- Added OpenAPI coverage for the new loans endpoints.
- Normalized loan date fields in API responses to a cleaner date-only contract.

### Library loans management view
- Added a dedicated `Loans` navigation surface in the library shell.
- Added active, overdue, returned, and all-loans filters.
- Added edit and return actions directly from the loans view.
- Added counts and urgency cues so the screen reads as the place to manage what is currently out.

### Item-attached loan actions
- Added a loan creation form and active-loan summary inside the media detail drawer.
- Added recent loan history visibility for individual titles.
- Kept loan entry attached to the title while still making the overall workflow browsable from one place.

### Verification and release evidence
- Added a Docker-backed loans workflow smoke proving create, update, return, and history behavior.
- Verified init parity and migration rehearsal after adding the loans schema.
- Regenerated the in-app release feed for `Help > Releases`.

## Breaking changes
- None intended for operators or end users.
- `3.3.0` is a new feature release on top of `3.2.10`, not a patch refinement.

## Environment/config changes
- No new required environment variables for `3.3.0`.

## Migration and data impact
- `3.3.0` adds the `media_loans` table and related indexes.
- Existing media data is not rewritten.
- No backfill is required for existing libraries before using the new loans workflow.
