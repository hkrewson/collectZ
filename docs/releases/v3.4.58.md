# v3.4.58

## Event schedule selected-recipient notification draft/send contract

- Version: `3.4.58`
- Date: `2026-05-02`
- Target milestone: `3.4.58 — Event Schedule Selected-Recipient Notification Draft/Send Contract`

## Summary

- Adds an Event-local selected-recipient notification record for schedule changes.
- Lets schedule change previews become either a saved draft or a local sent notice.
- Keeps notification delivery intentionally local: no push, email, device registration, realtime presence, or broad broadcast behavior.

## Changes

- Added `event_schedule_notifications` for draft/sent schedule notification records.
- Added `GET /api/events/:id/schedule-notifications` and `POST /api/events/:id/schedule-notifications`.
- Validates selected attendee/group recipients against the scoped Event-local preview recipients.
- Stores the subject snapshot, requested change, selected recipient snapshot, conflicts, message text, and delivery limitations.
- Updated the Event schedule drawer with `Save draft` and `Send local notice` actions after preview.

## Compatibility

- No push/email/device delivery is introduced in this release.
- Existing schedule plans, catalog sessions, and personal Sched ICS behavior are unchanged.
- New table is additive and covered by migration/init parity.
