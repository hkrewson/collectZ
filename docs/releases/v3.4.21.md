## Summary
This release finishes the post-Vite cleanup pass by making `VITE_*` the preferred frontend build/runtime configuration path while keeping legacy `REACT_APP_*` names as compatibility shims only.

## What changed
### Vite-first frontend configuration
- Frontend code now reads `VITE_API_URL`, `VITE_APP_VERSION`, `VITE_DEBUG`, and `VITE_CSRF_COOKIE_NAME` first.
- Legacy `REACT_APP_*` names remain available as build-time fallback values for older local/private workflows.
- The frontend Dockerfile now presents Vite build arguments first and keeps CRA-style names only as compatibility inputs.

### CI and docs cleanup
- CI frontend image builds now pass `VITE_*` build args instead of `REACT_APP_*` args.
- Local CI compose overrides now generate Vite-first frontend build args.
- Public docs and env examples now show `VITE_*` as the preferred frontend configuration names.
- Removed the stale tracked CRA-era `frontend/public/index.html`; Vite uses the root `frontend/index.html` and outputs `dist`.

## Breaking changes
- None intended. Existing `REACT_APP_*` build-time values still work as compatibility shims.

## Environment/config changes
- Prefer these names for new frontend configuration:
  - `VITE_API_URL`
  - `VITE_APP_VERSION`
  - `VITE_DEBUG`
  - `VITE_CSRF_COOKIE_NAME`
- `REACT_APP_*` names should be treated as legacy compatibility only.
