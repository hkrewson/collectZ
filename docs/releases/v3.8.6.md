# collectZ v3.8.6

## Version and date

- Version: `3.8.6`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.6 — Capture Backend Image OCR`
- Status: `implemented`

## Summary

This release lets Capture Inbox run backend-owned OCR against uploaded photo captures.

## What changed

- Adds authenticated `POST /api/capture-items/{id}/ocr-image`.
- Reads locally stored capture uploads from backend storage before calling the configured Vision/OCR provider.
- Stores extracted OCR text and reviewable ISBN, UPC/EAN, and ASIN candidates on the capture row.
- Adds a compact `Read image text` action for photo captures in Capture Inbox.
- Surfaces optional Vision/OCR provider settings through integration config readback and public compose env pass-through.
- Records image OCR extraction activity for timeline/readback.

## Breaking changes

- None.

## Environment/config changes

- Public compose now passes through optional `VISION_PRESET`, `VISION_PROVIDER`, `VISION_API_URL`, `VISION_API_KEY`, and `VISION_API_KEY_HEADER`.
- If unset, the backend defaults to OCR.space-compatible request settings, but an API key may still be required by the provider.

## Migration and data impact

- No new database migration.
- Image OCR updates existing `capture_items.ocr_text`, `review_decision`, and `source_context` metadata.

## Security triage

- Image OCR remains authenticated and scoped to the current workspace/library access boundary.
- This slice only reads backend-local `/uploads/` image paths for OCR, avoiding arbitrary remote image fetches.
- OCR candidates remain review-only and are not auto-imported or silently applied.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.6` closeout if any high findings appear.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.8.6`.
- Verify Help > Releases includes `3.8.6`.
- Verify Capture Inbox photo rows show `Read image text`.
- Verify backend image OCR returns extracted text/candidates when a Vision/OCR provider is configured.
- Verify missing or unsupported image storage returns a clear non-destructive error.

## Rollback guidance

- Revert the image OCR route/service/UI changes and restore `3.8.5` version metadata and release feed if backend OCR causes regressions.
- No schema rollback is required for this slice.

## Known issues and follow-up

- This is text OCR only, not cover/art recognition.
- Backend image OCR currently reads local upload-provider paths; broader object-storage/private media retrieval can be a later storage contract slice.
