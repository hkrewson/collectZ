# collectZ v3.8.19

## Version and date

- Version: `3.8.19`
- Date: `2026-05-20`

## Milestone target and status

- Milestone: `3.8.19 — Public Compose Latest Image Simplification`
- Status: `implemented`

## Summary

This release simplifies the public homelab Docker Compose file so it defaults directly to the GHCR `latest` images and only exposes the runtime environment values needed for normal homelab startup.

## What changed

- Changed public backend image to `ghcr.io/hkrewson/collectz-backend:latest`.
- Changed public frontend image to `ghcr.io/hkrewson/collectz-frontend:latest`.
- Removed public `IMAGE_REGISTRY`, `IMAGE_NAMESPACE`, and `IMAGE_TAG` indirection from the generated compose surface.
- Removed provider/bootstrap tuning variables from the default public compose environment block.
- Kept required database, session, encryption, CORS, reverse-proxy, secure-cookie, upload, and port settings in the public compose.
- Tightened public export validation so image-source indirection variables do not return to public deploy docs, compose, or env examples.

## Breaking changes

- Public compose no longer supports changing backend/frontend image source or tag through `IMAGE_REGISTRY`, `IMAGE_NAMESPACE`, or `IMAGE_TAG`.
- Users who want an exact pinned image should edit the image line directly or use a private override file.

## Environment/config changes

- No new required environment variables.
- Public compose now defaults to GHCR `latest` images directly.
- Public compose keeps `NODE_ENV=production` and `SESSION_COOKIE_SECURE=true` by default.

## Migration and data impact

- No database migration.
- No data changes.

## Security triage

- Public compose exposes fewer runtime knobs by default.
- Provider secrets and private/test controls are no longer advertised through the public compose environment block.
- Public export validation now rejects image-source indirection variables in public deploy surfaces.

## Known issues and follow-up

- Exact image pinning is still available by editing the compose image lines or layering a private compose override, but it is no longer a first-class public `.env` default.
- A separate public environment variable reference can document optional advanced runtime settings without expanding the default compose.
