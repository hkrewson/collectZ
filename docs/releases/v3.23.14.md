## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `AuthPage.jsx` was the top remaining frontend lint pocket after `3.23.13`. The slice clears Auth Page import, accessibility, and documented route-sync findings without changing auth behavior.

## What changed

- Removed the stale default React import from the auth page.
- Associated auth form labels with their controls.
- Documented auth route/query synchronization effects for invite, email, and token URL values.
- Documented route feedback clearing and automatic email-verification consumption as intentional auth side effects.
- Reduced the full frontend quality baseline from `31` errors / `25` warnings to `25` errors / `24` warnings.
