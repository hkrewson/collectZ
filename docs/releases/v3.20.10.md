## Summary

This patch adds a Cairn-owned platform analytics setting for Rybbit and loads the configured tracking script globally in the collectZ browser app. Platform operators can save the Rybbit script URL, site id, and optional `data-*` attributes from Platform Settings without storing arbitrary inline JavaScript.

## What changed

- Added Cairn analytics tracking settings under `/api/admin/settings/analytics`.
- Added public Cairn browser config at `/api/platform/analytics`.
- Added Platform Settings controls for enabling analytics, setting the script URL, site id, and optional data attributes.
- Added a collectZ root-level analytics loader so enabled tracking is applied across auth, dashboard, and standalone app routes.
- Updated Cairn OpenAPI docs and helper coverage for the new settings contract.

## Breaking changes

- None.

## Environment/config changes

- No new environment variable is required.
- The frontend uses the existing `VITE_PLATFORM_API_URL` bridge to read Cairn analytics config.

## Migration and data impact

- Cairn stores the setting in the existing `platform_settings` key/value table.
- No collectZ Core schema migration is required.
- Existing installs remain disabled until analytics tracking is explicitly enabled and configured.

## Rollback guidance

- Disable analytics tracking from Platform Settings before rolling back if the external script causes browser issues.
- Roll back Cairn and collectZ images to `3.20.9` if the Platform Settings or analytics loader behavior regresses.
- The stored Cairn setting can remain; older collectZ frontend builds will not read `/api/platform/analytics`.

## Known issues and follow-up

- Rybbit must be configured for the deployed collectZ domain before production events are accepted.
- Localhost tracking may require Rybbit's development API-key workflow if domain validation blocks local events.
