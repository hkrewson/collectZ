## Summary
This minor release adds the first optional market valuation foundation for collectZ. Workspace-scoped integrations can now carry valuation provider configuration, media records can store normalized read-only value bands, the library detail drawer can display and manually refresh valuation data, and imports can queue non-blocking follow-on valuation jobs for newly created media. The milestone keeps pricing optional, read-only, and safely decoupled from core catalog CRUD and import flows.

## What changed
### Workspace-scoped valuation providers
- Added optional workspace-level valuation provider settings for:
  - `PriceCharting`
  - `eBay Browse`
- Provider execution remains optional and scoped to the active workspace so one workspace’s pricing traffic does not hold the whole platform hostage.
- Provider dry-run tests remain fixture-backed by default for safe local/operator validation.

### Normalized valuation fields
- Added read-only valuation fields on media:
  - `estimated_value_low`
  - `estimated_value_mid`
  - `estimated_value_high`
  - `valuation_currency`
  - `valuation_source`
  - `valuation_last_updated`
- Media detail views now render valuation information when present and degrade gracefully when unavailable.

### Manual valuation refresh
- Added an item-level valuation refresh flow from the media detail drawer.
- Refresh execution supports:
  - queued sync jobs
  - optional synchronous execution
  - fixture mode outside production for safe proofing without live provider calls
- The current execution model is identifier-first with title fallback when needed.

### Import-time valuation refresh
- CSV, Plex, and comics import paths can now queue a non-blocking valuation follow-on batch for newly created media.
- Pricing failures do not block the underlying import.
- Import responses and job summaries now expose valuation follow-on metadata so operators can tell whether a valuation batch was queued.

### eBay Browse execution path
- Added real eBay Browse execution logic and normalized listing-derived value bands.
- `PriceCharting` remains supported as an optional provider, including a serialized rate-limit policy and provider abstraction, but live credential-level validation was intentionally deferred in this release closeout.
- Automated proof stayed fixture-backed to avoid accidental live-provider abuse during local and CI-style testing.

## Breaking changes
- No API-breaking changes are introduced in `2.11.0`.
- The milestone adds optional fields and routes without changing existing media CRUD/import contracts in a breaking way.

## Environment/config changes
- Optional valuation provider bootstrap/config now includes:
  - `PRICECHARTING_API_URL`
  - `PRICECHARTING_API_KEY`
  - `PRICECHARTING_RATE_LIMIT_MS`
  - `EBAY_BROWSE_API_URL`
  - `EBAY_BROWSE_CLIENT_ID`
  - `EBAY_BROWSE_CLIENT_SECRET`
  - `EBAY_BROWSE_MARKETPLACE_ID`
- Valuation execution remains optional and should not be treated as required for normal app operation.

## Migration and data impact
- `2.11.0` adds normalized valuation columns to `media`.
- `2.11.0` also extends integration storage with optional valuation-provider settings for PriceCharting and eBay Browse.
- Existing media/import behavior remains valid after migration even if no valuation provider is configured.
