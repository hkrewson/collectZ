# v2.0.0-alpha.7

- Date: 2026-02-25
- Version: `2.0.0-alpha.7`
- Scope: deployment hardening for public alpha environments

## Added

- Production startup now fails fast when:
  - `DB_PASSWORD` is missing.
  - `SESSION_SECRET` is weak (placeholder or shorter than 32 chars).
  - `INTEGRATION_ENCRYPTION_KEY` is weak (placeholder or shorter than 32 chars).

## Changed

- Version metadata bumped to `2.0.0-alpha.7` across app metadata, compose defaults, and package versions.
- `env.example` updated to `APP_VERSION=2.0.0-alpha.7` with explicit secret-generation guidance.
- Deployment docs updated to reflect production secret-strength expectations and new startup validation behavior.

## Verification

- `node --check backend/server.js`
- `node scripts/sync-app-meta.js`
- `docker compose --env-file .env config`
- `docker compose --env-file .env up -d --build backend`
