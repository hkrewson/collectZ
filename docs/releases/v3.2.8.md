## Summary
This patch release removes the comic-library full-fetch exception from the main browsing paths. Comic issues, comic series summaries, and selected-series issue browsing now all run through server-backed pagination and stable comic ordering contracts built from existing series, issue, and volume metadata instead of browser-only full-list sorting.

## What Changed
### Server-backed comic issue ordering
- Added a backend `comic_issue` ordering contract built from existing comic metadata with title fallback parsing.
- Switched the default comic `issues` tab to use normal paged API results instead of forcing a one-shot full fetch into browser memory.

### Server-backed comic series summaries
- Added a grouped `/api/media/comic-series` API for paginated comic series summaries.
- Switched the comic `series` tab to use grouped server summaries with stable counts, poster selection, and paging.

### Server-backed selected-series issue browsing
- Added `/api/media/comic-series/issues` for paginated issue browsing inside a selected comic series.
- Switched the `series_issues` tab off the last comic full-fetch path so selected-series issue browsing now uses server results with stable in-series ordering.
- Added Docker-backed proof coverage for all three comic query contracts across both platform and homelab stacks.

## Breaking changes
- None intended for operators or end users.
- `3.2.8` is a comic query and pagination normalization patch on top of `3.2.7`.
