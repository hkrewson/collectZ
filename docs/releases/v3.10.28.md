## Summary

This patch turns the dependency PR cleanup into a documented security-maintenance process and adds GitHub CodeQL source scanning for JavaScript/TypeScript. It keeps CodeQL advisory at introduction while preserving the existing release gates for dependency audit, secret scan, image security, SBOM, runtime smoke, RBAC, browser regression, and edition boundaries.

## What changed

- Added `.github/workflows/codeql.yml` for JavaScript/TypeScript CodeQL analysis on push, pull request, weekly schedule, and manual dispatch.
- Added `docs/wiki/49-Dependency-PR-and-CI-Security-Coverage.md` with open PR dispositions and a CI security coverage map.
- Updated CI/release docs to include CodeQL as an advisory source-scanning layer.
- Moved the selected dependency/security backlog item out of the backlog and into the numbered roadmap.
- Documented that `@zxing/library@0.23.0` remains deferred until its browser peer relationship is intentionally resolved.

## Breaking changes

None.

None for runtime operators. The repository now includes a CodeQL workflow that uploads code scanning results to GitHub Security for supported repository contexts.

## Rollback guidance

Rollback is safe because this patch has no schema, API contract, or runtime behavior changes. If CodeQL creates unacceptable noise, remove `.github/workflows/codeql.yml` or disable the workflow while keeping the dependency PR disposition document.

## Known issues and follow-up
