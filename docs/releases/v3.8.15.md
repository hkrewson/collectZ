# collectZ v3.8.15

## Version and date

- Version: `3.8.15`
- Date: `2026-05-20`

## Milestone target and status

- Milestone: `3.8.15 — Capture Batch Session Summary`
- Status: `implemented`

## Summary

This release makes batch scanning easier to wrap up by keeping a compact summary on Capture Inbox after a batch is stopped.

## What changed

- Added a completed batch summary to Capture Inbox.
- Shows added, linked, needs-choice, no-match, and failed counts after batch mode stops.
- Keeps a short recent outcome list so the user can see what just happened.
- Adds direct review actions for choices and no-match captures.
- Keeps the existing batch scan flow and exact ISBN auto-add behavior intact.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- Batch summary state is browser-local and does not add new stored fields.

## Security triage

- Capture lookup and import remain authenticated and scoped.
- The summary only displays capture titles/statuses already visible in Capture Inbox.

## Security vulnerability triage

- High findings: none identified in local checks for this slice.
- Owner: collectZ maintainers.
- Target remediation milestone: before `3.8.15` closeout if any high findings appear.

## Known issues and follow-up

- The summary is intentionally session-local and disappears when dismissed or when a new batch starts.
- Scanner-app queue shortcuts, true visible offline queue management, cover/art recognition, and broader unified review queue routing remain future Capture workflow work.
