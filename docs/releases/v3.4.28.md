## Summary
This release documents the movie lookup rate-limit boundary and removes one avoidable TMDB search in the add/edit drawer when an identifier result already includes provider metadata.

## What changed
### TMDB lookup pressure
- Confirmed the app-side external-provider limiter is mounted on `/api/media/search-tmdb` and `/api/media/lookup-upc`.
- Confirmed that app-side limiter behavior is controlled by `RATE_LIMIT_EXTERNAL_API_MAX`.
- Confirmed upstream TMDB request failures continue to be wrapped with provider status/path details through the TMDB service.

### Search selection optimization
- Identifier-led lookup results that already contain TMDB, book, or media-type provider enrichment now apply directly instead of performing a redundant follow-up title search.
- Identifier-led results without enrichment still keep the existing follow-up title enrichment behavior.
- Added source/browser assertions so the dual-search drawer path keeps one title search when selecting an already enriched identifier result.

## Breaking changes
- None intended. Runtime API contracts and stored media fields are unchanged.

## Environment/config changes
