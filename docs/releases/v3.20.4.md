- Status: completed

## Summary

This patch makes Plex activity entries easier to read. Workspace activity now recognizes Plex imports, webhook queue events, webhook processor runs, and webhook import outcomes instead of showing generic provider activity.

## What changed

- Added explicit Plex import summaries to the activity feed.
- Added readable Plex webhook queued and processed activity summaries.
- Added readable Plex webhook auto-processor activity summaries.
- Added Plex job, rating key, and section readback to activity snapshots.
- Kept existing Plex import, sync, webhook, readback, and writeback behavior unchanged.

- Rebuild frontend and backend containers with `APP_VERSION=3.20.4`.

  - `node --check backend/scripts/unit-tests.js`

## Rollback guidance

- Roll back to the previous frontend image if activity feed rendering regresses.
- Backend rollback is not required for this UI-only readability slice, though the release version should stay consistent across images.

## Known issues and follow-up

- This patch improves activity rendering for existing Plex events; it does not add new scheduler or writeback controls.
- Scheduled pull sync controls, explicit writeback controls, and reconciliation review filters remain follow-up work in the Plex true-sync plan.
