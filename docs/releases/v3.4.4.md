## Summary
This release finishes the product-layer Art split by giving Art its own frontend surface, reducing Collectibles-specific compatibility assumptions, and documenting the shared Event purchase relationship. Art remains bridge-safe for migrated rows and existing IDs, but users now interact with Art as its own object domain rather than as a Collectibles subtype.

## What changed
### Art UI divergence
- Replaced the Art wrapper around `CollectiblesView` with a native Art component.
- Added Art-owned list, card, row, detail drawer, and editor behavior.
- Kept event-aware vendor and booth display in Art without carrying Collectibles category controls into the Art editor.
- Kept Art search, event filtering, exclusive filtering, sort, image, notes, price, artist, and series flows intact.

### Collectibles decoupling
- Removed Art mode assumptions from the Collectibles component.
- Kept Collectibles focused on non-Art classifications such as `Card`, `Funko`, `Comic Panels`, and related collectible categories.
- Added backend safeguards so `/api/collectibles` does not expose or accept Art records.
- Split Art request validation away from Collectibles request validation.

### Art taxonomy cleanup
- Removed Art category taxonomy documentation from OpenAPI.
- Retired the Art categories route from the public API contract.
- Added a clear unavailable response for `/api/art/categories` so it does not behave like a malformed Art detail request.

### Event purchase contract
- Documented the long-term relationship between Art, Collectibles, and Events in the roadmap.
- Confirmed Events continue to read tracked purchases through the shared `event_purchased_items` relationship.
- Kept `event_purchased_items.item_type = 'art'` pointed at native `art_items.id` and `item_type = 'collectible'` pointed at `collectibles.id`.

## Breaking changes
- `/api/art/categories` is no longer part of the documented public API contract.
- Art creation and editing no longer accepts collectible category fields through the Art request schema.
- `/api/collectibles` rejects Art creation/reclassification paths and excludes Art rows from Collectibles reads.

## Environment/config changes
- No new required operator configuration in `3.4.4`.

## Migration and data impact
- No new database migration in this release.
- Existing migrated Art rows remain bridge-safe through `art_items.source_collectible_id` while the product identity is native Art.
- Existing shared Event purchase links remain compatible.
