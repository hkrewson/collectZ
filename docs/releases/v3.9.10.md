This patch lets saved Apple/iTunes Wishlist rows reopen their store page directly from the Wishlist list when collectZ has a stored Apple store URL.

## Changes

- Added an `Open store` action to saved Wishlist rows that have a safe `http` or `https` store URL.
- Reused the Apple/iTunes store URL already saved in Wishlist source metadata.
- Added regression coverage proving saved Apple/iTunes Wishlist rows expose the store link.
- Kept search-result store links unchanged.

## Security and privacy

- No auth, scope, token, or provider-write behavior changed.
- Store links are user-initiated outbound links and are limited to `http` or `https` URLs.
- No secrets or provider credentials are added to release artifacts.
