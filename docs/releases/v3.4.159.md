## Milestone target and status

- Milestone: `3.4.159 — Barcode Provider Title Variant Lookup`
- Status: `released`

## Summary

This patch surfaces distinct barcode provider title variants from nested offer/listing rows so both the web UPC lookup and iOS scanner lookup can show messy real-world UPC choices like `Before the Storm` when the provider includes that title beneath a top-level product.

## What changed

- Expanded normalized barcode matches to include distinct `offers[].title` and listing title variants.
- Kept the top-level provider product candidate while adding meaningful provider title variants as additional candidates.
- Added `alternateTitles`/`alternate_titles` readback so clients can show related provider title strings.
- Added title-variant metadata fields such as `titleVariantSource`, `titleVariantIndex`, and `offerIndex`.
- Updated scanner smoke coverage so both `/api/media/lookup/barcode` and `/api/media/lookup-upc` prove offer title variants are returned.

## Security triage

- Lookup auth behavior is unchanged.
- Scanner lookup remains authenticated with `media:read`; import remains authenticated with `import:run`.
- The new smoke fixture uses local fake provider data and does not write provider secrets to evidence.

- collectZ does not currently assign confidence scores to title variants.
