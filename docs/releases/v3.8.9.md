# collectZ v3.8.9

## Version and date

- Version: `3.8.9`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.9 — Capture Inbox Mobile Barcode Camera Fix`
- Status: `implemented`

## Summary

This release tightens Capture Inbox on mobile Safari by making the new-capture form scroll correctly inside the dashboard shell and adding a direct camera scan control for barcodes.

## What changed

- Gives Capture Inbox its own scroll container so the iOS Safari form can move past the Title field.
- Adds a `Scan` action beside Barcode / ISBN that opens the device camera and decodes the captured image through the existing BarcodeDetector/ZXing client pipeline.
- Keeps manual barcode entry and photo upload intact.
- Marks photo upload controls as mobile-camera capable.
- Adds regression coverage for the mobile scan affordance and scrollable capture editor.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- Barcode camera capture fills the existing capture form fields before save; it does not store the temporary scan image.

## Security triage

- Capture Inbox remains authenticated and scoped to the active workspace/library.
- Camera barcode scanning runs in the browser and does not upload the temporary barcode image unless the user separately chooses the photo upload path.

## Security vulnerability triage

- High findings: none identified in local release checks for this slice.
