## Summary

This release starts the maintainability guardrails foundation after the Comic-Con readiness build. It adds report-first frontend quality tooling, a Vitest unit-test seam, and the first small behavior-preserving extraction from `LibraryView.jsx` without changing the Comic-Con field-test workflow.

## What changed

- Promoted the maintainability guardrails work from backlog into the numbered `3.23.0` roadmap milestone.
- Added ESLint flat config with React hooks and JSX accessibility rules for `frontend/src`.
- Added Prettier check configuration without mass-formatting existing files.
- Added `npm run quality:frontend`, which writes ignored local quality reports and exits successfully while the baseline is still being tuned.
- Added warning-only line-count reporting for the largest source files identified by review.
- Added frontend Vitest setup and `npm --prefix frontend run test:unit`.
- Extracted comic issue series/order helpers from `LibraryView.jsx` into `frontend/src/lib/comicIssueOrder.js` with focused Vitest coverage.
- Documented the strangler extraction rule in `docs/wiki/58-Maintainability-Guardrails.md`.
