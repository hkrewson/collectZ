# v3.4.42

## Event social vendor, booth, and location notes

- Version: `3.4.42`
- Target milestone: `3.4.42 — Event Social Vendor Booth Location Notes`
- Date: 2026-04-30

### What changed

- Event meetups can now store vendor, booth, and location-note context.
- Event schedule plans can now store vendor, booth, and location-note context.
- Mobile Event drawers show that context in overview cards and row summaries when present.
- Expanded meetup and schedule rows now include clearer location, vendor/booth, and location-note readback.
- Meetup and schedule quick-edit controls can update these fields without changing source-owned Sched session details.

### Why it matters

This makes day-of-con planning more useful on a phone: a plan can now say not just the room, but the table, booth, vendor, or exact meeting note that matters when someone is already moving through a crowded venue.

### Security and privacy notes

- No new notification, presence, or location-sharing behavior was added.
- Personal Sched ICS source URLs remain hidden and encrypted as before.
- Release evidence should continue to redact transient smoke-test credentials and browser bypass tokens.

### Validation expectations

- Docker-first frontend/backend rebuild should report `3.4.42` from `/api/health`.
- Backend unit, OpenAPI, init parity, migration rehearsal, event social smoke, and browser coverage should pass.
- Help > Releases should serve this release as the latest in-app note.
