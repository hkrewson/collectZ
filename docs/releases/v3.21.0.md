# collectZ v3.21.0

## Version and date

- Version: `3.21.0`
- Date: 2026-06-26

## Milestone target and status

- Roadmap slice: `3.21.0 - First-Class App Route Semantics`
- Status: completed

## Summary

This release makes collectZ dashboard destinations first-class browser routes instead of representing every app view as `/dashboard?tab=...`. Library, workspace, profile, help, and platform surfaces now have canonical paths that match their navigation meaning, while existing dashboard query links remain compatible.

## What changed

- Added a central canonical route map for dashboard destinations.
- Added platform routes such as `/platform/settings`, `/platform/runtime`, `/platform/runtime/logs`, `/platform/users`, and `/platform/workspaces`.
- Added workspace routes such as `/workspace/settings` and `/workspace/review`.
- Kept library routes under `/library/...`, including `/library/movies`, `/library/wishlist`, `/library/capture`, and `/library/import`.
- Updated app-shell history handling so navigation pushes first-class routes and legacy query URLs are replaced with the canonical path.
- Preserved compatibility for older `/dashboard?tab=...` and `/admin/integrations/...` links.
- Removed the remaining Core-hosted platform support UI, frontend support-session plumbing, support-session banner, and Help Admin copy so Core Help now contains only guidance and releases.
- Removed the unreachable Core platform integration diagnostic handlers for PriceCharting, eBay, and structured-log tests while keeping explicit Core-boundary 404 responses for those `cairn`-owned routes.
- Removed dormant platform email-delivery and analytics settings controls from the Core settings view.
- Removed the unmounted platform API docs and metrics route modules from Core after those surfaces moved to `cairn`.
- Removed the obsolete admin-space control smoke and updated Core support/tenancy smokes so fixture workspaces are created directly while `cairn`-owned admin workspace routes remain blocked.

## Breaking changes

- None. Existing `/dashboard?tab=...` links continue to resolve to the correct app destination.

## Environment/config changes

- No new environment variables are required.
- Core no longer emits or consumes `VITE_PLATFORM_API_URL`; platform-backed surfaces now belong in `cairn`.

## Migration and data impact

- No database migration is required.
- No persisted user or workspace data changes.

## Rollback guidance

- Roll back backend and frontend images to `3.20.10` if route canonicalization regresses navigation.
- Existing first-class paths can be manually translated back to their legacy `/dashboard?tab=...` equivalents during rollback.

## Known issues and follow-up

- The internal component state still uses the existing dashboard tab ids. This release fixes browser-visible semantics without yet splitting each destination into independent route modules.
