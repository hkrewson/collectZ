# collectZ v3.4.127

## Summary

This patch makes saved Plex webhook receivers visible again after refresh without exposing the raw receiver token.

## Changes

- Added masked Plex webhook receiver URL readback for existing receiver tokens.
- Added a stable token fingerprint so admins can tell whether the saved receiver is the one they expect.
- Updated the Plex Integrations UI to show existing receiver details, including masked URL, fingerprint, rotation time, last event, and last received time.
- Kept raw Plex webhook receiver tokens one-time only; regenerate the receiver URL when Plex needs the full token again.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- Existing Plex webhook tokens remain hash-only at rest.
- Existing receiver readback exposes only a masked URL and short hash fingerprint, never the raw token.
