## Summary
This release moves new Art writes off the bridge-era Collectibles write path. Art now creates, edits, uploads images, removes images, and archives records through native `art_items` first, while migrated Art records remain readable through their existing bridge-compatible IDs.

## What Changed
### Native Art writes
- New `/api/art` creates now persist directly to `art_items` with `source_collectible_id = NULL`.
- Art updates now apply to native `art_items` rows first.
- Art image upload, image removal, and archive/delete flows no longer require a hidden Collectibles row for native-only Art.
- Existing migrated Art rows with `source_collectible_id` remain editable and readable through the bridge-safe route ID.

### Bridge compatibility
- `/api/art/:id` resolves native-only Art by native ID and migrated Art by `source_collectible_id`.
- Public Art responses continue to include `native_art_id` and `source_collectible_id` so the UI can distinguish native-only and migrated records during the compatibility window.
- The `source_collectible_id` column intentionally remains in place for migrated rows.

### Event purchase links
- Art create/update flows preserve shared Event purchase linking through `event_purchased_items`.
- Event purchase filters and readback continue to resolve Art through native `art_items.id`.
- Collectibles remain excluded from Art writes and `/api/collectibles` continues to reject Art records.

### Backlog tracking
- Added explicit backlog follow-ups for Art signature provenance, fandom/franchise metadata, Event purchase linking polish, and Collectibles naming review.

## Breaking changes
- No user-facing breaking changes are expected.
- New Art records no longer create hidden `collectibles.subtype = art` rows, so any direct database tooling that expected a Collectibles row for new Art should read `art_items` instead.

## Environment/config changes
- No new required operator configuration in `3.4.6`.

## Migration and data impact
- No schema migration is required.
- Existing migrated Art rows keep their bridge mapping.
- New Art rows are native-only and use `art_items.id` as their public route ID.

## Rollback guidance
- Roll back application images to `3.4.5` if native Art writes cause runtime issues.
- Because this release does not add a schema migration, rollback does not require a schema downgrade.
- New Art rows created under `3.4.6` are native-only. If rolling back to `3.4.5`, verify whether those rows need manual bridge backfill before editing them through older application code.

## Known issues and follow-up
- Full removal of `source_collectible_id` remains intentionally deferred until legacy/public-ID compatibility is proven safe.
- A later public-ID cutover may be needed if the product decides migrated Art should expose native IDs everywhere.
