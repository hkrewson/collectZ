# v2.8.6

## Version and date
- Version: `2.8.6`
- Date: `2026-03-30`

## Milestone target and status
- Target milestone: `2.8.6 — Events and Collectibles UX Alignment`
- Status: `Completed`

## Summary
This release rounds out Events and Collectibles so they behave much more like the rest of the library. Shared object cards now align these surfaces with the main poster-style object language, drawers are read-first instead of implicitly editable, collectibles gained an optional artist field, and events now support their own images so they feel like first-class objects rather than lightweight utility records.

## What Changed
### Shared object treatment and interaction alignment
- Introduced and reused a shared poster-style object card primitive so Events, Collectibles, and the main library object surfaces follow the same size, shape, badge placement, and hover-action language.
- Fixed the shared hover action strip so invisible actions no longer capture clicks in hidden areas of the card.
- Aligned card-click and row-click behavior so Events and Collectibles open detail drawers first, with edit as an explicit secondary action.

### Read-first detail drawers
- Updated Events and Collectibles detail drawers so they present object information in a read-first label/value rhythm like other object detail views.
- Removed the odd summary badge band from these view drawers.
- Kept editing explicit:
  - object click opens detail
  - `Edit` opens the edit drawer
  - artifact editing in Events stays behind an explicit `Edit artifacts` action

### Events object improvements
- Events now support first-class event images instead of only artifact-level images.
- Event cards can render their own image treatment.
- Event detail can show an image and provide a direct open-image action.
- Event edit supports image URL, upload, device capture, and image removal.

### Collectibles object improvements
- Added an optional `Artist` field to collectibles.
- Collectible cards and detail drawers now surface artist information when present.
- Collectibles continue to support first-class image handling and now fit more naturally beside Movies, Games, Books, Audio, and Comics.

## Breaking Changes
- None intended at the API contract level for existing clients.
- This release adds small schema fields for Events and Collectibles but does not remove prior behavior.

## Environment and Config Changes
- No new required environment variables were introduced in this release.
- Existing image upload behavior continues to use the current storage configuration.

## Migration and Data Impact
- Added schema migration `47`:
  - `events.image_path`
  - `collectibles.artist`
- `init.sql` was updated to stay in parity with the migrated schema.
- Existing Events and Collectibles data remains valid; the new fields are optional.
