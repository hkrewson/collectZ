# collectZ v3.16.32

## Version and date

- Version: `3.16.32`
- Date: 2026-06-07

## Milestone target and status

- Roadmap slice: `3.16.32 — CodeQL Graylog Digest Boundary Hardening`
- Status: completed

## Summary

This patch continues the maintained-source CodeQL baseline cleanup by reshaping the Graylog release-evidence digest helper to match the runtime requirement without looking like app password-storage code.

## What changed

- Changed the observability release-evidence Graylog root-password digest helper from the streaming hash API to Node's one-shot `crypto.hash()` API.
- Preserved the exact SHA-256 hex digest that Graylog requires for `GRAYLOG_ROOT_PASSWORD_SHA2`.
- Replaced the fixed Graylog evidence password-secret value with a runtime-generated secret.
- Added the missing `v3.16.31` release-note follow-up heading required by the publish workflow before moving the version forward.

## Breaking changes

- None.

## Environment/config changes

- None.
- `GRAYLOG_ROOT_PASSWORD_SHA2` remains the required Graylog container setting produced by the local observability evidence script.
- `GRAYLOG_PASSWORD_SECRET` is now generated at runtime for the local evidence stack.

## Migration and data impact

- No schema, migration, or data impact.
- Observability evidence output shape is unchanged.

- No schema rollback is required.
