## Summary

This patch adds a compact pending-update readback to Dashboard Review drawers, so users can see what lookup, cover upload, or manual edits have staged before saving the record.

## What changed

- Added a `Pending updates` section to Dashboard Review drawers.
- Derives pending rows by comparing the loaded media record with the current drawer form.
- Shows staged title, year, format, cover, provider, and type-specific metadata values when they differ from the saved record.
- Keeps assisted lookup, cover upload, known identity readback, manual details, defer/dismiss, and decision history behavior unchanged.
- Preserved the drawer-first product rule: no standalone Review page or nav item.

## Breaking changes

None.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. The pending-update readback is a frontend-only comparison against already loaded record data.

## Known issues and follow-up

- The readback shows values staged by the current drawer form; it does not yet explain provider trust or rank candidate confidence.
- Clearing a field remains governed by the existing save behavior and is not changed by this patch.
- Standalone Review pages/nav items remain out of scope unless a future workflow truly cannot live in Dashboard Review.
