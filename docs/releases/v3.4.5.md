## Summary
This release simplifies Collectibles taxonomy and gives Art the medium/type boundary needed for comic-panel-style artwork. Collectibles keeps one visible classification selector, while Art now captures medium/type and signed status directly.

## What changed
### Collectibles taxonomy cleanup
- Removed `Anime` from new Collectibles category choices because it describes fandom/source rather than the owned object.
- Removed `Comic Panels` from new Collectibles category choices because comic panels fit the Art object model better.
- Kept `Card` as a normal Collectibles classification in the single visible category selector.
- Preserved legacy category resolution so older records remain readable.

### Art medium boundary
- Added Art `Medium / Type` with `Original`, `Print`, `Comic Panel`, `Sketch`, `Commission`, and `Other`.
- Added a simple Art `Signed` field.
- Updated Art cards, rows, detail drawer, and editor to show and edit medium/type plus signed status.

### Migration and API contract
- Added migration `76` for `art_items.medium` and `art_items.signed`.
- Migrated active `Comic Panels` collectibles into native Art with `medium = comic_panel`.
- Updated validation, OpenAPI, init parity schema, and Art route serialization for the new fields.

## Breaking changes
- `Anime` and `Comic Panels` are no longer offered as active Collectibles category choices for new entry/filtering.
- Existing legacy rows remain readable; active Comic Panels records are moved into Art by migration.

## Environment/config changes
- No new required operator configuration in `3.4.5`.

## Migration and data impact
- Adds `medium` and `signed` columns to `art_items`.
- Active Collectibles rows with `category_key = 'comic_panels'` are converted into Art bridge rows and assigned `medium = comic_panel`.
- Event purchase links for converted comic-panel records are preserved through `event_purchased_items`.

## Rollback guidance
- Roll back application images to `3.4.4` if the taxonomy or Art medium boundary causes runtime issues.
- Because this release adds Art columns and migrates Comic Panels into Art, database rollback should use the normal backup/restore path if the schema/data migration itself must be reversed.
- Leaving the migration in place while rolling back application images may leave converted Comic Panels visible through Art rather than Collectibles.
