# collectZ v3.18.5

## Version and date

- Version: `3.18.5`
- Date: 2026-06-13

## Milestone target and status

- Roadmap slice: `3.18.5 — Public Runtime Contract Alias`
- Status: completed

## Summary

This patch adds public-safe runtime contract names for auth responses and moves frontend/OpenAPI usage to those names. The backend keeps the previous fields for compatibility, while the candidate public source audit now reaches zero findings.

## What changed

- Added `runtime_mode` and `runtime_contract` aliases to auth config and user-session responses.
- Updated the frontend to prefer `runtime_mode` while retaining a compatibility fallback for older responses.
- Updated OpenAPI to document the public-safe runtime contract names.
- Reduced the public-source boundary audit from `12` findings across `3` files to `0` findings.

## Breaking changes

- None. Existing backend compatibility fields remain present for current clients.

## Environment/config changes

- None.

## Migration and data impact

- No schema, migration, or collection data impact.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.18.5`.
- Verify `/api/health` reports version/frontend/backend/build `3.18.5`.
- Verify Help > Releases serves `3.18.5`.
- Run `npm run audit:public-source-boundary` and confirm zero findings.
- Run backend unit tests, OpenAPI validation, frontend build, and `npm run release:local-gate`.

## Rollback guidance

- Revert to `3.18.4` if the runtime contract alias needs to be backed out.
- No schema rollback is required.

## Known issues and follow-up

- The public export still intentionally publishes the install/support mirror only. Adding more frontend/OpenAPI source to the public manifest remains a separate follow-up.
- The legacy compatibility fields can be removed in a later compatibility-breaking cleanup only after clients have moved to `runtime_mode` and `runtime_contract`.

## Security vulnerability triage

- High findings: none.
