## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `AdminMergeReviewView.jsx` was the top remaining frontend lint pocket after `3.23.6`. The slice clears stale hook dependencies and documents intentional debounced API/search synchronization without changing merge-review product behavior.

## What changed

- Stabilized Admin Merge Review context and lane-state capture helpers with React callbacks.
- Stabilized Admin Merge Review queue-load helpers so effect dependencies point at the actual API sync functions.
- Documented intentional initial-load, seeded-discovery, and debounced-search effects.
- Removed a stale default React import.
- Updated the focused admin-shell browser regression to enter Merge Review through the visible sidebar control.
- Reduced the full frontend quality baseline from `89` errors / `39` warnings to `81` errors / `34` warnings.
