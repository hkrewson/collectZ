# v3.4.49

## Event schedule quick plan states

- Version: `3.4.49`
- Target milestone: `3.4.49 — Event Schedule Quick Plan States`
- Date: 2026-05-01

### What changed

- Added compact plan-state controls to event catalog sessions.
- Added the same controls to Now / Next catalog items.
- Users can mark a catalog session as planned, maybe, backup, or skipped.
- If the catalog session is not already in the user's schedule, choosing a state creates a private linked schedule plan.
- If the catalog session is already linked, choosing another state updates the existing plan instead of creating a duplicate.
- Preserved the `schedule_catalog` plus catalog-session-id linkage used by the platform companion contract.

### Why it matters

The Now / Next and catalog views are now actionable. Users can make quick day-of-event planning decisions from the catalog without opening a separate manual schedule form or creating duplicate schedule entries.

### Security and privacy notes

- This patch does not add provider import automation, notifications, friend/group attendance, realtime location, presence, or offline mutation queues.
- Quick state changes create private schedule plans by default.
- Personal Sched ICS feed URLs remain hidden and are not used as a full event catalog source.
