## Summary

This release continues the maintainability guardrails work with a focused Library view cleanup. It makes the touched card/list-row and drawer surfaces lint-clean while preserving existing article-based browser contracts and avoiding broad Library view data-flow rewrites.

## What changed

- Removed stale `LibraryView.jsx` default React import, unused card props, unused catch bindings, and stale lint disables.
- Preserved article-based media/list card containers while adding keyboard activation for the list-row surface.
- Documented narrow a11y exceptions around stop-propagation wrappers that protect card/row click behavior for nested rating and selection controls.
- Replaced collection editor/detail overlay divs with native button backdrops while preserving the media editor overlay's existing non-focusable click-to-close behavior.
- Tightened safe memo/effect dependencies and documented remaining external synchronization effects.
- Reduced the full frontend quality baseline from `153` errors / `61` warnings to `115` errors / `47` warnings.
