## Summary
This patch release removes the remaining “control plane” language from the mobile shell so the app header reads like a normal product instead of a product-marketing or ops-console surface. Mobile admin and support context is still visible, but the wording is now plainer and more honest about what the user is actually looking at.

## What changed
### Mobile header copy normalization
- Replaced `Homelab control plane` with `Homelab`.
- Replaced `Platform control plane` with `Platform admin`.
- Replaced `Support control plane` with `Support`.

### Shell language cleanup
- Preserved the existing mobile header structure, current-space path, and role-aware context.
- Removed the strongest remaining control-plane phrasing from the mobile shell path without changing behavior.

## Breaking changes
- No backend or API-breaking changes are introduced in `2.10.9`.
- This release intentionally changes mobile admin/support wording to be plainer and more product-native.
