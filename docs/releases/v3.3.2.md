## Summary
This patch release builds on `3.3.1` loan reminders by adding automatic reminder scheduling for due-soon and overdue loans. The shipped scope keeps the implementation deliberately tight: a shared backend reminder service, a lightweight in-process scheduler, phase-aware duplicate prevention, and Docker-backed proof that automatic sweeps send the right reminders without repeating them immediately.

## What changed
### Automatic reminder scheduler foundation
- Added a shared backend loan-reminder service so manual sends, automatic sweeps, and runtime proofs all use the same phase rules.
- Added automatic reminder sweep execution to the backend startup timer pattern instead of introducing a separate worker or queue system.
- Added a restricted admin trigger endpoint for running the automatic reminder sweep on demand.

### Phase-aware duplicate prevention
- Added narrow phase-specific reminder tracking for due-soon and overdue sends.
- Due-soon reminders now send once per loan lifecycle inside the due window.
- Overdue reminders now send at most once per day.
- Manual reminder sending remains intact and now shares the same reminder rules as the scheduler path.

### Runtime proof and contract sync
- Added Docker-backed platform and homelab smokes for automatic reminder sweeps.
- Kept init parity and migration rehearsal aligned with the new reminder-tracking migration.
- Regenerated the in-app release feed for `Help > Releases`.

## Breaking changes
- None intended for operators or end users.
- `3.3.2` is a patch follow-up on the shipped loans reminder workflow, not a redesign of loans or reminder history.

## Environment/config changes
- Automatic reminder scheduling is runtime-configurable and remains gated by reminder-delivery readiness.
- Existing SMTP/email delivery configuration is still required for actual outbound reminder sending.
- The scheduler interval is controlled by the backend runtime settings used by the automatic reminder service.

## Migration and data impact
- `3.3.2` adds a schema migration for phase-specific reminder tracking on `media_loans`.
- The migration adds:
  - `due_soon_reminder_last_sent_at`
  - `overdue_reminder_last_sent_at`
- Full reminder-event history is still not part of `3.3.2`; that remains follow-up backlog work.
