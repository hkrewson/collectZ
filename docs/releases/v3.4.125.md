# collectZ v3.4.125

## Summary

This patch processes queued Plex `library.new` webhook import hints by fetching the single PMS metadata item and running it through the existing Plex import path.

## Changes

- Added `POST /api/media/process-plex-webhook-import-hints` for admins to process one queued Plex webhook import hint.
- Added Plex metadata readback for `/library/metadata/:ratingKey`.
- Reused the existing Plex import path for the fetched item, including duplicate protection, scoped media writes, Plex metadata aliases, and valuation follow-up behavior.
- Kept watched-state sync, rating writeback apply behavior, scheduled cadence, and broad Plex import rewrites out of scope.
- Extended the Plex webhook smoke to prove queueing, duplicate reuse, single-rating-key processing, real media-row readback, revocation, and secret-free evidence.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- Webhook receiver tokens remain hash-only at rest and redacted from request/error logs.
- The processor stores sanitized job scope and summary readback only; it does not expose Plex tokens, PMS URLs, raw PMS payloads, or media file paths.
