# collectZ v3.4.72

## Event Schedule Delivery Attempt Persistence and Readback

This patch turns the Event schedule notification delivery-attempt model into a local audit/readback path without enabling push, email, native device delivery, or external providers.

### What's New

- Added persisted Event-local delivery-attempt records for sent schedule notifications.
- Created one `event_local` attempt per selected notification recipient.
- Added delivery-attempt summaries to schedule notification history responses.
- Added scoped delivery-attempt readback for platform/native planning and diagnostics.
- Backfilled local attempt rows for existing sent Event schedule notifications.
- Updated OpenAPI, smoke coverage, and platform companion docs for the new local audit contract.

### Notes

Delivery attempts in this release are local audit records only. They mean collectZ recorded the selected-recipient local send path; they do not indicate push, email, native device, realtime, or external provider delivery.
