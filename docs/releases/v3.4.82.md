# collectZ v3.4.82

## Event Attendee Duplicate Guardrails

This patch stabilizes Event social planning attendee entry after the recent self-attendee improvements. The People panel now catches likely duplicate attendee names before save, points users at the existing row when there is one, and still allows intentional Event-local non-user duplicates through an explicit acknowledgement.

### What changed

- Added frontend guardrails for duplicate and very similar Event attendee display names.
- Keeps manual non-user attendee entry intact through an explicit `Add anyway` path.
- Warns when a manually entered attendee name looks like the signed-in user's self attendee and points users back to `Add me to this event`.
- Improves linked self-attendee duplicate API responses so the existing attendee row is returned with clearer copy.
- Preserves both the explicit `Add me to this event` flow and first-social-action self-attendee auto-create behavior.
- Keeps external contacts, cross-event identity, Discord delivery, realtime presence, and broader friend-graph work out of scope.
