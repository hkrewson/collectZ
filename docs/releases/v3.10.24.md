## Summary

This patch upgrades the backend validation library to Zod 4 and keeps collectZ's request-validation error shaping compatible with the new error surface.

## What changed

- Updated backend `zod` from `^3.22.4` to `^4.4.3`.
- Updated validation middleware to read `ZodError.issues`, while retaining a fallback for older error shapes.
- Kept validation failure responses in the same `{ error, details }` structure used by existing clients.

Rollback to `3.10.23` is safe because this patch has no schema, API route, or data-format changes. If validation behavior regresses, restore the Zod 3 dependency and the previous middleware error mapping.

## Known issues and follow-up

- Zod 4 is a major dependency upgrade, so validation edge cases should be watched in CI and during normal import/scanner/admin use.
- React 19, Tailwind 4, and Express 5 remain separate PR-phase upgrades.
