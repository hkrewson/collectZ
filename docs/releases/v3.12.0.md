## Milestone target and status

- Status: `completed`

## Summary

This minor release adds the first unified Review queue. Instead of forcing reviewable work to stay scattered across Dashboard, Library, Capture Inbox, Wishlist, Plex, and Import, collectZ now has one read-only queue that explains what needs attention and links back to the workflow that owns the fix.

## What changed

- Added `GET /api/review-queue` for active-scope review readback.
- Added a Review page in the app navigation.
- Aggregated Library health items, Capture Inbox review items, Apple/iTunes target-price hits, Plex reconciliation conflicts, and failed syncs.
- Added source/type filters, search, source counts, and workflow actions.
- Kept resolution actions in the existing owning surfaces for this foundation slice.
- Updated OpenAPI, route scope mapping, unit source-contract coverage, and browser regression coverage.

## Breaking changes

None. This is an additive read endpoint and UI page.

## Environment/config changes

None.

## Migration and data impact

No schema migration or data backfill is required. Review Queue reads existing media, capture, wishlist, Plex conflict, and sync-job data.
