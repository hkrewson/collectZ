## Summary
This release lets native Art records track physical artwork dimensions and framed status as first-class Art metadata, keeping those fields out of Collectibles while preserving event purchase linking.

## What changed
### Art physical metadata
- Added native Art height and width fields for physical artwork/image dimensions.
- Added a framed boolean on Art records.
- Art create, edit, list, and detail readback now include the new metadata.

### Drawer experience
- Added compact `H` and `W` fields to the Art drawer to match the collection shorthand.
- Added framed state to the Art drawer alongside other boolean Art attributes.
- Kept vendor and booth behavior tied to event-linked purchase context.

### Regression coverage
- Added source-level coverage for the Art schema, routes, and OpenAPI contract.
- Added runtime coverage for creating and updating Art records with dimensions and framed metadata.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing Art records remain compatible; unset height, width, and framed values read back as empty or false metadata.

## Environment/config changes
- No new required operator configuration in `3.4.14`.

## Migration and data impact
- Adds nullable Art metadata columns for height, width, and framed status.
- Existing Art rows are not destructively rewritten.
- Dimensions are stored as decimal numeric values without enforcing a unit in this slice.
