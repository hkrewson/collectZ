## Summary

This release clarifies the app boundary between workspace-owned collection work and platform-level setup.

## What changed

- Renamed the visible `Admin` nav group to `Platform`.
- Renamed platform settings to `Platform Settings` with copy that explains the instance-wide scope.
- Renamed workspace settings to `Workspace Settings` with copy that explains the selected-workspace scope.
- Renamed workspace provider setup to `Workspace Integrations`.
- Changed the hosted platform runtime surface to show platform runtime concerns instead of editable data-provider setup.
- Dashboard provider health now points toward Workspace, where provider/data-sync setup belongs.
