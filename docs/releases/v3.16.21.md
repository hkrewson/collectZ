## Summary

This patch starts the CodeQL baseline remediation work by hardening backend input boundaries for user-supplied calendar feeds, Kavita connector URLs, and CSV import upload handling.

## What changed

- Added a shared outbound URL policy for user-supplied ICS feeds.
- Blocked non-HTTP(S), credential-bearing, localhost, private, loopback, and link-local ICS feed targets by default.
- Added `ALLOW_PRIVATE_ICS_FEEDS=true` for deliberate private-feed deployments or controlled local test topologies.
- Tightened Kavita base URL normalization to HTTP(S) URLs without embedded credentials while preserving LAN-hosted Kavita support.
- Moved generic, Calibre, and Delicious CSV imports to multer memory buffers instead of request-controlled temp file paths.
- Added unit coverage for the URL policy, Kavita normalization, and in-memory CSV import boundary.

## Breaking changes

- User-supplied ICS feeds that point to localhost, private, loopback, or link-local addresses are rejected unless `ALLOW_PRIVATE_ICS_FEEDS=true` is set.

## Environment/config

- Optional: `ALLOW_PRIVATE_ICS_FEEDS=true` permits private ICS feed hosts for controlled deployments that intentionally sync from private calendar endpoints.
