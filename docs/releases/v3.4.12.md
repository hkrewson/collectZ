## Summary
This release brings the event autograph linking contract into the Event drawer. Autograph artifacts can now capture signer details, show their event-side provenance, and link directly to canonical Art or media signature records from the UI.

## What changed
### Event autograph capture
- Added autograph-specific fields to Event artifact editing for signer name, signer role, signed date, signed location, proof image URL, and signature notes.
- Event autograph rows now display their captured signer/provenance details in the Event drawer.
- Proof image links remain simple URL/path references in this slice; multi-proof management stays future work.

### Object signature linking UI
- Added a compact `Link signature` workflow on autograph artifact rows.
- Users can search Art or media records from the Event drawer and attach the autograph as canonical object signature evidence.
- Linked rows show object signature readback so the Event remains capture/history while Art/media remains the signed-object evidence surface.

### Regression coverage
- Added source-level coverage for Event autograph linking UI wiring.
- Added a browser regression that creates an Event autograph artifact, links it to Art from the drawer, and verifies the Art detail API is signed with event provenance.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing event artifacts, purchased-item links, Art records, and media records remain compatible.

## Environment/config changes
- No new required operator configuration in `3.4.12`.

## Migration and data impact
- No schema migration is included in this release.
- Uses the `signature_records` and `event_artifacts.signature_record_id` contract introduced in `3.4.11`.
- Existing rows are not destructively rewritten.

- Prefer rolling forward with a corrected `3.4.x` build if the drawer linking workflow needs adjustment.
- If rolling back application images to `3.4.11`, the API-level autograph linking contract remains available, but the dedicated drawer linking workflow and autograph capture fields added here are not visible in the older frontend.
