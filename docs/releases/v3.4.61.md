## Event schedule user-linked attendee identity

- Version: `3.4.61`
- Date: `2026-05-02`
- Target milestone: `3.4.61 — Event Schedule User-Linked Attendee Identity`

### Summary

This release lets an Event attendee optionally link to the current app user, so Event-local notification readback can identify which recipient rows are yours without adding push delivery, email, device registration, or a global friend graph.

### Changes

- Added optional event-scoped attendee-to-user linkage with active duplicate prevention.
- Added safe `link_current_user` attendee create/update support.
- Added linked identity readback for attendees, group members, schedule previews, and notification inbox rows.
- Added `recipient=me` filtering and `mine` counts for Event-local notification inbox readback.
- Added quiet drawer UI for linking an attendee to your app user and reading back “Linked to you.”
- Kept linked identity scoped to app user id/name; attendee readback does not expose email.
