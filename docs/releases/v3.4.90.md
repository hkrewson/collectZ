# collectZ v3.4.90

## Kavita Cover Art Source Hardening

This patch makes Kavita-imported cover art render through a collectZ-authenticated proxy so Kavita API keys, OPDS keys, and bearer tokens stay server-side while imported book and comic rows still get useful cover images.

### What changed

- Added a collectZ Kavita cover proxy at `/api/media/kavita-cover/{seriesId}` that only serves covers for Kavita rows visible in the active scope.
- Kept Kavita cover source metadata on imported rows with `kavita_cover_url`, `kavita_cover_proxy_url`, `kavita_cover_source`, and `kavita_cover_status`.
- Updated Kavita imports to use the collectZ cover proxy as `poster_path` while preserving the raw Kavita cover image path for troubleshooting.
- Updated frontend image URL handling so authenticated collectZ `/api/...` image paths are not rewritten as TMDB poster paths.
- Added smoke and unit coverage proving proxied Kavita cover readback works without exposing Kavita credentials.
- Kept Metron/Google Books enrichment, metadata writeback, embedded reading, and reading progress sync out of scope.
