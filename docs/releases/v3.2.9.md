## Summary
This patch release fixes a comic edit and provider-enrichment failure in the Metron flow. When a reselection from Metron returns an oversized description, the app now clamps that overview safely instead of failing save validation, while keeping the field editable in the drawer and preserving deterministic backend behavior for direct API clients.

## What Changed
### Backend-safe overview normalization
- Added overview normalization in request validation so incoming overview text is trimmed, empty values resolve to `null`, and oversized descriptions are truncated deterministically to the current 10,000-character limit before validation completes.
- Kept the existing save contract stable while making the API resilient to provider payloads that exceed the local overview cap.

### Drawer-side truncation and edit usability
- Clamped provider overview text in the comic edit drawer before save so the common Metron re-enrichment path no longer surprises the user with a validation failure.
- Kept the overview field editable and added a compact character count so the drawer remains usable when provider text is long.
- Added a lightweight warning when provider overview text had to be truncated to fit the save limit.

### Docker-backed Metron regression proof
- Added a Docker-backed fake-Metron smoke covering the exact Alpha Flight-style repro: existing comic, Metron reselect, oversized provider overview, successful save, and preserved provider issue identity.
- Proved on both platform and homelab stacks that an oversized Metron overview now saves with deterministic truncation instead of failing validation.
