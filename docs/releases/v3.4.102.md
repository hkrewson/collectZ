## Kavita Progress Writeback and Page Proxy Reader

## Summary

This patch adds the first explicit Kavita progress writeback and page-proxy reader slice. Chapter-backed Kavita rows can now load a proxied reader page, then save a selected page number back to Kavita without exposing Kavita API keys or bearer tokens to the browser.

## What Changed

### Shipped

- Added a scoped progress write endpoint for Kavita chapter-backed media rows using `POST /api/Reader/progress`.
- Added scoped reader metadata and single-page image proxy endpoints for Kavita chapter-backed rows.
- Added media drawer controls to load a Kavita page image, step through pages, and explicitly save progress.
- Kept Kavita credentials server-side for reader page and progress calls.
- Extended the fake-Kavita import/sync smoke to prove progress writeback, reader metadata readback, page image proxying, and secret-free browser responses.
