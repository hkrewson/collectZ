# collectZ v3.4.69

## Event Schedule Notification Delivery Boundary Platform Contract

This patch adds a stable Event-scoped delivery boundary for schedule notifications so web and future platform/native clients can clearly distinguish local coordination records from external delivery.

### What's New

- Added `GET /api/events/:id/schedule-notification-delivery-boundary`.
- Documented the current supported channel: Event-local notification records with recipient readback.
- Explicitly reports push, email, native device delivery, realtime fanout, global inboxes, and broadcast behavior as unsupported.
- Added Event social smoke coverage for the delivery-boundary contract.
- Added OpenAPI coverage for the delivery-boundary response.
- Added a small Event drawer readback note so users see that schedule notices are local event records only.

### Notes

This does not add push, email, or native device delivery. It makes the current boundary explicit so later platform work can build on a versioned contract instead of guessing from web UI behavior.
