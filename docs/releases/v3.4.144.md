# collectZ v3.4.144

- Version: `3.4.144`
- Status: `released`

## Summary

This patch adds a guarded attach-existing path for Plex reconciliation conflicts so an admin can link a Plex identity to an existing collectZ title when the identifiers are safe.

## Changes

- Adds an `attach_existing` Plex conflict resolution action with active-library scoping.
- Rejects unsafe attaches when strong identifiers, media type, Plex item key, or Plex GUID disagree.
- Stores Plex identity metadata on the selected existing collectZ row when the attach passes guardrails.
- Shows an Attach to existing action in the Plex conflict review queue when a suggested existing row is available.
- Keeps Plex writeback disabled and leaves automatic conflict resolution out of scope.
