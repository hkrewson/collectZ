## Summary

This patch standardizes search behavior across the app: search fields update results as the user types instead of requiring a separate Search button. Surfaces that call backend/provider search endpoints now use short debounce windows so the interface stays immediate without turning every keystroke into a hard submit action.

## What changed

- Removed remaining visible Search submit buttons from search/filter surfaces.
- Kept Wishlist and Capture Inbox desktop search behavior aligned with their mobile live-search behavior.
- Made Activity search reload automatically after search text changes.
- Made Apple/iTunes Wishlist intake search automatically with debounce.
- Made event object-link pickers search automatically while their panels are open.
- Made media add/edit lookup matches update automatically from title or identifier.

## Rollback guidance

Rollback to `3.10.9` is safe because this patch has no schema or data-format changes. If live search creates excessive provider traffic or confusing timing on a specific surface, restore that surface's prior explicit submit behavior and document the exception.

## Known issues and follow-up

- Provider-backed lookup surfaces use debounce rather than a global query scheduler. If future providers need stricter rate shaping, promote that as a backend/provider throttling task instead of reintroducing UI submit buttons.
