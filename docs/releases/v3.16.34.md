## Summary

This patch continues maintained-source CodeQL cleanup by making the existing custom CSRF middleware recognizable to CodeQL while preserving the browser cookie-session CSRF contract.

## What changed

- Kept mutating browser requests on the existing double-submit CSRF cookie and `x-csrf-token` header validation path.
- Made the default `csrf_token` cookie read explicit when the deployment uses the default CSRF cookie name, while preserving custom `CSRF_COOKIE_NAME` support.
- Added unit coverage for accepted matching CSRF tokens and rejected missing CSRF headers.
