## Summary
This patch cuts the Art library read path over to native Art storage while keeping bridge-compatible response ids during the transition. Events now read tracked Art purchases and tracked Collectibles purchases through the shared purchased-item relationship, and the Art drawer treats vendor and booth as event purchase context instead of always-on Art identity fields.

## What changed
### Native Art read cutover
- Updated `/api/art` list and detail reads to query `art_items` as the primary source.
- Preserved bridge-compatible response ids through `source_collectible_id` so existing UI edit, delete, and upload flows keep working during the cutover.
- Added native Art response fields for:
  - `native_art_id`
  - `source_collectible_id`
  - `event_id`
  - `purchased_item_id`
  - `series`
  - `artist`
  - `vendor`
  - `booth`
  - `price`
  - `exclusive`
- Kept native Art list filters working for search, series, vendor, booth, event, exclusive state, scope, sorting, and pagination.

### Event purchase readback
- Added Event detail readback for tracked purchases from `event_purchased_items`.
- Events now show linked Art and Collectibles purchases through the same relationship model.
- Purchase readback displays resolved item context such as title, type, artist/series, vendor/booth, price, and image when available.

### Art drawer behavior
- Updated the shared Art drawer so vendor and booth appear only when purchase context exists:
  - linked event,
  - existing vendor,
  - existing booth,
  - or legacy combined vendor/booth value.
- Preserved existing vendor and booth values so editability and data fidelity are not lost during the native read cutover.
- Fixed the Art detail drawer crash caused by checking purchase context before the detail row had loaded.

### Verification coverage
- Added `test:native-art-read-cutover-smoke` to prove the native `/api/art` detail, list, and event-filter read contract against the running backend stack.
- Expanded the Events and Collectibles Playwright browser regression to cover Art drawer vendor/booth visibility and readback behavior.
- Added source-level regression assertions for the native Art read cutover, event purchase readback UI, and null-safe Art purchase context helper.

## Breaking changes
- None intended.
- `/api/art` still preserves bridge-compatible ids during this transition so existing frontend actions and external callers are not forced to switch to native Art ids immediately.

## Environment/config changes
- No new required operator configuration in `3.4.3`.

## Migration and data impact
- No new database migration in this patch.
- This release depends on the native Art storage and shared event purchased-item schema introduced in `3.4.1` and the existing-row backfill completed in `3.4.2`.
- Existing Art data remains in place; runtime reads now prefer native Art rows.
