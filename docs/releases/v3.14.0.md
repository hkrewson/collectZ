# collectZ v3.14.0

## Version and date

- Version: `3.14.0`
- Date: 2026-06-05

## Milestone target and status

- Roadmap slice: `3.14.0 — Reusable Collectible Traits Readback Foundation`
- Status: completed

## Summary

This minor release starts the reusable collectible traits milestone with a shared readback contract for special item details that already exist across the app.

## What changed

- Added backend `collectible_traits` summaries for media, Art, and Collectibles responses.
- Derived shared traits from existing data instead of adding new required fields.
- Covers signed/autographed, numbered/limited, certificate/proof, event-acquired/exclusive, and edition/variant readback where current data supports it.
- Added compact trait chips and drawer readback in Library, Art, and Collectibles surfaces.
- Added an OpenAPI `CollectibleTrait` schema and documented optional trait arrays on concrete response schemas.
- Promoted the selected backlog task into the roadmap as `3.14.0`.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- No schema migration is required.
- Existing media, Art, Collectibles, signature, event, and edition fields are unchanged.
- The new trait arrays are read-only derived response fields.

- Roll back to `3.13.0` if the derived trait readback causes unexpected Library, Art, or Collectibles rendering behavior.

## Known issues and follow-up

- This slice does not add a generalized trait persistence table.
- Grading, richer COA/provenance editing, bundle relationships, and library-specific edition editors remain follow-up milestones.
- Existing Art signed/numbered fields and media signature records remain the source of truth for this first trait readback.
