# collectZ v3.4.74

## Platform Companion Now/Next Schedule Experience

This patch adds a native-friendly Now/Next schedule contract to the Event companion snapshot. Platform clients can now read current, upcoming, soon, and nearby catalog sessions from `/api/events/:id/companion/today` without depending on web UI state.

### What changed

- Added `now_next` with contract version `event-companion-now-next.v1` to the Event companion response.
- Clearly distinguishes catalog-only sessions from linked manual plans and personal Sched ICS plans.
- Adds quick-action endpoint hints for `planned`, `maybe`, `skipped`, and `backup` using existing schedule-plan APIs.
- Adds read-only conflict hints for overlapping personal/shared schedule plans.
- Keeps native UI, offline mutation queues, push/email/device delivery, realtime presence, and friend discovery out of scope.
