## Summary

This release continues the maintainability guardrails work by clearing the remaining hard ESLint error baseline without broad UI refactors. The full frontend quality report now has zero ESLint errors, while legacy warnings and Prettier findings remain report-first.

## What changed

- Tuned `react-hooks/set-state-in-effect` from error to warning because the remaining findings are mostly established async load/synchronization effects rather than immediate correctness failures.
- Fixed the Design Lab slider accessibility error with explicit range input labeling.
- Adjusted the mobile dashboard shell icon render path so the component is not recreated during render.
- Reduced the full frontend quality baseline from `19` errors / `24` warnings to `0` errors / `40` warnings.
