## Summary
This patch release hardens duplicate prevention for the re-enabled OPDS / digital-library import path. It proves repeat OPDS syncs reuse canonicals safely, fixes the stored OPDS link contract for browse versus download actions, and closes a comic-heavy import gap so parsed comic identity metadata survives persistence without recreating duplicate rows on later syncs.

## What Changed
### OPDS importer reactivation and repeat-sync proof
- Re-enabled the live `/api/media/import-cwa` route instead of leaving the existing importer stranded behind a deferred response.
- Added a Docker-backed repeat-sync smoke proving one OPDS entry imported twice updates or no-ops the same canonical row through persisted `provider_item_id` / `calibre_entry_id` identities.

### OPDS link contract cleanup
- Separated OPDS browse/detail links from acquisition/download links in the importer contract.
- Stopped treating OPDS external URLs as `tmdb_url` stand-ins for books and comics.
- Preserved download-link fields through generic import mapping and drawer rendering so Calibre/OPDS actions stay truthful and deduped in the UI.

### Comic-heavy dedupe hardening
- Added a Docker-backed comic-heavy OPDS smoke that proves a clearly comic-shaped OPDS entry imports as `comic_book`, persists parsed `series`, `issue_number`, and `volume`, and then reuses the same canonical row on repeat sync.
- Fixed generic import mapping so row-backed OPDS `type_details` fields are not dropped before persistence, which previously stripped comic identity metadata from live OPDS imports.
