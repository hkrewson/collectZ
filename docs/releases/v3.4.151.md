## Version and date

## Milestone target and status

- Milestone: `3.4.151 — Plex Sync Operating Model Cleanup`
- Status: `released`

## Summary

This patch cleans up the Plex admin operating model after the main Plex integration run. The settings surface now explains which Plex actions are manual, which can run on schedules, which are webhook-triggered hints, and which writeback actions stay explicit user controls.

## What changed

- Adds a Plex operating-model readback panel to Integrations.
- Renames user-facing reconciliation scan controls from preview wording to read-only check wording.
- Clarifies that manual and scheduled Plex sync share the same safe create/update policy and send ambiguous rows to conflict review.
- Clarifies that Plex webhooks can feed import hints while watched-state and rating writeback remain manual controls.
- Moves the Plex operating-model cleanup task out of backlog and into the completed roadmap milestone.

## Breaking changes

- None.

## Environment/config changes

- None. Existing Plex API URL, token, library section, webhook receiver, reconciliation scheduler, watched-state refresh, and writeback settings continue to apply.

## Migration and data impact

- No database migration or data backfill is required.
- Plex import, sync, conflict review, webhook processing, watched-state writeback, rating writeback, and provider-advertised section discovery behavior are unchanged.

## Deployment and verification

## Rollback guidance

- Revert the `3.4.151` UI/docs changes to return the Plex admin copy to the previous labels.

## Known issues and follow-up

- CI-only secure-cookie compose smoke, secret scan, and image security/SBOM gates still run in GitHub Actions.
