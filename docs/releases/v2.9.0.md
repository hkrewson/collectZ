# v2.9.0

## Version and date
- Version: `2.9.0`
- Date: `2026-04-01`

## Milestone target and status
- Target milestone: `2.9.0 — Assisted Capture and Barcode Completion`
- Status: `Completed`

## Summary
This release completes the barcode-assisted entry milestone by turning barcode lookup and capture into a first-class workflow across the library and import surfaces. Shared barcode capture now supports direct camera or photo-assisted entry where browsers allow it, barcode lookup is media-type aware across books, audio, games, comics, movies, and TV, and direct cover-image attachment remains available without the abandoned OCR path. The release also hardens normalization, diagnostics, and result application against real-world UPC and ISBN cases encountered during milestone validation.

## What Changed
### Barcode-assisted entry completion
- Added and hardened a shared barcode-assisted capture path for manual entry and import flows.
- Barcode lookup is now available directly in the relevant library media types rather than being limited to a movie/TV-only path.
- The add/edit flow supports camera or photo-assisted barcode capture where the browser can provide it, while preserving direct manual barcode entry as a fallback.

### Media-type-aware lookup and application
- The shared barcode route now normalizes provider responses and enriches results by media type:
  - books
  - audio
  - games
  - comics
  - movie/TV via TMDB
- ISBN-scanned books now populate structured book fields like title, author, ISBN, publisher, and edition instead of collapsing metadata into a single title string.
- TV season box set lookups now normalize to series-level TMDB searches instead of being pushed through the movie path.

### Normalization and diagnostics hardening
- Barcode input is normalized more defensively across typed, pasted, and captured values.
- Weak or noisy provider titles are cleaned before enrichment so packaging noise does not dominate search queries.
- Representative real-world barcode cases were used to improve parsing for books, TV season box sets, and combo-pack movie releases.
- Barcode lookup failures now preserve stage-aware diagnostics and request context so debugging stays practical when an upstream provider misbehaves.
- Fixed the library/import button-handler bug that could send the click event object instead of the intended UPC.

### Capture and direct image attachment
- Manual entry continues to support direct cover-image capture and upload where appropriate.
- The OCR/vision identification path was removed from the product surface so the milestone now focuses on dependable barcode completion rather than unreliable image identification.

## Breaking Changes
- OCR/Vision-assisted image identification is no longer exposed in the product surface or integrations UI.
- Direct image attachment remains supported; only image-based identification was removed.

## Environment and Config Changes
- No new required environment variables were introduced in this release.
- Vision/OCR configuration is no longer part of the active runtime/admin surface for this milestone.
- Existing barcode, TMDB, and media-type-specific provider settings continue to be used.

## Migration and Data Impact
- No new schema migration was introduced as part of `2.9.0`.
- Existing media records remain valid.
- This release is primarily workflow, normalization, and integration-path hardening.
