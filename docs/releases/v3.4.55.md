# v3.4.55

## Event schedule catalog ICS import

- Version: `3.4.55`
- Target milestone: `3.4.55 — Event Schedule Catalog ICS Import and Taxonomy Normalization`
- Date: 2026-05-02

### What changed

- Added a one-time Event drawer catalog import for Sched-style or calendar ICS URLs.
- Imported sessions now write to the canonical event schedule catalog, not the user's personal schedule plans.
- Catalog imports are idempotent by provider source reference, so re-importing the same feed updates existing catalog sessions instead of duplicating them.
- Provider taxonomy is normalized for catalog browsing: generic provider categories such as `PROGRAMS` are filtered, useful categories remain, a track is inferred, and room is inferred from location when possible.

### Why it matters

The catalog filters added in the last patches are much more useful when users can seed the full event calendar instead of manually entering every session. This keeps personal Sched ICS sync separate from the full event catalog while giving the web app a safer import-backed discovery path.

### Security and privacy notes

- Catalog import does not store the submitted ICS URL and does not return the raw URL in responses.
- Personal Sched ICS sync remains personal-schedule-only and still stores its feed URL encrypted at rest.
- This patch does not add recurring catalog sync, scraping automation, selected-recipient notifications, friend discovery, push delivery, realtime location, presence, or offline mutation queues.
