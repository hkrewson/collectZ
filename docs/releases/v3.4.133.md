# collectZ v3.4.133

## Summary

This patch adds explicit Plex watched-state writeback for one existing Plex-linked collectZ row.

## Changes

- Added an admin-only watched-state writeback endpoint for `scrobble` and `unscrobble`.
- Resolves the target from an existing Plex-linked collectZ row by media id or Plex rating key.
- Writes to Plex using the proven `PUT /:/scrobble` and `PUT /:/unscrobble` request shape.
- Stores lightweight writeback provenance metadata on the collectZ row.
- Keeps UI auto-sync, scheduled writeback, rating writeback, full-library reconciliation, and broad import rewrites out of scope.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- Smoke evidence records sanitized request shape only and does not include Plex token values, token query strings, private IPs, or raw media file paths.
