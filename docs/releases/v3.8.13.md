# collectZ v3.8.13

## Version and date

- Version: `3.8.13`
- Date: `2026-05-19`

## Milestone target and status

- Milestone: `3.8.13 — Capture Inbox Review Filters`
- Status: `implemented`

## Summary

This release makes Capture Inbox easier to work through by adding review filters for captures that need a decision, are ready to add, have no match, are missing details, or have provider/replay problems.

## What changed

- Added server-backed `review_filter` support to `GET /api/capture-items`.
- Added capture review counts for all review filters.
- Added Capture Inbox review tabs: `All`, `Needs choice`, `Ready to add`, `No match`, `Missing details`, and `Problems`.
- Kept all capture records on the existing Capture Inbox page instead of creating a separate queue.
- Updated OpenAPI to document the new query parameter and response counts.

## Breaking changes

- None.

## Environment/config changes

- No new required environment variables.

## Migration and data impact

- No database migration.
- Review filters are derived from existing capture metadata and review decisions.

## Security triage

- Capture filters remain behind the existing authenticated/scoped Capture Inbox API.
- No provider credentials or scanner secrets are exposed.
