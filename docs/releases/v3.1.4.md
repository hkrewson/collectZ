## Summary
This patch release reshapes account management into a calmer, more intentional shell surface after the library cleanup from `3.1.3`. The profile page, account menu, and profile-image flow now feel more integrated with the rest of the app, and platform and homelab were brought back onto the same refreshed shared UI path where intended.

## What Changed
### Profile surface cleanup
- Reshaped the profile page into a calmer account-management surface with cleaner spacing, flatter supporting treatments, and a more intentional identity summary.
- Reduced the boxed, generic account-page feel so the profile page sits more naturally inside the shell.
- Kept the milestone focused on account/profile experience instead of widening into broader auth or settings redesign.

### Consolidated account navigation
- Replaced the duplicated footer-account treatment with one clearer account entry point in the shell.
- Standardized the account menu contents around `My profile`, `Discord`, `GitHub`, and `Sign out`.
- Tightened the profile/account entry so navigation feels more deliberate and less duplicated across the shell.

### Profile image support and auth/profile persistence
- Added profile-image support across the profile surface and shell account entry.
- Simplified the avatar interaction into a clickable upload target instead of a separate preview box and direct URL field.
- Persisted `users.image_path` through auth/profile handling and added the follow-on schema/migration repair needed so `/api/auth/me` and profile edits recover cleanly on the running stack.

### Edition UI alignment
- Rebuilt homelab and platform onto the same refreshed shared shell and library-adjacent UI path so comics, search, pagination, nav, and profile/account updates land consistently across both editions.
- Preserved intended feature-level differences while removing presentation drift caused by stale frontend/runtime divergence.

## Breaking changes
- None intended for operators or end users.
