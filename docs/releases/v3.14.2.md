## Summary

This patch adds the first editable workflow on top of reusable collectible traits: manual grading details for supported collection records.

## What changed

- Added a compact grading editor to Library, Art, and Collectibles detail drawers.
- Grading details are stored through the reusable trait API as the `graded` family.
- Supported fields include grader, grade, certificate number, grading date, and slab notes.
- Saved grading details now appear through existing collectible trait readback and trait pills.
- The editor remains drawer-based so ordinary add/edit flows stay lightweight.

## Breaking changes

- None.

## Environment/config changes

- None.

## Migration and data impact

- No new migration.
- Uses the `collectible_trait_records` table introduced in `3.14.1`.
- Existing records are unchanged until a user adds grading details.

## Known issues and follow-up

- COA/provenance editing, bundle relationships, and library-specific edition editors remain separate follow-up slices.
- Grading does not affect valuation in this patch.
