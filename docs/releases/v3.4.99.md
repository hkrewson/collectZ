## Kavita Writeback Field Selection UI

## Summary

This patch makes Kavita metadata writeback more deliberate by letting workspace admins choose the exact changed fields to apply after preview.

## What Changed

### Shipped

- Added field-level checkboxes to the media-detail Kavita Metadata diff.
- Default selected all changed unlocked fields after preview, while unchanged rows remain read-only.
- Changed `Apply to Kavita` so it sends only the selected changed fields.
- Added selected-field readback beside the existing changed/skipped counts.
