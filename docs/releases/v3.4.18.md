## Summary
This release lets shared Art and media signature records keep multiple proof images. Existing single-proof compatibility fields remain intact, so older clients can continue reading `proof_path`, `signature_proof_path`, and `signed_proof_path` while newer drawer UI can manage proof evidence as a list.

## What changed
### Multi-proof evidence model
- Added a shared `signature_proofs` child table for signature evidence images.
- Backfilled active `signature_records.proof_path` values into the new proof table.
- Kept `signature_records.proof_path` as the primary/first active proof projection for compatibility.
- Signature readback now includes a `proofs` array.

### Art and media API
- Art and media per-signature proof uploads now append proof evidence instead of replacing the existing proof list.
- Added individual proof removal endpoints for Art and media signatures.
- Existing `/proof` upload/remove compatibility routes remain available.
- Primary Art and media compatibility readback continues to project through `signature_proof_path` and `signed_proof_path`.

### Drawer experience
- The shared Art/media signature manager now lists proof images per signature.
- Users can add additional proof images and remove one proof without deleting the rest.
- Art and media detail readback can show multiple proof links.

### Regression coverage
- Added source-level coverage for schema, route, service, OpenAPI, and shared drawer wiring.
- Expanded Art and media browser/API regressions to prove multiple proof uploads, one-proof removal, and compatibility projection.
- Re-ran init parity and migration rehearsal through migration `82`.

## Breaking changes
- No user-facing breaking changes are expected.
- Existing proof paths remain readable and are backfilled into the new child proof table.

## Environment/config changes
- No new required operator configuration in `3.4.18`.

## Migration and data impact
- Adds nullable/append-only signature proof evidence rows in `signature_proofs`.
- Existing active proof paths are copied into `signature_proofs` without deleting the original compatibility value.
- Archived signatures are not backfilled into active proof evidence.
