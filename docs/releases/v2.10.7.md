## Summary
This patch release finishes the shared tab-strip cleanup so the app’s remaining secondary tab and mode-toggle surfaces now use the same flattened tab language as the newer import, admin, help, and drawer flows. The old pill-shell tab strip is gone from the shared frontend layer, and auth, space management, library, events, and collectibles now all rely on the same shared tab primitive instead of preserving one-off segmented-control styling.

## What changed
### Shared tab-strip flattening
- Replaced the remaining legacy `.tab-strip` and `.tab` callers with the shared `SectionTabs` primitive.
- Extended `SectionTabs` to support compact dividerless usage for small mode toggles without reintroducing the older pill-shell styling.
- Removed the old shared tab-strip CSS from the global frontend style layer.

### Surface alignment
- Updated the auth page sign-in/register switch to use the shared flattened tab treatment.
- Updated Space Manager people tabs to use the shared tab primitive.
- Updated library cards/list view toggles and collection/comics subview tabs to use the shared tab primitive.
- Updated events and collectibles cards/list toggles to use the same shared tab treatment.

### UI language consolidation follow-through
- Added a final `2.10.13 — Shared UI Language Consolidation` roadmap milestone so the remaining auth-page visual cleanup and shared UI centralization are explicitly tracked before `3.0.0`.

## Breaking changes
- No backend or API-breaking changes are introduced in `2.10.7`.
- This release intentionally changes shared tab presentation:
  - legacy pill-shell segmented controls are retired,
  - remaining tabbed surfaces now read more like normal navigation or compact product toggles.
