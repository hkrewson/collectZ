## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `ProfileView.jsx` was the top remaining frontend lint pocket after `3.23.7`. The slice clears Profile form accessibility findings, documents intentional account/token synchronization effects, and keeps token status rendering pure without changing profile product behavior.

## What changed

- Associated Profile form labels with their controls.
- Kept the Profile token-scope group semantic without using a label for a non-control group.
- Documented intentional personal access token loading and authenticated-user form synchronization effects.
- Removed a stale default React import.
- Kept personal access token expiry status based on token-data updates rather than an impure render-time clock call.
- Reduced the full frontend quality baseline from `81` errors / `34` warnings to `70` errors / `33` warnings.
