## Summary

This patch moves Plex reconciliation conflict review match-reason filtering into the backend query. The Plex Sync conflict review dropdown now reflects server-provided reason counts, and selected match reasons are applied before the endpoint row limit instead of filtering only the rows already loaded in the browser.

## What changed

- Added `matchedBy` filtering to the Plex reconciliation conflict-review endpoint.
- Returned match-reason counts for the selected open or resolved conflict-review status.
- Updated the Plex Sync conflict review dropdown to use backend-provided reason counts.
- Reloaded conflict review rows when the selected match reason changes.
- Added source coverage for the backend filter contract and frontend filter wiring.
- Added a local `ci:prepare` script that mirrors the GitHub prepare gate checks.
