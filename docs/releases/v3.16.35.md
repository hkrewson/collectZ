## Summary

This patch keeps maintained-source CodeQL focused on actionable source findings by moving the authoritative query selection into a checked-in suite and dispositioning the intentional HTTP-to-file upload/artifact pattern.

## What changed

- Added `.github/codeql/collectz-maintained-source.qls`, which imports the JavaScript/TypeScript `security-and-quality` suite and excludes only `js/http-to-file-access`.
- Pointed the CodeQL workflow at the checked-in maintained-source suite while keeping the existing generated-output `paths-ignore` config.
- Documented that the excluded HTTP-to-file findings are expected authenticated upload persistence or local smoke/proof artifact writes to fixed/generated paths.
- Normalized upload payloads to a Buffer copy before local or S3 persistence.
