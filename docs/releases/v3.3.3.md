## Summary
This patch release deepens the shipped library-loans reminder workflow by adding event-level reminder history, exposing that history through the loan-detail API, and surfacing compact reminder audit context in the loan-first media drawer. The release keeps the design intentionally narrow: reminder history is now explainable and reviewable without widening the main Loans list into a heavier operator console.

## What changed
### Reminder event persistence
- Added a dedicated `media_loan_reminders` history table for reminder attempts and outcomes.
- Manual and automatic reminder sends now persist event rows with phase, trigger source, status, timestamp, failure summary, and a stable delivery window key.
- Kept the existing `media_loans` reminder fields as the fast current-state source for eligibility and drawer/list status.

### Loan-detail reminder history readback
- Expanded the loan-detail/history API path so `GET /api/media/:id/loans` returns `reminder_events` for each loan record.
- Kept the main `GET /api/media/loans` list response lean instead of widening it into a global reminder-audit payload.
- Updated the OpenAPI contract so the reminder-history shape is documented and testable.

### Drawer and loans-surface refinement
- Aligned the Loans footer with the shared library pagination treatment.
- Surfaced compact reminder history inside the loan-first drawer for both active and recent returned loans.
- Refined loan drawer action hierarchy so status reads as status, primary actions stay primary, and the unopened-state `Loan Out` entry point is quieter and more in line with the rest of the interface.
- Reworked the loan-history drawer treatment to reduce repeated panel weight and keep audit context secondary to the active loan.

### Runtime proof and contract sync
- Added and updated Docker-backed reminder workflow smokes to prove reminder event persistence and API readback on platform and homelab stacks.
- Kept init parity and migration rehearsal aligned with the new reminder-history migration.
- Regenerated the in-app release feed for `Help > Releases`.

## Breaking changes
- None intended for operators or end users.
- `3.3.3` is a patch follow-up on the shipped loans and reminder workflows, not a redesign of the core loan lifecycle.

## Environment/config changes
- No new required operator configuration beyond the reminder/runtime settings already introduced by `3.3.1` and `3.3.2`.
- Existing reminder delivery and scheduler settings continue to govern reminder execution.

## Migration and data impact
- `3.3.3` adds a schema migration for event-level reminder history.
- The migration adds a new `media_loan_reminders` table and related indexes.
- Existing shallow reminder state on `media_loans` remains in place and is still used for fast current-state UI and duplicate-prevention checks.
