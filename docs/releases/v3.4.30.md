## Summary
This release starts the event social planning foundation so con/event planning can track people, groups, meetups, and manual schedule plans inside collectZ before the mobile day-of-con experience is built.

## What changed
### Event social planning data
- Added event-scoped attendees, groups, group members, meetups, and schedule-plan storage.
- Added privacy intent fields with `private`, `selected_people`, `group`, and `event_workspace` visibility values.
- Kept access bounded by the existing Events feature gate and active library/space scope checks.

### API and drawer surface
- Added Event child APIs for attendees, groups, meetups, and schedule plans.
- Documented the new routes and records in OpenAPI.
- Added a simple Social planning section to the Event detail drawer for adding/removing people, groups, meetups, and schedule plans.

### Planning boundary
- Added `docs/wiki/40-Event-Social-Planning-Foundation.md` to document the product/privacy boundary and follow-up order.
- Kept mobile web polish, Now/Next discovery, Sched sync, notifications, and native companion behavior as later milestones.

## Breaking changes
