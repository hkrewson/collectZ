## Version and date

## Milestone target and status

- Roadmap slice: `3.18.1 — Public Mirror Export Builder Automation`
- Status: completed

## Summary

This release adds the local public mirror export builder for the sanitization strategy introduced in `3.18.0`. It can generate a reviewed public tree and optional clean local commit without pushing or publishing anything automatically.

## What changed

- Added `npm run public:export` for building a sanitized local public export tree from `public-export.manifest.json`.
- Added optional `--commit` support to initialize a separate git repository inside the export tree and create a clean public export commit.
- Added local-gate enforcement so normal exports require a current passing `npm run release:local-gate` report.
- Added export-tree validation for denied paths, symlinks, setup-surface content patterns, and secret-like public setup values.
- Updated the public README language so the generated mirror does not depend on maintainer-only `docs/wiki` paths.

## Deployment and verification

- Rebuild backend and frontend containers with `APP_VERSION=3.18.1`.
- Verify `/api/health` reports version/frontend/backend/build `3.18.1`.
- Verify Help > Releases serves `3.18.1`.
- Run `npm run release:local-gate`.
- Run `npm run public:export -- --force`.
- Optionally run `npm run public:export -- --force --commit` to prove clean local commit creation.

## Rollback guidance

- Revert to `3.18.0` if the public export builder needs to be backed out.
- No schema rollback is required.

## Known issues and follow-up

- The builder creates a local export tree and optional local commit only. Publishing to a real public mirror remains an explicit maintainer action.
- A future slice can add a dedicated publish handoff once the mirror target is chosen.
