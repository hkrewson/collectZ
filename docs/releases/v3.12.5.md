## Milestone target and status

- Status: `completed`

## Summary

This patch adds scoped defer and dismiss actions to Dashboard Review drawers so legitimate `not now` and `not needed` items can be handled without creating a separate Review page.

## What changed

- Added `media_review_decisions` to track Dashboard Review decisions by media item and finding type.
- Dashboard Review counts and row samples now hide active deferred or dismissed findings.
- Deferred findings are hidden for seven days.
- Dismissed findings are hidden until the underlying media row changes.
- Added drawer actions for `Defer 7 days` and `Dismiss` beside the existing save controls.
- Added an authenticated `POST /api/dashboard/review-decisions` endpoint and OpenAPI contract.
- Added source-contract coverage for migration/init parity, endpoint shape, PAT scope behavior, and UI controls.

## Breaking changes

None.

## Environment/config changes

None.

## Migration and data impact

This release adds the `media_review_decisions` table. Existing media rows are not modified. New decision rows are created only when a user defers or dismisses a Dashboard Review item.

## Rollback guidance

Rollback is safe. Older application versions ignore the new table. If decision behavior needs to be disabled, roll back to `3.12.4`; existing decision rows can remain in the database without affecting older code.

## Known issues and follow-up

- This patch does not add a global undo surface for deferred or dismissed review items.
- Dismiss/defer remains intentionally scoped to Dashboard Review rows and drawers.
- Future Dashboard Review work should keep improving type-specific repair actions before asking users for manual identifiers.
- Standalone Review pages/nav items remain out of scope unless a future workflow truly cannot live in Dashboard Review.
