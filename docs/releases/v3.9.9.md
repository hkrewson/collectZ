## Version and date

- Milestone: `3.9.9 — Wishlist Source Details Cleanup`

## Milestone target and status

Status: Complete.

This patch makes Wishlist source metadata read like product information instead of raw provider linkage, while preserving the provider keys collectZ needs for matching, refresh, and import workflows.

## Changes

- Replaced raw Wishlist row source text such as provider names plus provider keys with readable provider labels.
- Hid technical Apple/iTunes provider identifier fields from normal Wishlist row summaries.
- Kept identifiers like ISBN, UPC, EAN, barcode, and ASIN visible when they are useful to users.
- Kept source/provider fields available in the advanced edit form for troubleshooting or manual entry.
- Added regression coverage so Apple/iTunes Wishlist rows no longer expose raw provider keys in the normal list.

## Security and privacy

- No auth, scope, token, or provider-write behavior changed.
- Provider keys remain stored for backend identity matching, but are no longer exposed in the default Wishlist row copy.
- No secrets or provider credentials are added to release artifacts.
