# collectZ v3.5.0

## Version and date

- Version: `3.5.0`
- Date: `2026-05-17`

## Milestone target and status

- Milestone: `3.5.0 — Dashboard / Command Center Foundation`
- Status: `released`

## Summary

This minor release starts the `3.5` line with a new Dashboard command center: a first-screen operational summary of what changed, what failed, and what needs attention in the active workspace/library scope.

## What changed

- Added authenticated `GET /api/dashboard/summary` for scoped Dashboard data.
- Dashboard now summarizes collection totals, missing covers, missing identifiers, recent syncs, failed syncs, open Plex reconciliation conflicts, provider status, upcoming Events, and recent activity.
- `/dashboard` now opens Dashboard by default for normal platform and homelab users.
- Added Dashboard navigation and quick links back into Import, Library, Events, and Integrations.
- Documented the Dashboard summary endpoint in OpenAPI and mapped PAT access to `media:read`.
- Promoted `Dashboard / Command Center` out of the backlog and into the `3.5.0` roadmap.

## Breaking changes

- None.

## Environment/config changes

- No new environment variables.
- No new provider credentials are required.

## Migration and data impact

- No database migration.
- Existing collection, sync, event, activity, and integration rows are read to build the Dashboard summary.
- No existing rows are changed by this release.

## Security triage

- Dashboard summary is authenticated.
- The endpoint uses existing active-scope enforcement.
- Personal access tokens can read it with `media:read`; write access is not introduced.
- Provider secrets remain masked and server-side.

## Security vulnerability triage

- High findings: none in local dependency audits.
- Owner: collectZ maintainers.
- Target remediation milestone: not needed for this release because no high findings were present locally.

## Deployment and verification

- Verify `/api/health` reports frontend/backend/build `3.5.0`.
- Verify `GET /api/dashboard/summary` returns `200` for an authenticated user and includes `collection`, `attention`, `providers`, `upcoming_events`, and `recent_activity`.
- Verify `/dashboard` opens Dashboard and shows `Needs attention`, `Provider health`, `Recent syncs`, and `Upcoming events`.
- Verify Help > Releases serves `3.5.0`.

## Rollback guidance

- Revert the `3.5.0` code and docs changes to restore the previous dashboard default of Movies and remove the Dashboard summary API/view.

## Known issues and follow-up

- This is the foundation slice, not the full future review system.
- Unified Review Queue, Smart Collections, Universal Search, and Human-Readable Activity Timeline remain backlog items.
- CI-only secure-cookie compose smoke, secret scan, and image security/SBOM gates still run in GitHub Actions.
