# collectZ v3.4.153

## Version and date

- Version: `3.4.153`
- Date: `2026-05-10`

## Milestone target and status

- Milestone: `3.4.153 — Kavita Comic Series Title Normalization and Issue Mapping`
- Status: `released`

## Summary

This patch improves Kavita comic imports when Kavita exposes an issue-shaped comic file as a series row. collectZ now recognizes titles such as `Alpha Flight#130 - The Hollow Man! - Unknown`, keeps the raw Kavita title for troubleshooting, and stores cleaner comic series and issue metadata.

## What changed

- Normalizes issue-like Kavita comic series titles into cleaner display titles, series names, issue numbers, and issue titles.
- Keeps true Kavita comic series rows at the series level instead of copying first-chapter issue number and cover date onto the series row.
- Prevents chapter fan-out from creating a duplicate issue row when the Kavita series row already represents an issue-like comic.
- Preserves raw Kavita series/localized/original/sort names plus parse status metadata for troubleshooting.
- Extends the Kavita import smoke fixture with the real `Alpha Flight#130 - The Hollow Man! - Unknown` pattern.

## Breaking changes

- None. Existing Kavita provider ids and raw Kavita metadata remain available.

## Environment/config changes

- Adds a non-production-only `x-import-enrichment-mode: skip` test header so import smokes can avoid external enrichment provider timeouts. Normal imports still use live enrichment behavior.

## Migration and data impact

- No schema migration.
- New Kavita comic imports may use cleaner title/series/issue metadata for issue-like series rows.
- Existing imported rows are not rewritten by this patch; a re-sync or later repair task is needed to refresh older affected rows.

## Known issues and follow-up

- This patch recognizes hash-number issue-style Kavita series names first. Broader annual/special/non-standard numbering handling remains separate backlog work.
- Existing affected Kavita rows are not automatically repaired.
- Live external comic enrichment remains enabled for normal imports and can still be affected by provider availability.
- CI-only secure-cookie compose smoke, secret scan, and image security/SBOM gates still run in GitHub Actions.
