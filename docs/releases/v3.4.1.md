## Summary
This patch builds the foundation for separating Art from Collectibles without forcing an immediate read cutover. Art still works through the bridge-era `/api/art` surface, but new and edited Art records now dual-write into dedicated native Art storage. Events also gain a shared purchased-item relationship so tracked purchases can link to either native Art rows or non-Art Collectibles through one event-facing contract.

## What changed
### Native Art storage foundation
- Added dedicated `art_items` storage for native Art records.
- Kept a bridge pointer through `source_collectible_id` so the current Art surface can coexist with native storage.
- Preserved Art-native fields:
  - `artist`
  - `series`
  - `title`
  - `vendor`
  - `booth`
  - `price`
  - `exclusive`
  - `notes`
  - image and ownership metadata
- Added native Art indexes and updated-at trigger parity.

### Bridge-era Art dual-write
- Updated `/api/art` create and edit flows to create or update the matching native Art row.
- Exposed `native_art_id` on bridge-era Art responses.
- Archived the mapped native Art row when a bridged Art record is archived or reclassified away from Art.
- Fixed scoped Art and Collectibles reads after the native Art join by qualifying shared scope columns.

### Shared event purchased-item contract
- Added `event_purchased_items` as the canonical tracked purchase relationship for Events.
- Added event-scoped purchased-item CRUD:
  - `GET /api/events/:id/purchased-items`
  - `POST /api/events/:id/purchased-items`
  - `PATCH /api/events/:id/purchased-items/:purchasedItemId`
  - `DELETE /api/events/:id/purchased-items/:purchasedItemId`
- Supported tracked purchase links for:
  - `art`
  - `collectible`
- Preserved stable event display snapshot fields:
  - `title_snapshot`
  - `vendor_snapshot`
  - `booth_snapshot`
  - `price_snapshot`

### Contract and verification
- Updated validation and OpenAPI for native Art and shared purchased-item contracts.
- Added Docker-backed smoke coverage for:
  - Art dual-write into native storage
  - event purchased-item creation
  - event purchased-item readback
  - snapshot update
  - archived purchased-item removal
- Kept `event_artifacts` as the freeform event-history lane for notes, ad-hoc purchases, freebies, signings, and other non-canonical event memories.

## Breaking changes
- None intended.
- Art reads still use the bridge-era `/api/art` surface in this release.
- Full existing-row Art migration and read cutover are intentionally deferred to later `3.4.x` milestones.

## Environment/config changes
- No new required operator configuration in `3.4.1`.

## Migration and data impact
- Adds migration `v74 — Add native art storage and shared event purchased item links`.
- Adds new tables:
  - `art_items`
  - `event_purchased_items`
- Existing `collectibles.subtype = art` rows are not bulk-migrated in this release.
- New and edited Art records through `/api/art` dual-write native Art rows going forward.

## Rollback guidance
- Roll back application images to `3.4.0` if Art creation no longer works through the bridge surface, if native Art dual-write corrupts Art metadata, or if event purchased-item links fail to resolve tracked Art and Collectibles correctly.
- Because this release adds storage, rollback should keep the database in place and restore the prior application image unless a maintainer explicitly chooses a database restore path.
