# collectZ v3.4.89

## Kavita External Reader Launch Contract

This patch adds a read-only launch contract for Kavita-backed book and comic imports so collectZ can point users to Kavita's native series or reader views without embedding Kavita, proxying pages, syncing progress, or writing metadata back.

### What changed

- Added Kavita web launch URL helpers that build native series and first-chapter reader routes from the configured server base URL plus Kavita library, series, chapter, and format ids.
- Stored `kavita_series_url`, `kavita_launch_url`, `kavita_launch_label`, and `kavita_launch_target` in provider detail metadata for Kavita imports.
- Prefer first-chapter native reader links when volume/chapter detail is available, with a series-detail fallback when a reader target cannot be derived.
- Surfaced `Read in Kavita` / `Open in Kavita` actions in book and comic detail views while hiding the raw launch-contract fields from generic type details.
- Added smoke and unit coverage proving launch URLs are secret-free and route comics/images through manga reader links, EPUBs through book reader links, and PDFs through pdf reader links.
- Kept embedded iframe reading, page streaming/proxying, progress sync, metadata writeback, per-space Kavita administration, and shared provider abstraction work out of scope.
