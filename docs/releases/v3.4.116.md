## Version

## Milestone

- Roadmap slice: `3.4.116 — Plex Now Playing UI Readback`
- Status: closed after local verification

## Summary

This release adds a read-only Plex now-playing view inside Integrations. The Plex settings panel can now call the saved now-playing readback endpoint and show sanitized active-session rows without adding webhooks, scheduled sync, watch-state writes, or import changes.

## What Changed

- Added an `Active Sessions` action to the Plex Integrations panel.
- Displayed sanitized Plex session rows with title, type, progress, user label, and player state/platform.
- Added an explicit `No active Plex sessions.` empty state after checking now-playing readback.
- Reused the same platform/workspace endpoint-base wiring as the existing Plex diagnostics.
- Preserved existing Plex provider discovery, section discovery, imports, and now-playing endpoint behavior.

## Compatibility

- No database migration.
- No dashboard surface is added in this slice.
- Existing Plex imports and readback endpoints remain unchanged.

## Deployment Notes

- Build and deploy backend/frontend images with `APP_VERSION=3.4.116`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.116`.
- Verify Help > Releases serves `v3.4.116` as the latest release.

## Rollback

Roll back to `v3.4.115` if the now-playing UI readback needs to be removed. The backend readback endpoint and Plex import behavior are unchanged by this slice.

## Known Issues and Follow-Up

- This does not add Plex webhooks or scheduled sync.
- This does not write watch-state back to Plex.
- This does not create a dashboard Now Playing widget.
- Real PMS session payloads may vary by Plex server version, player type, and active session state.
