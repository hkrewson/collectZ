## Summary

This patch continues the maintained-source CodeQL baseline cleanup by hardening file/HTTP boundaries, removing a Playwright storage-state race, and reshaping Kavita connector calls so CodeQL and the intended trusted-connector boundary agree.

## What changed

- Removed the Playwright storage-state check-then-write race by reading the auth state directly and handling missing files through `ENOENT`.
- Added strict generated-key validation and path confinement for backend local upload writes and reads.
- Restricted the Portainer template builder to the two committed HTTPS upstream template sources and confined generated output writes to `portainer-templates/dist`.
- Changed the Kavita connection test/sample calls to use fixed endpoint paths with the normalized admin-configured base URL in Axios config.
- Clarified the Graylog evidence script's runtime-secret naming around the required `GRAYLOG_ROOT_PASSWORD_SHA2` setting.
- Updated source-shape unit coverage for the CodeQL request-forgery boundary expectations.
