# collectZ v3.4.146

- Version: `3.4.146`
- Status: `released`

## Summary

This patch adds a read-only Plex provider item-listing discovery proof. It checks whether `/media/providers` can expose safe content listing candidates before any future migration away from the current legacy section import path.

## Changes

- Parses sanitized Plex provider `Feature.Directory` entries from `/media/providers` responses.
- Extracts provider-advertised movie/show content listing candidates without exposing tokens, absolute provider URLs, private IPs, or file paths.
- Adds a fake-PMS smoke that proves candidate discovery is read-only and does not call import endpoints.
- Keeps Plex import, reconciliation, watched-state writeback, rating writeback, and automatic sync behavior unchanged.
- Documents that legacy Plex import remains current until real-server provider item rows prove identity, artwork, variant, episode, watched-state, and rating parity.

## Security vulnerability triage

- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- This change is read-only and ignores unsafe absolute provider URLs or token-bearing provider directory keys.
