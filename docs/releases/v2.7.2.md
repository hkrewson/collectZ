## Summary
This patch release clears the remaining backend parser vulnerability path left after `2.7.1`. It updates the backend S3/XML dependency chain so the shipped runtime no longer carries the vulnerable `fast-xml-parser` copy that had been pulled in through the AWS SDK XML builder path.

## What changed
### Backend
- Bumped `@aws-sdk/client-s3` to the current safe 3.x line so the transitive `@aws-sdk/xml-builder` dependency resolves to a fixed parser chain.
- Bumped the direct backend `fast-xml-parser` dependency to `^5.5.8`.
- Regenerated the backend lockfile so both the direct dependency and the AWS XML builder path resolve `fast-xml-parser@5.5.8`.

### Frontend
- No functional frontend changes beyond version sync to `2.7.2`.

### Docs and release evidence
- Refreshed local dependency-audit artifacts.
- Refreshed SBOM artifacts.
- Recorded a new local preflight result in `preflight-go-no-go.md`.
