# collectZ v3.7.0

## Version and date

- Version: `3.7.0`
- Date: `2026-05-17`

## Milestone target and status

- Milestone: `3.7.0 — Human-Readable Activity Timeline`
- Status: `implemented`

## Summary

This minor milestone turns the existing technical activity feed into a readable timeline for people using the app day to day, while keeping the raw audit details available when operators need them.

## What changed

- Promoted `Human-Readable Activity Timeline` from the product backlog into the `3.7.0` roadmap.
- Reworked the shared Activity surface so Admin Activity and Workspace Activity show readable timeline entries first.
- Added timeline copy for imports, scanner imports, Plex and Kavita provider activity, ratings, watched-state changes, valuation refreshes, library changes, wishlist conversions, workspace people/invite activity, security events, and failures.
- Added filters for imports, providers, library, events, people, security, and failures.
- Moved raw action names, exact timestamps, IPs, statuses, reasons, and JSON details behind a `Technical details` disclosure.
- Moved the working app/version metadata to `3.7.0` so the minor milestone is aligned across app meta, packages, public compose defaults, release notes, and the in-app release feed.

## Breaking changes

- None.

## Environment/config changes

- No new environment variables.

## Migration and data impact

- No database migration.
- Existing `activity_log` rows are reused; historical records remain intact.

## Security triage

- The timeline uses the existing scoped activity endpoints, so workspace and support-access boundaries are unchanged.
- Technical audit details remain behind an explicit disclosure and continue to use the existing audit-detail sanitization path.
