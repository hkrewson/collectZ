## Summary
This minor release adds the first full operator-facing manual merge workflow across supported same-type media. Operators can now discover likely duplicates, review exact pairs inline, apply or revert merges, reject or defer bad matches, inspect suppressed history, work through comic duplicate runs more efficiently, and rely on preserved identity aliases so future provider and Plex syncs can resolve back to canonical merged records instead of recreating obvious duplicates.

## What Changed
### Manual merge workflow
- Added operator-only same-type merge preview, apply, and revert contracts for supported media families.
- Added inline pair review inside merge-review lanes so operators can work inside discovery, recommended, comic, suppressed, and collection queues without context switching.
- Added activity evidence for manual merge apply events and shared drawer-visible merge provenance across media drawers.

### Recommendation and discovery review
- Added conservative merge recommendations, discovery queue surfacing, comic duplicate candidate review, and duplicate collection review.
- Added operator outcomes for reject, defer, suppressed-history review, and restore-to-queue.
- Tightened false-positive suppression for structured title families and conflicting movie identity fields while preserving good messy matches such as packaging-heavy movie variants.

### Throughput and durability
- Added comic cluster pair selection, auto-advance, skip/defer handling, and remaining-count guidance for issue-level duplicate runs.
- Preserved absorbed provider and Plex identities as canonical alias evidence during manual merge apply.
- Added Docker-backed runtime proof that provider-item and Plex re-syncs can update canonical merged rows through preserved alias identity instead of recreating duplicates.

## Breaking changes
- None intended for operators or end users.
- `3.2.0` adds new operator/admin merge capabilities and review surfaces; it does not change the normal end-user library workflow into a general-purpose merge UI.

## Environment/config changes
- No new operator-facing environment variables are required for `3.2.0`.

## Migration and data impact
- `3.2.0` extends the operator merge and recommendation feedback model, including persisted deferred outcomes and alias-preserved identity evidence used during later re-syncs.
- Manual merge actions can intentionally rewire duplicate records to a canonical row, but they remain preview-first, same-type only, evidence-backed, and revert-safe through the repair-history model.
