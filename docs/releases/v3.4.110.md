# collectZ v3.4.110

## Version and date

- Version: `v3.4.110`
- Date: `2026-05-07`
- Status: `Ready for CI release gates`

## Milestone target and status

- Roadmap slice: `3.4.110 — Release Channel Automation and Stable Promotion`
- Status: implementation complete locally; CI release gates must pass before tag/publish.

## Summary

This release formalizes the collectZ release cadence for homelab users. `latest` remains the fast-moving release channel, while `stable` becomes an intentional maintainer-promoted channel after a release has passed CI, soaked in real homelab use, and has no known blocker open.

## What changed

### Backend

- No backend runtime API behavior changed.
- Added source assertions that keep release-channel automation, security policy wording, and release-channel docs aligned.

### Frontend

- No frontend runtime UI behavior changed.
- Version metadata now points Help and health surfaces at `v3.4.110`.

### Database

- No schema or data migration changes.

### Docs/Ops

- Added `SECURITY.md` with supported release channels and vulnerability reporting guidance.
- Documented `latest`, `stable`, exact semver pins, moving minor tags, and stable promotion rules.
- Added a manual `Promote Stable Images` workflow that verifies the requested release tag, release note, successful publish workflow, and backend/frontend exact-version images before retagging existing image digests as `stable` and `stable-<major.minor>`.
- Updated homelab deployment docs to recommend `IMAGE_TAG=stable` for the safest channel, `IMAGE_TAG=latest` for newest releases, or exact semver tags for pinned deployments.

## Breaking changes

None.

## Environment/config changes

- No new required environment variables.
- `IMAGE_TAG=stable` is now documented as the recommended homelab channel.

## Migration and data impact

None.

## Deployment and verification

- Build and deploy backend/frontend images with `APP_VERSION=3.4.110`.
- Verify `/api/health` reports frontend, backend, and build version `3.4.110`.
- Verify Help > Releases serves `v3.4.110` as the latest release.
- Verify the `Promote Stable Images` workflow syntax and source assertions pass before using it to promote a release.

## Rollback guidance

- Roll back to the previous exact image tag if release-channel documentation or workflow behavior needs to be reverted.
- The stable-promotion workflow only retags existing image digests and does not mutate database state.

## Known issues and follow-up

- Stable promotion is intentionally manual; maintainers still decide when a release has soaked long enough to become `stable`.
- The workflow verifies release artifacts and exact-version images, but the actual production promotion still depends on GHCR and GitHub Actions availability.
