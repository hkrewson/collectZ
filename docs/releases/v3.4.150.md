# collectZ v3.4.150

## Version and date

- Version: `3.4.150`
- Date: `2026-05-10`

## Milestone target and status

- Milestone: `3.4.150 — Plex Provider-Advertised Sections Root Runtime Migration`
- Status: `released`

## Summary

This patch moves Plex section discovery onto the provider-advertised sections root at runtime. When Plex advertises `/library/sections/all` through the official library provider, collectZ uses it; when it is absent or unavailable, collectZ keeps the existing `/library/sections` fallback.

## What changed

- Updates Plex section discovery to resolve the sections root from `/media/providers`.
- Uses `/library/sections/all` when the official Plex library provider advertises it.
- Falls back to `/library/sections` if provider discovery fails, the root is not advertised, or the advertised root is unavailable.
- Adds a fake-PMS smoke proof covering advertised-root success, advertised-root failure fallback, and non-advertised fallback.
- Keeps Plex item import, reconciliation, webhooks, watched-state writeback, rating writeback, and automatic sync behavior otherwise unchanged.

## Breaking changes

- None.

## Environment/config changes

- None. Existing Plex base URL, token, library section, webhook, reconciliation, and writeback settings continue to apply.

## Migration and data impact

- No database migration or data backfill is required.
- Plex section discovery may make one additional read-only `/media/providers` request before section listing.
- Existing Plex item imports continue to use documented Plex library paths for item rows and metadata.

- Existing saved Plex integration settings and imported media rows do not need cleanup for rollback.

## Known issues and follow-up

- Provider item-listing migration remains intentionally out of scope unless a future real-PMS proof finds provider-advertised item rows with identity, metadata, and repeat-sync parity.
- CI-only secure-cookie compose smoke, secret scan, and image security/SBOM gates still run in GitHub Actions.
