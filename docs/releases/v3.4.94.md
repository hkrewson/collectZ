# collectZ v3.4.94

## Kavita Workspace-Owned Integration Administration Contract

This release defines Kavita administration as workspace-owned integration behavior before moving settings storage and UI.

### Shipped

- Added a dedicated workspace-owned Kavita administration contract.
- Defined workspace-admin ownership for Kavita save, test, import, fan-out, readback, and clear actions.
- Defined cross-workspace isolation expectations for overlapping Kavita series and chapter ids.
- Documented legacy platform-config migration recommendations.
- Linked the contract from the Kavita setup guide.

### Boundaries

- This is a contract slice only; workspace-owned Kavita storage, UI, and migration implementation remain future work.
- Kavita remains read-only.
- Embedded reading, reader page proxying, progress sync, metadata writeback, special-chapter fan-out, and shared provider abstractions remain future work.
