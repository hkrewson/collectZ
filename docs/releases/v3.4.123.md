# collectZ v3.4.123

## Summary

This patch adds Plex webhook receiver administration. Admins can generate, rotate, and revoke a receiver URL for Plex webhooks while the receiver remains contract-only.

## Changes

- Added a token-scoped Plex webhook receiver endpoint for `library.new`, `media.scrobble`, and `media.rate` hints.
- Added admin integration controls to generate, regenerate, and revoke the receiver URL.
- Added receiver status readback for enabled state, last received time, and last event.
- Added request-log redaction for generated Plex webhook receiver tokens.
- Added a Docker-runnable admin smoke that proves invalid tokens are rejected, valid events are accepted, revocation works, and smoke evidence stays secret-free.
- Kept automatic imports, watched-state mutation, and ratings writeback apply behavior out of this release.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- Receiver tokens are generated with a `czpw_` prefix, stored only as hashes, and returned only once at generation time.
- Receiver URLs are redacted from request/error logs.
- Browser-visible receiver status does not expose the raw token.
