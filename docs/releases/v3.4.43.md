# v3.4.43

## Event social platform companion contract

- Version: `3.4.43`
- Target milestone: `3.4.43 — Event Social Platform Companion Contract`
- Date: 2026-04-30

### What changed

- Added `GET /api/events/:id/companion/today`, a compact event-social planning snapshot for platform companion clients.
- The companion payload includes event metadata, attendees, groups, meetups, schedule plans, counts, personal ICS sync freshness, cache guidance, privacy rules, and contract metadata.
- The endpoint references the existing Event social write endpoints instead of adding duplicate companion-specific write APIs.
- OpenAPI now documents `EventCompanionTodayResponse` for generated clients.
- Added companion-contract documentation for Apple/platform app planning.

### Why it matters

This gives the platform app conversation a stable backend shape to build against without blurring the product boundary. The web app remains the main planning/admin surface, while platform clients can start with a fast day-of-con companion snapshot.

### Security and privacy notes

- Raw personal Sched ICS URLs are not returned by the companion snapshot.
- No realtime location, presence, broad social discovery, push notification, or full schedule catalog behavior was added.
- Offline guidance is read-only snapshot mode; the backend remains authoritative.
