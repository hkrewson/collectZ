## Summary

This patch keeps the mobile header cleanup moving by reducing secondary filter bulk on dense utility pages. Wishlist and Capture Inbox now keep search and primary actions immediately visible while putting secondary filters behind a compact mobile readback control.

## What changed

- Added a shared mobile secondary-filter disclosure primitive.
- Moved Wishlist type filtering behind a compact mobile readback while keeping desktop controls visible.
- Moved Capture Inbox type, source, and review filters behind a compact mobile readback while keeping status primary.
- Preserved live search behavior and removed redundant mobile Search-button chrome from these utility headers.
- Added a backlog entry for the remaining header/search refinement work discovered during the UI/UX pass.
