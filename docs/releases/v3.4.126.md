# collectZ v3.4.126

## Summary

This patch lets collectZ automatically process queued Plex `library.new` webhook import hints on a lightweight backend cadence.

## Changes

- Added a backend auto-processor for queued `plex_webhook_import_hint` jobs.
- Preserved queued Plex webhook hints across backend restarts instead of treating them as abandoned import jobs.
- Added admin readback for the auto-processor runtime, last run, and batch settings.
- Reused the existing single-rating-key Plex import path from `3.4.125`.
- Kept watched-state sync, rating writeback apply behavior, scheduled full-library reconciliation, and broad Plex import rewrites out of scope.

## Security vulnerability triage

- High findings: none identified locally; dependency preflight remains the local source for dependency audit status.
- Owner: collectZ maintainers.
- Target remediation milestone: none required for this release.
- Webhook receiver tokens remain hash-only at rest and redacted from request/error logs.
- Auto-processor evidence stores sanitized job status and counts only; it does not expose Plex tokens, PMS URLs, raw PMS payloads, or media file paths.
