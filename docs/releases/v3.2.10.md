## Summary
This patch release makes comic provider re-enrichment easier to scan by adding compact cover thumbnails directly in the edit drawer lookup results. The lookup rows now reuse the provider image data already returned by Metron and other search flows, keep the result list compact, and fall back cleanly when a result has no image.

## What Changed
### Compact lookup thumbnails
- Added a compact thumbnail treatment to provider lookup results in the edit drawer overlay.
- Reused the existing image fields already present in lookup payloads instead of introducing any new provider or API contract.
- Tightened the thumbnail corners to a more deliberate utility treatment so the results feel functional rather than soft or card-like.

### Graceful no-image fallback
- Added a lightweight fallback tile for results that do not include an image.
- Kept the current title, source badges, subtitle details, and `Apply` action behavior intact.

### Regression coverage
- Added source-backed regression coverage proving the lookup overlay still includes the compact thumbnail rendering contract.
- Rebuilt the platform and homelab frontend bundles so the shipped app includes the new result-row treatment.
