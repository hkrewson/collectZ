# v2.8.5

## Version and date
- Version: `2.8.5`
- Date: `2026-03-29`

## Milestone target and status
- Target milestone: `2.8.5 — Navigation Shell Cleanup and Integrations Surface Simplification`
- Status: `Completed`

## Summary
This release simplifies the shell and Integrations surface so admin configuration reads more like supported product setup and less like a low-level control panel. The stale sidebar scope card is gone, Integrations now uses lighter horizontal tabs, and redundant or overly technical integration settings were removed or derived from known provider behavior.

## What Changed
### Navigation shell cleanup
- Removed the old active-space/active-library card treatment from the sidebar shell.
- Kept only the lighter library selector behavior where it remains meaningful for non-admin tenant use.

### Integrations surface simplification
- Converted Integrations to the same calmer horizontal-tab language used elsewhere in admin surfaces.
- Flattened the remaining heavy card framing so the active integration pane reads like the rest of Settings.
- Moved `Metrics` and `External Logs` into first-class Integrations tabs instead of keeping them as a detached operator-export card.

### Integration config cleanup
- Removed redundant integration toggles that no longer represented meaningful product choices:
  - barcode lookup
  - cover recognition / OCR gating
- Removed single-provider preset selectors where there was only one real supported source:
  - Vision
  - TMDB
  - Plex
  - Books
  - Audio
  - Games
- Removed editable low-level protocol fields such as custom key-header/query-param controls and dropped the unsupported `custom` preset path from the admin UI.
- Shifted known provider protocol details back into backend-owned preset behavior for supported integrations.

### CWA, Plex, Comics, and Games cleanup
- Derived `CWA Base URL` from the OPDS feed URL instead of requiring a second host field.
- Removed CWA timeout as an admin-facing setting and normalized it to the backend default.
- Removed Plex server-name editing from the admin UI.
- Simplified Metron credentials so username and password sit together and the password field reads as a password instead of a token hybrid.
- Tightened paired field layouts for CWA credentials, Games credentials, and Plex connection settings so related values sit together on desktop.

## Breaking Changes
- Admin Integrations no longer supports arbitrary/custom protocol authoring through editable key-header or query-param fields.
- Several legacy settings remain stored for backward compatibility but are no longer first-class admin controls:
  - CWA base URL
  - CWA timeout
  - Plex server name
  - barcode/OCR integration toggles
- Runtime-owned observability endpoint configuration still remains outside the UI and is deferred to the later observability control-plane milestone.

## Environment and Config Changes
- Removed operator-facing env guidance for feature-flag precedence on:
  - metrics export
  - external log export
- Removed operator-facing env/setup guidance for manual `CWA_BASE_URL` configuration.
- No new required environment variables were introduced in this release.

## Migration and Data Impact
- No schema migration was added for `2.8.5`.
- Existing stored integration settings are normalized by the backend response/write path so unsupported legacy values do not continue to shape the active admin experience.
