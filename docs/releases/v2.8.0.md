## Summary
This release completes the focused UI refinement sprint across the admin, global, and space-management surfaces. It clarifies navigation scope, simplifies the settings/control-plane experience into flatter page layouts, and streamlines member, space, and integration workflows so they read more like a coherent product control surface and less like a collection of older admin tools.

## What changed
### Navigation and information architecture
- Reorganized the sidebar so management surfaces are split more clearly between:
  - `Admin`
  - `Global`
- Moved the active-space management surface into the clearer `My Space` admin context.
- Renamed and regrouped global controls around:
  - `All Spaces`
  - `All Members`
- Removed the visible `Import Review` navigation entry while keeping its later phased retirement documented separately.

### Settings and integrations
- Simplified `Settings` into a flatter preferences-style page.
- Removed low-value density controls from the visible settings UI.
- Moved app-level feature controls and integration-facing controls into more appropriate homes.
- Reduced duplicate integration controls by placing `Barcode Lookup` and `Cover Recognition` toggles on their respective integration sections instead of treating them as separate top-level rows.

### Global and active-space management
- Flattened `All Spaces`, `All Members`, and `Activity` so they use the newer settings-style visual language.
- Added a drawer-based workflow for `All Spaces` owner recovery and space-scoped onboarding actions.
- Simplified active space management:
  - flatter layout
  - clearer copy
  - `Members` / `Invitations` tab treatment
  - more direct row-based management actions

### Feature control cleanup
- Hid permanently enabled rollout flags from the operator-facing feature control surfaces.
- Kept those baked-in behaviors enabled in the backend while scheduling the deeper retirement work for a later milestone.

### Docs and roadmap
- Updated roadmap planning for:
  - baked-in feature-flag retirement follow-up
  - Import Review retirement and debug diagnostics follow-up
- Maintained the `2.8.0` UI review checklist as the active planning artifact for this sprint.
