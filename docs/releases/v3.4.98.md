## Kavita Metadata Writeback Apply

## Summary

This patch adds the first explicit Kavita metadata writeback apply path. Workspace admins can review the Kavita preview diff and then manually write changed, unlocked collectZ metadata fields to the linked Kavita series or chapter.

## What Changed

### Shipped

- Added a workspace-admin-only Kavita metadata apply endpoint for Kavita-linked media rows.
- Recompute the preview diff immediately before writing so locked, missing, and unchanged fields are not sent.
- Added Kavita series and chapter metadata mutation helpers targeting `POST /api/Series/metadata` and `POST /api/Chapter/update`.
- Added a guarded `Apply to Kavita` action beside the media-detail Kavita Metadata preview.
- Extended the Kavita import/sync smoke fake server to prove series and chapter apply payloads without mutating a real Kavita server.
- Added activity/audit events for applied, skipped, and failed writeback attempts.
