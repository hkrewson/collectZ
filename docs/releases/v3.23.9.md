## Summary

This release continues the maintainability guardrails work where the current quality report shows clear value: `ArtView.jsx` was the top remaining frontend lint pocket after `3.23.8`. The slice clears Art view accessibility and hook-baseline findings without changing Art product behavior.

## What changed

- Removed the stale default React import from the Art view.
- Made Art list rows keyboard reachable while preserving row-level open behavior and edit/delete actions.
- Replaced the Art editor backdrop click target with a native button control.
- Documented intentional Art state synchronization effects for artist lookup, detail loading, editor form resets, event option loading, list loading, and external focus targets.
- Reduced the full frontend quality baseline from `70` errors / `33` warnings to `60` errors / `32` warnings.
